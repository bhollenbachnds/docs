
These properties allow you to **limit access** to specific web pages when using an **interactive web page element**. This prevents users from navigating freely across the **Internet** and restricts them to **approved content**.  

---

## **Web Page Restrictions Settings**  

### **Prevent Unwanted Web Page Views**  
- **Enable or disable** web page restrictions.  
- When enabled, only **approved websites** will be accessible.  

### **Allowed Web Pages**  
- Define a **list of permitted websites** for users.  

---

## **Managing Allowed Websites**  

### **To Add a Website:**  
1. Click **Add**.  
2. Enter the **URL** of the website to allow.  
3. Click **OK**.  

### **To Edit a Website:**  
1. Select the **existing website** from the list.  
2. Click **Edit**.  
3. Make the necessary changes and **click OK**.  

### **To Remove a Website:**  
1. Select the **website** from the list.  
2. Click **Remove** to delete the entry.  

---

## **Best Practices & Tips**  

- **Use Generic URLs**:  
  - Example: Adding **"nds.eu"** allows all pages under that domain (e.g., **nds.eu/products, nds.eu/about**).  
  - Adding **"nds.eu/products"** only allows access to the **products** page.  

- **Remove "www." in URLs**:  
  - Some websites **automatically redirect** (e.g., from **www.example.com** to **example.com**).  
  - Entering just **"example.com"** ensures access is not blocked due to **redirect mismatches**.  

---

By setting **web page restrictions**, you can control **which sites are accessible**, ensuring a **safe and focused** browsing experience within your **interactive presentation**.
