
The **Video Stream URL Editor** assists in **structuring and configuring** the correct URL format for streaming media. It ensures compatibility with **entertainment and communication systems** by managing streaming server connections effectively.

---

## **Preview**  
- Displays the **complete URL** based on the **settings entered**.  
- Updates dynamically as parameters are modified.  

---

## **Connection Information**  

| **Field** | **Description** |
|-----------|---------------|
| **Server IP/Name** | Enter the **IP address** or **hostname** of the streaming server. |
| **Port Number** | Define the **port used for broadcasting** (e.g., `554` for RTSP, `8080` for HTTP). |
| **Stream Path** | Specify the **optional path** where the stream is hosted (e.g., `/live/stream1`). |
| **Protocol** | Choose the **streaming protocol** (RTSP, RTP, UDP, MMS, HTTP). |

---

## **Credentials**  
If the stream **requires authentication**, enter:  
- **Username**  
- **Password**  

This ensures **secure access** to protected streams.

---

## **Options**  

| **Option** | **Description** |
|-----------|---------------|
| **Ignore Audio** | Prevents PADS4 from waiting for an **audio signal** if one **is not available**. Useful when a stream **reports an audio track but has none**. |
| **Query Parameters** | Add **custom query parameters** required by the stream. |

---

### **Troubleshooting Tip**  
If the **video stream does not display**, try **enabling "Ignore Audio"**. Some streams falsely report an audio signal, causing **playback delays or failures**.
