
The **body** of the table is the **main section** where **multiple rows of data** are displayed. These properties allow you to configure the **layout, height behavior, column structure, and background appearance**.

---

## **Body Layout & Row Settings**  

### **Edit Row Content**  
- **Defines the layout of a single row** that is **repeated** in the table body.  
- Clicking **Edit** allows you to **customize row properties**, such as adding **text, images, data fields, or formatting styles**.  
- For detailed row configurations, refer to **Row Properties**.

---

## **Height Settings**  

The **height of the table body** can be controlled using the following options:

| **Option** | **Description** |
|------------|----------------|
| **Fixed Height** | The body height remains at a **fixed value**, unaffected by header or footer changes. |
| **Top Shrink/Grow** | The **top of the body adjusts** based on header/footer size, while the **bottom remains fixed**. |
| **Bottom Shrink/Grow** | The **bottom of the body adjusts** based on header/footer size, while the **top remains fixed**. |

---

## **Column Structure**  

### **Split into 2 Columns**  
- **Divides the table width** into **two sections** to fit more data in **limited space**.  
- Useful when displaying **short data entries** to optimize screen real estate.  
- **Ensure row elements do not exceed half the table width** to prevent overlapping.

#### **Example Use Case**  
If your table has **a list of names and numbers**, using **two columns** allows **more entries** to be displayed simultaneously, reducing the need for scrolling.

---

## **Background Settings**  

### **Edit Background**  
- Allows **customization** of the table body background.  
- Can include **solid colors, gradients, textures, or patterns**.  
- For detailed background configurations, refer to **Background Properties**.

---

## **Behavior & Considerations**  

- **Fixed Height + Automatic Shrink/Grow in Header:**  
  - If the **header expands**, the entire body and possibly the **footer** will be **moved down**.  
  - This ensures the table adapts to **dynamic content** in the **header** without disrupting data display.  

- **Using 2 Columns:**  
  - Ensure **row elements fit within half the table width** to prevent layout misalignment.  

---

### **Use Cases**  
✔ **Fixed Height** → Ideal for tables where **consistent row visibility** is required.  
✔ **Shrink/Grow Options** → Best for **dynamic headers** that may **expand or contract** based on content.  
✔ **2 Column Layout** → Optimizes space when **short data entries** need to be **displayed efficiently**.
