
The **Invue POD** provider in **PADS4** allows you to connect to an **Invue Merchandising DIB (Device Interface Box)**, which manages **POD (Point of Display) security devices**. This setup helps in integrating **merchandising security solutions** with **digital signage systems**.

---

## **Steps to Add/Modify an Invue Connection in PADS4**

### **1. Start the Data Provider Wizard**
- Open the **Data Provider Wizard** using the steps in **Adding a Data Provider** or **Editing a Data Provider**.
- Select **Invue POD** as the **provider type**.

---

### **2. Configure Invue POD Connection Settings**
- **Select the COM Port:**  
  - Choose the **COM port** to which the **DIB (Device Interface Box)** is connected.
  - If unsure, check the **Windows Device Manager** to find the correct **serial port**.

Click **Next** to proceed.

---

### **3. Complete the Setup**
- Click **Finish** to finalize the connection.
- The **PADS4 Viewer** will now be able to retrieve real-time **Invue POD data**.

---

## **Important Considerations**
### **PADS4 Data Server Configuration**
- If using **PADS4 Data Server**, ensure that the **Invue DIB device is connected** to the **Data Server**.
- If **not using a Data Server**, the **DIB device must be connected** to the **Viewer PC** instead.

### **Device Connectivity**
- Make sure the **DIB device** is properly installed and recognized by the system.
- The **Invue software drivers** should be installed on the **server or Viewer PC**.

---

## **Use Cases**
- **Retail Security:** Monitor **merchandise security** status.
- **Customer Engagement:** Integrate **product interaction data** into **digital signage**.
- **Loss Prevention:** Alert staff if **security conditions change**.

By following these steps, **PADS4** successfully integrates **Invue POD devices** for **real-time security monitoring and analytics** in **retail and merchandising environments**.
