
The **first step** in the **Data Provider Wizard** is selecting the type of provider that will supply **real-time data** to your presentation.

---

## **Steps to Select a Provider Type**  
1. Open the **Data Provider Wizard**  
   - Follow the instructions in **Adding a Data Provider**  
2. Choose a **Provider Type**  
3. Click **Next** to proceed with configuration  
4. Complete the **Data Provider Wizard**  

---

## **Available Provider Types**  
Each provider type connects to a specific **data source**.  

| **Provider Type** | **Description** |
|------------------|----------------|
| **.NET Framework** | Connects to data sources using **Microsoft .NET**. |
| **Bluefox** | Retrieves content from **Bluefox**. |
| **Cognimatics** | Retrieves **audience measurement data** from **Cognimatics**. |
| **Facebook** | Connects to **Facebook** data feeds. |
| **Flickr Feed** | Retrieves **images** from **Flickr**. |
| **Folder Content** | Retrieves files from a **local folder**. |
| **FTP Folder** | Connects to an **FTP directory** to retrieve files. |
| **Generic ODBC** | Connects to a **data source** using an **ODBC driver**. |
| **Generic OLEDB** | Connects to a **data source** using an **OLEDB driver**. |
| **Google Calendar Feed** | Connects to **Google Calendar** events. |
| **Google+ Feed** | Retrieves **Google+** data. |
| **Intel AIM** | Retrieves **facial recognition data** from **Intel AIM**. |
| **Instagram Feed** | Retrieves **images** from **Instagram**. |
| **JSON Feed** | Connects to an **online JSON feed**. |
| **JSON File** | Retrieves **JSON data** from a local file. |
| **Microsoft Access** | Connects to a **Microsoft Access database**. |
| **Microsoft Excel** | Retrieves data from an **Excel spreadsheet**. |
| **Microsoft Exchange Server** | Connects to **Microsoft Exchange**. |
| **Microsoft SQL Server** | Connects to **SQL Server databases, views, or stored procedures**. |
| **Microsoft SQL Server CE** | Connects to **SQL Server Compact Edition databases**. |
| **Media RSS Feed** | Retrieves data from a **Media RSS source**. |
| **Media RSS File** | Retrieves **Media RSS** content from a local file. |
| **Newmarket Delphi Feed** | Connects to **Newmarket Delphi**. |
| **OData** | Connects to an **Open Data Protocol** data source. |
| **Office 365 Exchange** | Connects to **Microsoft Exchange Online**. |
| **Office 365 Excel** | Connects to an **Excel spreadsheet stored in Office 365**. |
| **Open Office** | Connects to an **Open Office Calc file**. |
| **PADS4 Viewer Connections** | Retrieves the **status of all PADS4 Viewers**. |
| **PADS4FIDS** | Connects to the **PADS4 Flight Information Display System (FIDS)**. |
| **Picasa Feed** | Retrieves **images** from **Picasa**. |
| **PostgreSQL** | Connects to a **PostgreSQL database**. |
| **Quividi Feed** | Retrieves **audience measurement data** from **Quividi**. |
| **Seenspire** | Retrieves content from **Seenspire**. |
| **Static Table** | Uses an **embedded table** inside your presentation. |
| **SQL Server Database** | Connects to a **SQL Server database**. |
| **SQL Server CE Database** | Connects to a **SQL Server Compact Edition database**. |
| **Text Feed** | Retrieves data from a **text-based web feed**. |
| **Text File** | Retrieves **plain text data** from a local file. |
| **Twitter Feed** | Connects to **Twitter** for real-time posts. |
| **RSS Feed** | Retrieves **RSS content** from the web. |
| **RSS File** | Retrieves **RSS data** from a local file. |
| **SOAP Web Service** | Connects to a **SOAP-based web service**. |
| **WMI Class** | Retrieves **Windows Management Instrumentation (WMI) data**. |
| **XML Feed** | Retrieves **XML data** from the internet. |
| **XML File** | Retrieves **XML data** from a local file. |
| **YouTube Feed** | Connects to **YouTube** for video retrieval. |

---

## **Next Steps**  
- Click **Next** to continue the **Data Provider Wizard**  
- Follow the steps in **Configuring Connection Settings**  

---

## **Additional Notes**  
- The **provider type** you select determines the **next configuration steps**  
- **Some provider types require additional drivers** (e.g., ODBC, OLEDB, OData)
