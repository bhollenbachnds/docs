---
title: "PADS4 Modules"
---



PADS4 Digital Signage is made up of several modules that work together to create, schedule, publish, and monitor content across your network. Web modules run in a browser and are ideal for day-to-day content and scheduling work; Windows modules offer advanced design and administration capabilities.

## Web modules

### PADS4 CTRL Center

The central hub for managing and configuring your PADS4 environment. Administrators use it to configure system settings, manage users and permissions, connect integrations, and monitor overall playback health.

### PADS4 CMS

A web-based content management system for creating, organizing, and distributing digital content. Teams can upload media, manage templates, and collaborate on content without installing anything.

### PADS4 Web Designer

A browser-based design tool for building dynamic layouts. It supports multimedia, real-time data, and interactive elements so you can design engaging signage from anywhere.

### PADS4 Web Scheduler

A streamlined online scheduler for planning what plays, where, and when. Use it to assign content to viewers and keep messaging timely across your network.

## Windows modules

Windows modules complement the web tools with advanced capabilities for designers, schedulers, and IT administrators.

### PADS4 Designer

A desktop design application with advanced layout, media, and live-data features for producing high-quality signage content.

### PADS4 Scheduler

A desktop scheduling tool that gives you fine-grained control over content distribution — detailed schedules, per-screen assignments, and automated playback sequences.

### PADS4 Agent

A lightweight application that runs and monitors playback on connected devices. It ensures scheduled content is displayed correctly and reports playback status back to the system.

### PADS4 System Management

An administrator tool for managing system configuration, network settings, and device performance. It gives IT insight into system health and keeps the PADS4 ecosystem running smoothly.

<Tip>
  You don't need every module to get started. Most teams begin with CTRL Center, CMS, Web Designer, Web Scheduler, and Agent, then add the Windows modules as their needs grow.
</Tip>
