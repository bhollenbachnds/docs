Cette section décrit des scénarios d'espace de travail avancés qui peuvent impliquer des configurations spécifiques, des approbations ou un accès délégué.

### **Salles Modulaires**

PADS4 Workspace prend en charge les **salles de réunion modulaires**, qui sont des salles pouvant être combinées ou séparées à l'aide de cloisons mobiles.

Si configuré par les administrateurs, les utilisateurs peuvent :

* Réserver l'une des salles individuelles
* Ou réserver la salle combinée, plus grande, dans son ensemble

Cette configuration est courante pour les espaces de formation flexibles ou les salles de conseil qui peuvent être redimensionnées selon les besoins.

### **Flux de Travail d'Approbation de Réservation**

Certaines salles peuvent nécessiter une **approbation de réservation**. Dans ce cas :

* Le demandeur réserve une salle comme d'habitude
* La demande est envoyée à un **approbateur**
* La réservation reste **en attente** jusqu'à ce qu'elle soit approuvée
* Une fois approuvée, la salle est officiellement réservée et confirmée

> Les étapes détaillées sont décrites dans le **Appendix: Booking Approval Workflow**

### **Booking Request Workflow**

Dans certaines configurations, les utilisateurs sont uniquement autorisés à **soumettre une demande de réservation** plutôt que de sélectionner une salle spécifique.

* Le demandeur remplit les critères souhaités (par ex., date, équipement, services)
* Un **gestionnaire de salle**examine la demande
* Selon les critères, le gestionnaire attribue une salle appropriée et confirme la réservation

> Processus complet expliqué dans le **Appendix: Booking Request Workflow**

### **Délégation**

PADS4 Workspace permet **délégation**, où un utilisateur peut gérer les réservations au nom d'un autre.

Ceci est utile pour :

* Les assistants de direction gérant des calendriers
* Les chefs d'équipe réservant au nom de leur groupe

Une fois configuré, le délégué peut consulter et réserver via le calendrier de l'autre utilisateur.
