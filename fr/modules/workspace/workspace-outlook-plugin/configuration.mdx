Ce guide décrit comment configurer le **PADS4 Workspace Outlook Add-In** pour **les environnements Microsoft Exchange sur site**.

***

## Étape 1 : Obtenir le fichier manifeste

Vous devrez d'abord télécharger le **fichier manifeste du plugin Outlook** depuis le portail **PADS4 CMS Admin**.

### Instructions:

1. Ouvrez un navigateur et accédez à votre serveur :
   ```
   https://<server-fqdn>.pads4.com
   ```

2. Connectez-vous avec un compte administrateur\
   *(identifiants par défaut : Nom d'utilisateur : `Administrator`, Mot de passe : `Admin1234!`)*

3. Sélectionnez **CMS Admin** dans la liste des modules disponibles.

4. Naviguez vers :\
   **Plugins** → **Calendar**

5. Téléchargez le fichier manifeste du plugin en cliquant sur :\
   **Download Outlook plugin Manifest file**

Enregistrez ce fichier dans un emplacement connu.

***

## Étape 2 : Télécharger le module complémentaire sur Microsoft Exchange

Pour déployer le module complémentaire à l'échelle de l'organisation, suivez les étapes ci-dessous.

### Pour Exchange 2016 / 2019 :

1. Ouvrez le **Centre d'administration Exchange (EAC)**\
   Consultez la documentation Microsoft pour accéder à l'EAC.

2. Naviguez vers :\
   `Organization` → `Add-Ins` *(ou `Apps` dans Exchange 2013)*

3. Cliquez sur **+ (Ajouter)** et sélectionnez **Add from file**

4. Cliquez sur **Browse**, localisez le **fichier manifeste** que vous avez enregistré précédemment, et cliquez sur **Next**

Une fois téléchargé, le **PADS4 Workspace Add-In** apparaîtra dans la liste des applications disponibles.

***

## Étape 3 : Rendre le module complémentaire disponible pour les utilisateurs

1. Double-cliquez sur **PADS4 Workspace** dans la liste des applications.

2. Dans la boîte de dialogue **Edit Add-In Settings**, cochez :\
   **Make this app available to users in your organization**

3. Choisissez l'un des modes de déploiement suivants :

* **Optional, enabled by default**\
  Le module complémentaire est activé, mais les utilisateurs peuvent le désactiver.

* **Optional, disabled by default**\
  Le module complémentaire est disponible, mais les utilisateurs doivent l'activer.

* **Mandatory, always enabled**\
  Le module complémentaire est imposé ; les utilisateurs ne peuvent pas le désactiver.

4. Cliquez sur **Save** pour appliquer les paramètres.
