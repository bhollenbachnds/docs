Ce guide explique comment configurer le **PADS4 Workspace Outlook Add-In** pour **les environnements Microsoft Exchange sur site**.

***

## Étape 1 : Obtenir le fichier Manifest

Vous devrez d'abord télécharger le **fichier manifest du plugin Outlook** depuis le **PADS4 CMS Admin** portail.

### Instructions:

1. Ouvrez un navigateur et accédez à votre serveur :

   ```
   https://<server-fqdn>.pads4.com
   ```
2. Connectez-vous avec un compte administrateur
3. Sélectionnez **CMS Admin** dans la liste des modules disponibles.
4. Accédez à :\
   **Plugins** → **Calendar**
5. Téléchargez le fichier manifest du plugin en cliquant sur :\
   **Download Outlook plugin Manifest file**

Enregistrez ce fichier dans un emplacement connu.

***

## Étape 2 : Téléverser le Add-In dans Microsoft Exchange

Pour déployer le add-in à l'échelle de l'organisation, suivez les étapes ci-dessous.

### Pour Exchange 2016 / 2019 :

1. Ouvrez le **Exchange Admin Center (EAC)**\
   Consultez la documentation Microsoft pour accéder à l'EAC.
2. Accédez à :\
   `Organization` → `Add-Ins` *`(or Apps in Exchange 2013)`*
3. Cliquez sur **+ (Ajouter)** et sélectionnez **Add from file**
4. Cliquez sur **Browse**, localisez le **fichier manifest** que vous avez enregistré précédemment, et cliquez sur **Next**

Une fois téléversé, le **PADS4 Workspace Add-In** apparaîtra dans la liste des applications disponibles.

### Pour Office 365 / Exchange Online

1. Ouvrez le Centre d'administration Office 365
2. Accédez à : `Settings` → `Integrated apps`
3. Cliquez sur **Upload custom apps** et sélectionnez `App type` → `Office Add-in`
4. Cliquez sur **Upload manifiest file (.xml) from device** et sélectionnez le **fichier manifest** que vous avez enregistré précédemment.
5. Suivez les étapes requises **(Utilisateur + Déploiement)** pour terminer la configuration.
6. Une fois terminé, le **PADS4 Workspace Add-In** apparaîtra dans la liste des applications disponibles.

***

## Étape 3 : Rendre le Add-In disponible aux utilisateurs

1. Double-cliquez sur **PADS4 Workspace** dans la liste des applications.
2. Dans la **boîte de dialogue Edit Add-In Settings**, cochez :\
   **Rendre cette application disponible aux utilisateurs de votre organisation**
3. Choisissez l'un des modes de déploiement suivants :

* **Facultatif, activé par défaut**\
  Le Add-In est activé, mais les utilisateurs peuvent le désactiver.
* **Facultatif, désactivé par défaut**\
  Le Add-In est disponible, mais les utilisateurs doivent l'activer.
* **Obligatoire, toujours activé**\
  Le Add-In est imposé ; les utilisateurs ne peuvent pas le désactiver.

4. Cliquez sur **Save** pour appliquer les paramètres.

<Note>
  Assurez-vous que les adresses e-mail des utilisateurs qui utilisent le plugin disposent d'un compte workspace.

  Ce compte doit utiliser la même adresse e-mail que l'utilisateur Outlook ; celle-ci est utilisée pour l'authentification avec le backend PADS4, ce qui vous permet d'utiliser le PADS4 Workspace Add-in.
</Note>
