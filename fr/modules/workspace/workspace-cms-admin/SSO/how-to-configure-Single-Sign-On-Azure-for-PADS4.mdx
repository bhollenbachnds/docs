Le processus ci-dessous vous aide à configurer votre installation PADS4 CMS pour utiliser Azure SSO.

<Warning>
  À partir de PADS4 2025.2, cette fonctionnalité fait partie de PADS4 CTRL Center. Pour plus d'informations, voir CTRL Center > Administration > Identity.
</Warning>

## Paramètres Azure

Il est nécessaire de configurer l'instance Azure pour permettre aux utilisateurs de votre organisation d'accéder à l'application CMS PADS4 (Legacy). Pour ce faire, veuillez suivre les étapes de ce document :

## Étape 1 : Accédez à votre page « Azure portal » et connectez-vous en tant qu'administrateur

[https://portal.azure.com/](https://portal.azure.com/)

## Étape 2 : Sélectionnez l'option « Microsoft Entra ID »

![Microsftentra ID Pn](/images/MicrosftentraID.png)

## Étape 3 : Dans le panneau de gauche, sélectionnez « Enterprise Applications »

![Enterprise Applications Pn](/images/EnterpriseApplications.png)

# **Important**

<Warning>
  Si vous \*\*<u>ne souhaitez pas</u> \*\* utiliser un Application Proxy sur votre application locale, continuez à l'étape 4 ci-dessous et ignorez les étapes 7 et 8.

  Si vous \*\*<u>souhaitez</u> \*\*utiliser un Application Proxy pour votre application locale, passez à l'étape 7 et ignorez les étapes 4, 5 et 6.
</Warning>

## Étape 4 : Sélectionnez l'option « + New application » en haut de l'écran

![Step4+newapplication Pn](/images/step4+newapplication.png)

## Étape 5 : Cliquez sur « Create your own application »

![Step5 Pn](/images/step5.png)

## Étape 6 : Remplissez les informations demandées dans le formulaire et cliquez sur créer

![Step6 Pn](/images/step6.png)

<Warning>
  Ignorez cette étape (7) si vous n'utilisez pas d'Application Proxy.
</Warning>

## Étape 7 : Création d'un groupe de connecteurs / connecteur de proxy d'application :

Sélectionnez la **Manage Application Proxy Connectors** **section**

![Step7 Pn](/images/step7.png)

1. Dans le menu Application Proxy, sélectionnez + Download ConnectorService
2. Acceptez les conditions et téléchargez le fichier **<u>AADApplicationProxyConnectorInstaller.exe</u>** sur le PC serveur.
3. Exécutez le programme d'installation et assurez-vous de valider vos identifiants d'administrateur Azure lors de la configuration.

   ![Step8 Pn](/images/step8.png)
4. Une fois l'installation terminée, sélectionnez la **+New Connector** section Group
5. Créez un nom par défaut pour votre connexion. Par exemple : **SSO Connector**
6. Sélectionnez votre machine serveur dans la liste des connecteurs
7. Sélectionnez la région où se trouve votre machine.

   ![Step9 Pn](/images/step9.png)
8. Enregistrez votre nouvelle connexion ici.
9. Retournez à la section Enterprise Applications et sélectionnez **« Add your own on-premises application »**

## Création d'une application locale dans Azure

1. Sélectionnez **Add an on-premises application**

   ![Step10 Pn](/images/step10.png)

<Warning>
  Ignorez cette étape (8) si vous n'utilisez pas d'Application Proxy.
</Warning>

## Étape 8 : Remplissez les informations demandées dans le formulaire et cliquez sur créer.

![Addyouownonpromiseapp Pn](/images/Addyouownonpromiseapp.png)

### Exemple :

Groupe de connecteurs : SSO Connector (par défaut). <u>Il s'agit du groupe de connecteurs que vous venez de configurer</u>

1. Accédez à l'application nouvellement créée en sélectionnant **« Enterprise Applications »** et utilisez l**a barre de recherche** pour trouver votre application. **Sélectionnez-la** puis accédez à « Users and Groups » :

![Userandgroups Pn](/images/userandgroups.png)

2. Ajoutez les utilisateurs et les groupes qui auront accès à l'application en sélectionnant l'option **« Add user/group »** option

![Robinsso Pn](/images/robinsso.png)

![Add2 Pn](/images/add2.png)

## Étape 10 : Pour configurer l'authentification unique (SSO)

Sélectionnez **« Single sign-on option »**  dans le volet de gauche, puis **« SAML »**

![10 Pn](/images/10.png)

## Étape 11 : Sélectionnez « Edit » dans la section « Basic SAML Configuration »

![Image23 Pn](/images/Image23.png)

Sélectionnez « Add identifier » et **« Add Reply URL »** en fournissant la valeur selon l'exemple ci-dessous :

![Image24 Pn](/images/Image24.png)

## Exemple :

**Identifiant** : URL PADS4 CMS locale avec /Saml2 **(ex :** [https://robin.pads365.com/crystal/domain/Saml2](https://robin.pads365.com/crystal/domain/Saml2))

Si un port réseau unique est utilisé, assurez-vous de l'inclure dans la chaîne ci-dessus.

Ex : [https://robin.pads365.com:444/crystal/domain/Saml2](https://robin.pads365.com:444/crystal/domain/Saml2)

**Assurez-vous** de spécifier votre domaine PADS4 à « domain » dans l'URL

![Image25 Pn](/images/Image25.png)

Si vous utilisez le nom de domaine par défaut, ce sera : [https://robin.pads365.com/crystal/pads/Saml2](https://robin.pads365.com/crystal/pads/Saml2)

URL de réponse : URL PADS4 CMS locale avec /Saml2/Acs **(ex :**[**https://robin.pads365.com/crystal/domain/Saml2/Acs**](https://robin.pads365.com/crystal/domain/Saml2/Acs) )

Si un port réseau unique est utilisé, assurez-vous de l'inclure dans la chaîne ci-dessus.

Ex [https://robin.pads365.com:444/crystal/domain/Saml2/Acs](https://robin.pads365.com:444/crystal/domain/Saml2/Acs)

**Assurez-vous** de spécifier votre domaine PADS4 à « domain » dans l'URL

![Image26 Pn](/images/Image26.png)

Si vous utilisez le nom de domaine par défaut, ce sera :

[https://robin.pads365.com/crystal/pads/Saml2/Acs](https://robin.pads365.com/crystal/pads/Saml2/Acs)

<Info>
  **Remarque :**  Si vous avez configuré le SSO sur une version antérieure à 2023.1, le nom de domaine n'était pas requis dans l'URL lors de l'utilisation d'un seul domaine. À partir de la version 2023.1, le nom de domaine est requis dans l'URL pour les configurations à un domaine et à plusieurs domaines.

  Par conséquent, lors de la mise à jour d'une version antérieure à 2023.1 avec le SSO configuré vers la dernière version, assurez-vous d'ajouter le domaine à la fois à l'identifiant et à l'URL de réponse.

  **Assurez-vous d'« Enregistrer » ces configurations.**
</Info>

# Informations requises pour la configuration SSO dans PADS4 CMS

## Afin de configurer SSO dans PADS4 CMS, vous aurez besoin des éléments suivants :

Certificat du serveur de fédération

Identifiant du service de fédération

URL SSO SAML

URL des métadonnées

URL de la partie de confiance et ;

ID de groupe du groupe d'utilisateurs que vous avez assigné à l'application.

## 1. Obtention du « Certificat du serveur de fédération »

Dans votre application, sélectionnez **" Single sign-on "** dans le volet gauche

Faites défiler jusqu'à l'en-tête **" SAML Certificates "** en-tête

Sélectionnez l'option **" Download "** en regard de l'option " Certificate (Raw) ".

![Image32 Pn](/images/Image32.png)

## 2. Obtention de l'« Identifiant du service de fédération »

Dans votre application, sélectionnez **" Single sign-on "** dans le volet gauche

Faites défiler jusqu'à l'en-tête **"Set up" your\_application\_name "** en-tête

Sélectionnez l'option **" Copy "** en regard de l'option **" Azure AD Identifier "** option.

![Image33 Pn](/images/Image33.png)

## 3. Obtention de l'« URL SSO SAML »

Dans votre application, sélectionnez **" Single sign-on "** dans le volet gauche

Faites défiler jusqu'à l'en-tête **" Set up "your\_application\_name "** en-tête

Sélectionnez\*\*" Copy "\*\*   en regard de l'option **" Login URL "**  option.

![Image34 Pn](/images/Image34.png)

## 4. Obtention de l'« URL des métadonnées »

Dans votre application, sélectionnez "**Single sign-on "**  dans le volet gauche

Faites défiler jusqu'à l'en-tête \*\*" SAML Certificates " \*\*

Sélectionnez l'option **" Copy "** en regard de l'option **" App FederationMetadata URL "** option.

![Image35 Pn](/images/Image35.png)

## 5. Obtention de l'« URL de la partie de confiance »

Dans votre application, sélectionnez **" Single sign-on "** dans le volet gauche

Faites défiler jusqu'à l'en-tête **" Basic SAML Configuration "** en-tête

Copiez la chaîne en regard de l'option \*\*" Identifier (Entity ID) " \*\*.

![Image36 Pn](/images/Image36.png)

## 6. Obtention de l'« ID de groupe » du groupe que vous avez assigné à votre application :

<Info>
  Pour **chaque groupe d'utilisateurs ajouté pour utiliser l'application PADS4 CMS, une table de correspondance sera nécessaire pour ajouter les groupes à l'interface. Veuillez fournir aux installateurs les ID de groupe et les noms des groupes depuis Azure afin de garantir que les utilisateurs pourront se connecter.**
</Info>

Sélectionnez **"home"** dans portal.azure.com et sélectionnez **"Microsoft Entra ID"**

Accédez à **"Groups"** dans le volet gauche

Recherchez le groupe que vous avez assigné dans votre application et copiez l'\*\*"Object Id"\*\*

![Image37 Pn](/images/Image37.png)

## 7. Maintenant que toutes les informations ci-dessus ont été collectées, PADS4 CMS peut désormais être configuré pour utiliser SSO.
