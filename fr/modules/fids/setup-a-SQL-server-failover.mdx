Pour garantir une haute disponibilité et une redondance du système, PADS4FIDS prend en charge une **configuration de basculement** utilisant deux serveurs PADS4 avec une base de données SQL partagée et un commutateur DNS API.

Cette configuration garantit que si un serveur PADS4 devient indisponible, le système peut automatiquement rediriger vers le serveur secondaire avec une interruption minimale.

***

## Aperçu

* **Base de données SQL** : Hébergée sur un serveur séparé (pas sur l'un ou l'autre des serveurs PADS4)
* **Deux serveurs PADS4FIDS** : Installés sur des machines séparées
* **Commutateur DNS API** : Redirige automatiquement le trafic entre les serveurs principal et secondaire
* **Accès unifié** : Les Viewers et les opérateurs utilisent une adresse DNS commune

***

## Configuration étape par étape

### Étape 1 : Préparer la base de données SQL

* Configurez un **SQL Server** sur une machine dédiée dans votre réseau.
* Suivez les étapes standard pour **créer une base de données PADS4FIDS** sur ce serveur.
* Assurez-vous que les deux serveurs PADS4 auront accès à ce SQL Server.

***

### Étape 2 : Installer deux serveurs PADS4FIDS

* Déployez **deux serveurs PADS4FIDS séparés** (sur deux machines virtuelles ou physiques).
* Ces serveurs vont **partager la même licence**, mais vous devez contacter l'**équipe commerciale PADS4** pour autoriser cette configuration.
* Référez-vous au **manuel d'installation PADS4 officiel** pour les instructions de configuration.

***

### Étape 3 : Installer un commutateur DNS API

* Installez le **commutateur DNS API** sur une machine qui **n'est pas l'un des serveurs PADS4**.
* Cette machine peut être le serveur SQL ou un autre système capable de **communiquer avec les deux serveurs PADS4 et tous les PADS4 Viewers**.
* Configurez le commutateur DNS API pour :
  * Utiliser le **serveur PADS4 principal** comme point de terminaison par défaut
  * Basculer automatiquement vers le **serveur secondaire** si le principal devient inaccessible

***

### Étape 4 : Configurer l'accès des Viewers et des opérateurs

* Configurez tous les **PADS4 Viewers** pour utiliser l'**adresse DNS** du commutateur API plutôt qu'une adresse IP de serveur fixe.

* L'adresse DNS doit être saisie comme **IP du serveur PADS4** dans les paramètres de connexion du viewer.

* Les ports par défaut restent inchangés sauf indication contraire :
  * **Port 80** (HTTP)
  * **Port 7237** (communication PADS4)

* Les utilisateurs accédant à l'**opérateur PADS4FIDS** doivent également utiliser l'**adresse DNS** pour assurer la continuité, quel que soit le serveur actif.
