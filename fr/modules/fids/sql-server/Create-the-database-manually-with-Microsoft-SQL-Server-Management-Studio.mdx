---
- title: Create-the-database-manually-with-Microsoft-SQL-Server-Management-Studio
---

Cette section fournit des instructions étape par étape pour créer manuellement la **PADS4FIDS** base de données à l'aide de **SQL Server Management Studio (SSMS)**.

***

## **Étape 1 : Ouvrir SQL Server Management Studio**

1. Lancer **SQL Server Management Studio**.
2. Se connecter au **Database Engine**.
3. Dans l' **Object Explorer**, développez votre instance SQL Server.
4. Faites un clic droit sur **Databases**.
5. Sélectionnez **New Database...**

***

## **Étape 2 : Créer la base de données**

1. Dans le **Database Name** champ, entrez :
   ```text
   PADS4FIDS
   ```
2. Cliquez sur **OK** pour créer la base de données.

***

## **Étape 3 : Vérifier la connexion NT AUTHORITY\SYSTEM**

1. Développez le nœud **Security**.
2. Développez **Logins**.
3. Recherchez la connexion :
   ```text
   NT AUTHORITY\SYSTEM
   ```

* Si elle **existe déjà**, faites un clic droit dessus et choisissez **Properties**, puis passez à l' **Étape 6**.
* Si elle **n'existe pas**, continuez avec l' **Étape 4**.

***

## **Étape 4 : Créer la connexion NT AUTHORITY\SYSTEM**

1. Faites un clic droit sur **Logins**.
2. Sélectionnez **New Login...**

***

## **Étape 5 : Configurer la nouvelle connexion**

1. Dans le **Login name** champ, entrez :
   ```text
   NT AUTHORITY\SYSTEM
   ```
2. Sélectionnez **Windows Authentication**.
3. Ne **pas** entrer de mot de passe.

***

## **Étape 6 : Attribuer les autorisations de base de données**

1. Dans la fenêtre **Login Properties**, sélectionnez **User Mapping** dans le menu de gauche.
2. Dans la section **Users mapped to this login**, cochez la case à côté de :
   ```text
   PADS4FIDS
   ```
3. Dans **Database role membership for: PADS4FIDS**, cochez le rôle :
   ```text
   db_owner
   ```
4. Cliquez sur **OK** pour enregistrer les modifications.

***

La base de données PADS4FIDS est maintenant configurée et prête à l'emploi.
