Ce document décrit les exigences matérielles pour exécuter PADS4 efficacement. Des directives de performance supplémentaires sont incluses pour optimiser votre configuration. Les informations sont fournies « telles quelles » sans garantie d'exhaustivité, d'exactitude, d'utilité ou d'actualité.

Les besoins matériels varient en fonction de l'utilisation du système — le contenu statique nécessite moins de ressources que les déploiements multimédias complets.

## Exigences du PADS4 Platform Server

Les exigences évoluent en fonction du nombre de Viewers. Utilisez des Backup et Sub-Servers pour les systèmes de plus de 1000 Viewers ; contactez [support@pads4.com](mailto:support@pads4.com) pour obtenir des conseils.

| Nombre de Viewers | CPU      | Mémoire | Stockage de l'application | Stockage du contenu | OS                                      | Quantité max. de Data Servers |
| ----------------- | -------- | ------- | ------------------------- | ------------------- | --------------------------------------- | ----------------------------- |
| 1 - 100           | 4 cores  | 16Gb    | 2Gb                       | 16Gb                | Windows Server 2019, 64bit et supérieur | 25                            |
| 100 - 500         | 8 cores  | 16Gb    | 2Gb                       | 64Gb                | Windows Server 2019, 64bit et supérieur | 25                            |
| 500               | 8 cores  | 24Gb    | 2Gb                       | 64Gb                | Windows Server 2019, 64bit et supérieur | 100                           |
| 1000\*            | 16 cores | 32Gb    | 2Gb                       | 256Gb               | Windows Server 2019, 64bit et supérieur | 200                           |

\*Pour >1000 Viewers, il est fortement recommandé d'utiliser des Backup/Sub-Servers.

## Conseils généraux de performance

Suivez ces directives pour des performances optimales de PADS4 lors de la configuration et de l'exploitation.

| Composant                | Description                                                                                                                                                                                                                   |
| ------------------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Disque                   | Utilisez des SSD pour accélérer la synchronisation du contenu, qui a un impact important sur les performances du serveur.                                                                                                     |
| Antivirus                | Excluez les dossiers PADS4 de l'analyse pour éviter les ralentissements lors de la synchronisation du contenu et des processus.                                                                                               |
| Serveur virtuel          | Attribuez une adresse MAC statique ; les adresses dynamiques perturbent la gestion des licences lors des redémarrages.                                                                                                        |
| Réseau                   | Assurez une connexion stable à 100 Mbps+ (1 Gbps recommandé) pour de meilleurs résultats.                                                                                                                                     |
| Stockage du contenu      | La taille dépend des besoins ; les SSD améliorent la vitesse du serveur et des Viewers. Redimensionnez les médias 4K aux spécifications Full HD pour les écrans HD afin de réduire la bande passante et la charge de lecture. |
| Autorisations de contenu | Limitez la synchronisation aux groupes de Viewers/Utilisateurs concernés pour réduire les transferts inutiles.                                                                                                                |
| Data Server              | Récupérez uniquement les données nécessaires (limitez les lignes/colonnes) ; évitez les intervalles de lecture trop courts sauf si les données changent en temps réel, afin d'économiser le CPU et la bande passante.         |
