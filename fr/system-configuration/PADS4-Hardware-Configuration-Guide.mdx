Les besoins en matériel varient selon l'utilisation du système — le contenu statique nécessite moins de ressources que les déploiements multimédias complets.

## Configuration requise pour le serveur de la plateforme PADS4

Les exigences évoluent en fonction du nombre de Viewers. Utilisez des serveurs de sauvegarde et des sous-serveurs pour les systèmes de plus de 1000 Viewers ; contactez [support@pads4.com](mailto:support@pads4.com) pour obtenir des conseils.

| Nombre de Viewers | CPU      | Mémoire | Stockage de l'application | Stockage du contenu | OS                                        | Quantité max. de serveurs de données |
| ----------------- | -------- | ------- | ------------------------- | ------------------- | ----------------------------------------- | ------------------------------------ |
| 1 - 100           | 4 cœurs  | 16Gb    | 2Gb                       | 16Gb                | Windows Server 2019, 64 bits et supérieur | 25                                   |
| 100 - 500         | 8 cœurs  | 16Gb    | 2Gb                       | 64Gb                | Windows Server 2019, 64 bits et supérieur | 25                                   |
| 500               | 8 cœurs  | 24Gb    | 2Gb                       | 64Gb                | Windows Server 2019, 64 bits et supérieur | 100                                  |
| 1000\*            | 16 cœurs | 32Gb    | 2Gb                       | 256Gb               | Windows Server 2019, 64 bits et supérieur | 200                                  |

\*Pour >1000 Viewers, il est fortement recommandé d'utiliser des serveurs de sauvegarde/sous-serveurs.

## Conseils généraux de performance

Suivez ces directives pour des performances optimales de PADS4 lors de la configuration et de l'exploitation.

| Composant                | Description                                                                                                                                                                                                           |
| ------------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Disque                   | Utilisez des SSD pour accélérer la synchronisation du contenu, qui a un impact important sur les performances du serveur.                                                                                             |
| Antivirus                | Excluez les dossiers PADS4 de l'analyse pour éviter les ralentissements lors de la synchronisation du contenu et des processus.                                                                                       |
| Serveur virtuel          | Attribuez une adresse MAC statique ; les adresses dynamiques perturbent la gestion des licences lors des redémarrages.                                                                                                |
| Réseau                   | Assurez une connexion stable à 100 Mbps+ (1 Gbps de préférence) pour de meilleurs résultats.                                                                                                                          |
| Stockage du contenu      | La taille dépend des besoins ; les SSD améliorent la vitesse du serveur et des Viewers. Redimensionnez les médias 4K en Full HD pour les écrans HD afin de réduire la bande passante et la charge de lecture.         |
| Autorisations de contenu | Limitez la synchronisation aux groupes de Viewers/utilisateurs concernés pour réduire les transferts inutiles.                                                                                                        |
| Serveur de données       | Récupérez uniquement les données nécessaires (limitez les lignes/colonnes) ; évitez les intervalles de lecture trop courts sauf si les données changent en temps réel, afin d'économiser le CPU et la bande passante. |
