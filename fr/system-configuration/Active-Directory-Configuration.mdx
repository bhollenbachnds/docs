De nombreuses organisations s'appuient sur **Microsoft Active Directory** pour gérer de manière centralisée les comptes utilisateurs des employés. En intégrant **Active Directory (AD)** avec **PADS4**, vous pouvez utiliser facilement cette structure existante pour l'authentification et la gestion des utilisateurs.

<Note>PADS4 fonctionne avec votre structure Active Directory existante **sans nécessiter de modifications** aux unités d'organisation (UO) ou aux groupes d'utilisateurs.</Note>

***

### **Comment PADS4 fonctionne avec Active Directory**

Une fois qu'une connexion à **Active Directory** est établie, vous pouvez créer des **utilisateurs LDAP** dans PADS4. Ces utilisateurs sont directement liés aux **comptes Active Directory**, assurant un processus d'authentification synchronisé et sécurisé.

* Lorsqu'un **utilisateur LDAP** se connecte, **PADS4 l'authentifie automatiquement** en utilisant les identifiants stockés dans **Active Directory**.
* Toute modification des **noms d'utilisateur ou mots de passe** dans Active Directory est **instantanément reflétée** dans PADS4.
* Si un **compte utilisateur est désactivé** dans Active Directory, il **ne pourra plus se connecter** à PADS4.
* Lorsqu'un utilisateur est **supprimé d'Active Directory**, il est **automatiquement supprimé** de PADS4.
* Si le **PADS4 Server** a des permissions limitées pour accéder à Active Directory, les mises à jour depuis AD **peuvent ne pas prendre effet immédiatement** et seront appliquées **après un redémarrage du serveur**.

> **Important:** La suppression d'un **utilisateur LDAP** de PADS4 **ne supprime pas** son compte de **Active Directory**.

***

### **Gestion des utilisateurs et des permissions**

Toutes les **appartenances aux groupes d'utilisateurs, permissions et attributions de licences** sont gérées dans **PADS4**, y compris pour les utilisateurs LDAP.

Si la **connexion Active Directory** est **désactivée**, les **utilisateurs LDAP créés précédemment restent actifs** dans PADS4. Cela vous permet de **les réutiliser** si vous réactivez la connexion AD à l'avenir.

***

## **Configuration d'Active Directory dans PADS4**

Pour intégrer **Active Directory** avec **PADS4**, suivez ces étapes :

1️. **Configurer les paramètres de connexion Active Directory**

* Ouvrez la fenêtre **Server Settings** dans **PADS4 System Management**.
* Saisissez les **détails de connexion Active Directory** requis.

2️. **Redémarrer le PADS4 Server**

* Cela garantit que les **paramètres de connexion Active Directory prennent effet**.

3️. **Créer des utilisateurs LDAP**

* Accédez à la fenêtre **Users** dans **PADS4 System Management**.
* Créez **un ou plusieurs utilisateurs LDAP** liés aux **comptes Active Directory**.

4️. **Attribuer des groupes d'utilisateurs et des licences**

* Assurez-vous que les utilisateurs sont **assignés aux groupes d'utilisateurs appropriés**.
* Attribuez les **licences** nécessaires à chaque utilisateur.

***

## **Étapes finales**

**Votre intégration Active Directory est maintenant terminée !**\
Les utilisateurs peuvent **se connecter en utilisant leurs identifiants Active Directory**.\
**L'authentification et les mises à jour des utilisateurs** sont **automatisées** et **transparentes**.

En utilisant **Active Directory avec PADS4**, les organisations peuvent maintenir une **gestion centralisée des utilisateurs**, améliorer la **sécurité** et **simplifier l'authentification** sans modifier leur **structure Active Directory** existante.
