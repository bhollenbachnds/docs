Cette section décrit toutes les API disponibles dans un déploiement PADS4 Foundation. Chaque API possède sa propre documentation Swagger/OpenAPI accessible via le PADS4 CTRL Center.

> **Note:** L'adresse du serveur dans toutes les URL ci-dessous utilise `https://saas.pads365.com` à titre d'exemple. Remplacez-la par l'adresse de votre propre serveur PADS4 lors de l'accès aux API.

***

## Comment accéder aux API

Accédez à `https://<your-server>/about` pour voir tous les services en cours d'exécution. Les services dotés d'une icône de lancement renvoient directement vers leur interface Swagger :

```text
https://<your-server>/rdx/<ServiceName>/swagger/index.html
```

La spécification JSON OpenAPI sous-jacente est disponible à l'adresse :

```text
https://<your-server>/rdx/<ServiceName>/swagger/v1/swagger.json
```

## Descriptions détaillées des services

### NDS.Messaging.Broker

Un courtier de messages interne qui facilite la communication asynchrone entre les microservices PADS4. Ce service gère l'infrastructure sous-jacente du bus de messages et n'est généralement pas appelé directement par des clients externes.

***

### NDS.Nginx.Domains

Gère la configuration des domaines pour le proxy inverse Nginx de PADS4. Utilisé pour ajouter, supprimer et valider le routage de domaine pour les déploiements multi-locataires.

**Points de terminaison clés :**

| Méthode | Chemin                    | Description                               |
| :------ | :------------------------ | :---------------------------------------- |
| GET     | `/api/v1/Domain`          | Lister tous les domaines configurés       |
| POST    | `/api/v1/Domain`          | Ajouter un nouveau domaine                |
| DELETE  | `/api/v1/Domain`          | Supprimer un domaine                      |
| GET     | `/api/v1/Domain/validate` | Valider une configuration de domaine      |
| POST    | `/api/v1/Domain/reset`    | Réinitialiser la configuration du domaine |
| POST    | `/api/v1/Domain/restart`  | Redémarrer le service de domaine          |

***

### NDS.Services.Authentication

Le service principal de gestion des identités et des accès. Gère les utilisateurs, les rôles, les groupes, les revendications et les intégrations de fournisseurs d'identité externes (LDAP, SAML, OAuth, SCIM).

**Contrôleurs / domaines fonctionnels :**

* **Account** — Gestion des comptes locaux (connexion, réinitialisation du mot de passe)
* **User** — Créer, lire, mettre à jour, supprimer des utilisateurs
* **Role** — Gestion et attribution des rôles
* **Membership** — Opérations d'appartenance aux groupes
* **Claim** — Mappage d'attributs/revendications pour les jetons d'identité
* **LdapProvider** — Configurer et interroger les connexions LDAP/Active Directory
* **ExternalAuthenticationProvider** — Enregistrer des fournisseurs d'identité externes (OAuth, SAML, etc.)
* **ExternalAuthentication** — Initier des flux d'authentification externes
* **ExternalUserGroupMapping** / **ExternalUserGroup** — Mapper des groupes externes aux rôles PADS4
* **AuthenticationIdentityProvider** — Configuration du fournisseur d'identité
* **Scim** — Points de terminaison de provisionnement SCIM (utilisateurs et groupes)
* **ReplicationClient** — Diagnostics de réplication interne

**Exemples de points de terminaison :**

| Méthode  | Chemin                                 |
| :------- | :------------------------------------- |
| GET/POST | `/api/v1/User`                         |
| GET/POST | `/api/v1/Role`                         |
| POST     | `/api/v1/ExternalAuthentication/Logon` |
| GET/POST | `/api/v1/LdapProvider`                 |
| GET/POST | `/api/v1/Claim/GetAll`                 |

### NDS.Services.Schedule

Gère les planifications et les règles de planification pour le contenu et les événements PADS4. Prend en charge les planifications récurrentes (via les RRULE iCalendar), les exceptions et les requêtes d'occurrences.

* **Schedule** — Créer, lire, mettre à jour, supprimer des schedules ; générer des descriptions lisibles par l'homme et des RRULEs
