Ce guide fournit des instructions étape par étape pour l'installation et la configuration du **PADS4 Viewer** sur les **appareils ChromeOS**.

***

## **1. Préparation de l'appareil**

Pendant la configuration, vous devrez configurer l'emplacement du **serveur PADS4**. Il est recommandé de **terminer l'installation du serveur PADS4 d'abord** avant de configurer votre **appareil ChromeOS**.

<Note>La version minimale de **PADS4 Foundation** requise pour ChromeOS est **2024.1**.</Note>

### **1.1 Prérequis**

Avant d'utiliser **PADS4 sur ChromeOS**, assurez-vous que les conditions suivantes sont remplies :

1. Vous êtes familier avec **ChromeOS** et la gestion des appareils dans la **Google Admin Console**.
2. La **plateforme de gestion Google** est basée sur le cloud, nécessitant un **accès internet** pour les viewers.
3. La **version ChromeOS** est **120 ou plus récente**.
4. L'appareil est **connecté à un réseau** avec **accès internet**.
5. Vous avez créé **une ou plusieurs unités organisationnelles (UO)** pour les **PADS4 Viewers**.
6. L'appareil est **provisionné dans la Google Admin Console**.
7. Vous avez acheté une licence **Chrome Enterprise** ou **Kiosk & Signage Upgrade**.
8. L'appareil s'est vu **attribuer une licence Chrome Enterprise ou Kiosk & Signage Upgrade**.

***

## **2. Ajout de PADS4 comme application**

Pour installer **PADS4 sur ChromeOS**, suivez ces étapes pour **ajouter PADS4 comme application** dans la **Google Admin Console** :

<Steps>
  <Step title="Étape Une">
    Ouvrez la **Google Admin Console** et connectez-vous.
  </Step>

  <Step title="Étape Deux">
    Naviguez vers : **Devices → Chrome → Apps & Extensions**.
  </Step>

  <Step title="Étape Trois">
    Allez à **Kiosks**.
  </Step>

  <Step title="Étape Quatre">
    Sélectionnez le **appropriate Organizational Unit**.
  </Step>

  <Step title="Étape Cinq">
    Cliquez sur le **signe plus (+)** pour ajouter une application.
  </Step>

  <Step title="Étape Six">
    Sélectionnez **"Add by URL"**.
  </Step>

  <Step title="Étape Sept">
    Spécifiez l'URL suivante :

    ```
    http://[PADS4server]/PADS4/apps/Viewer/  
    ```

    <Note>Si votre **PADS4 Server** fonctionne sous un **domain**, remplacez `"PADS4"` par votre **domain name**.</Note>
  </Step>

  <Step title="Étape Huit">
    Cliquez sur **Save**.
  </Step>

  <Step title="Étape Neuf">
    Sous l'**Organizational Unit**, sélectionnez **PADS4** comme **Auto-launch App**.
  </Step>

  <Step title="Étape Dix">
    **PADS4 Viewer** démarrera maintenant automatiquement comme **default application** pour les appareils dans cette **Organizational Unit**.
  </Step>
</Steps>

***

## **3. Configuration des paramètres d'appareil ChromeOS pour PADS4**

### **3.1 Ajustement des paramètres d'inactivité**

Pour empêcher **ChromeOS** de diminuer la luminosité ou d'éteindre l'écran, suivez ces étapes :

<Steps>
  <Step title="Étape Un">
    Ouvrez la **Google Admin Console** et connectez-vous.
  </Step>

  <Step title="Étape Deux">
    Naviguez vers : **Devices → Chrome → Settings**.
  </Step>

  <Step title="Étape Trois">
    Sélectionnez l'**Organizational Unit** pour les **PADS4 Devices**.
  </Step>

  <Step title="Étape Quatre">
    Allez à **Device Settings**.
  </Step>

  <Step title="Étape Cinq">
    Faites défiler jusqu'à **AC Kiosk Power Settings**.
  </Step>

  <Step title="Étape Six">
    Entrez `0` pour **Screen Dim Timeout (minutes)**.
  </Step>

  <Step title="Étape Sept">
    Entrez `0` pour **Screen Off Timeout (minutes)**.
  </Step>

  <Step title="Étape Huit">
    Cliquez sur **Save**.
  </Step>
</Steps>

***

### **3.2 Déplacement de l'appareil vers l'Organizational Unit**

Pour garantir que **PADS4 Viewer** démarre automatiquement au démarrage de l'appareil :

<Steps>
  <Step title="Étape Un">
    Ouvrez la **Google Admin Console** et connectez-vous.
  </Step>

  <Step title="Étape Deux">
    Naviguez vers : **Devices → Chrome → Devices**.
  </Step>

  <Step title="Étape Trois">
    Sélectionnez **all devices** qui doivent exécuter **PADS4 Viewer**.
  </Step>

  <Step title="Étape Quatre">
    Cliquez sur **Move**.
  </Step>

  <Step title="Étape Cinq">
    Sélectionnez l'**Organizational Unit** avec **PADS4 set as the Auto-launch App**.
  </Step>

  <Step title="Étape Six">
    **Redémarrez les appareils**.
  </Step>

  <Step title="Étape Sept">
    Les appareils redémarreront, et **PADS4 Viewer** se lancera automatiquement.
  </Step>
</Steps>

***

### **3.3 Renommer le PADS4 Viewer**

Une fois connecté au **PADS4 Server**, le serveur attribue un **temporary viewer name** tel que :

```
Chrome0001  
```

Pour renommer le viewer, utilisez **PADS4 CTRL Center**.

***

## **4. Mise à jour de PADS4 Viewer**

Après l'installation initiale, **PADS4 Viewer** se **mettra à jour automatiquement** lorsque le **PADS4 Server** est mis à jour.

***

## **5. Capacités des appareils et fonctionnalités prises en charge**

### **Fonctionnalités générales**

| Fonctionnalité | Description                                                                                                                |
| :------------: | -------------------------------------------------------------------------------------------------------------------------- |
|        ✅       | **Automatic Updates** – PADS4 Viewer se met à jour lorsque le **PADS4 Server** est mis à jour.                             |
|        ✅       | **Offline Mode** – L'appareil continue de diffuser les **scheduled presentations** pendant **7 days** s'il est hors ligne. |
|        ⚠       | **ChromeOS nécessite une connexion réseau pour démarrer PADS4 Viewer.**                                                    |

### **Capacités des appareils**

| Fonctionnalité | Description                                                                          |
| :------------: | ------------------------------------------------------------------------------------ |
|        ✅       | **Portrait Mode** – L'appareil pivote automatiquement.                               |
|        ✅       | **4K UHD Support**                                                                   |
|        ✅       | **Transfer Protocols** – HTTP, HTTPS                                                 |
|        ✅       | **Turn Off Display** – L'écran s'éteint lorsqu'aucune présentation n'est programmée. |
|        ✅       | **Cluster Viewer** – Pris en charge                                                  |
|        ✅       | **Volume Level Control** – Réglez le volume maximum au niveau de l'appareil.         |

### **Éléments pris en charge**

| Fonctionnalité | Description                                                                                |
| :------------: | ------------------------------------------------------------------------------------------ |
|        ✅       | **Video Element** – Jusqu'à **4 Full HD videos** par page.                                 |
|        ✅       | **Video Rotation, Seamless Videos, Reflection, Skew, Flip** – Pris en charge.              |
|        ❌       | **Streaming Video** – Non pris en charge.                                                  |
|        ✅       | **Images (JPG, PNG, BMP, GIF), Text, Tables, Shapes, Scrolling Text, Interactive Buttons** |
|        ✅       | **PowerPoint Element** – Requires **Microsoft PowerPoint installed on the PADS4 Server**.  |
|        ✅       | **PDF Element** – Requires **Xpert License**.                                              |

| Fonctionnalité | Description                                                                                                  |
| :------------: | ------------------------------------------------------------------------------------------------------------ |
|        ✅       | **Gestion à distance des appareils via PADS4 CTRL Center**                                                   |
|        ✅       | **Surveillance de l'état de connexion et alertes**                                                           |
|        ❌       | **Redémarrage, Arrêt, Wake-on-LAN** – Non pris en charge.                                                    |
|        ✅       | **Redémarrage du Viewer, Récupération des journaux, Installation des polices, Suivi de preuve de diffusion** |

***

## **Notes importantes**

<Warning> **Intégrité des fichiers de contenu** – La **PADS4 Content Library** vérifie l'intégrité des fichiers en utilisant **MD5 checksum** pendant la synchronisation.</Warning>

<Warning> **Limitations de la preuve de diffusion** – Un maximum de **50,000 records** peuvent être stockés localement. Les enregistrements plus anciens seront supprimés une fois la limite atteinte.</Warning>
