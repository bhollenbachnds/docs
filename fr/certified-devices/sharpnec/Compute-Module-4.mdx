# PADS4 Viewer sur Sharp/NEC CM4 & CM5

Cet article décrit comment installer et configurer le PADS4 Viewer sur les appareils Sharp/NEC Compute Module 4 (CM4) et Compute Module 5 (CM5).

## Installation de l'image Sharp/NEC Compute Module

Sharp/NEC fournit une image Raspberry Pi personnalisée qui est nécessaire pour exécuter des applications web sur le Compute Module.

Si votre appareil n'exécute pas l'image Sharp/NEC, installez-la en suivant la section **Flashage manuel du Raspberry Pi via USB** du *Guide de référence rapide pour Sharp MPi4 Kit 4.1*.

Pendant l'installation :

1. Sélectionnez **Custom Image**.
2. Sélectionnez l'image Sharp/NEC actuelle.

> **Note**
>
> Au moment de la rédaction, l'image actuelle est :
>
> `Sharp_4.1_EU_20250501.img`
>
> Cette version d'image peut changer dans les futures versions.

Après l'installation, configurez un nom d'hôte unique pour l'appareil.

## Autoriser l'accès à l'API Sharp/NEC

Par défaut, l'image Sharp/NEC n'autorise que les applications s'exécutant sur localhost à accéder à l'API.

Connectez-vous au Compute Module via SSH.

**Identifiants par défaut**

| Paramètre         | Valeur             |
| :---------------- | :----------------- |
| Nom d'utilisateur | `user`             |
| Mot de passe      | `CM@SHARPdisplays` |

Ouvrez le fichier de configuration :

```bash
cd /sharp/controls
sudo nano web-comms.py
```

Localisez la configuration CORS :

```python
CORS(
    app,
    resources={r"/*": {"origins": "http://127.0.0.1"}},
    methods=["GET", "POST"]
)
```

Remplacez `http://127.0.0.1` par l'URL de votre environnement PADS4.

Par exemple :

```python
CORS(
    app,
    resources={r"/*": {"origins": "https://pads4.contoso.com"}},
    methods=["GET", "POST"]
)
```

Pour autoriser toutes les origines, utilisez :

```python
resources={r"/*": {"origins": "*"}}
```

Enregistrez le fichier et redémarrez l'appareil.

## Configuration du Viewer

L'image Sharp/NEC contient un point de terminaison de configuration utilisé pour configurer le comportement de démarrage du navigateur.

Open:

```text
http://<ip-address-or-hostname>/config
```

Téléchargez le fichier de configuration.

**Identifiants par défaut**

| Paramètre         | Valeur             |
| :---------------- | :----------------- |
| Nom d'utilisateur | `user`             |
| Mot de passe      | `CM@SHARPdisplays` |

Ouvrez le fichier de configuration dans un éditeur de texte et configurez les paramètres.

### host

Spécifie le serveur PADS4.

Valeurs possibles :

* `127.0.0.1`
* Adresse IP du serveur PADS4
* Nom d'hôte du serveur PADS4

Lorsqu'une adresse IP ou un nom d'hôte est spécifié, le Viewer attend que le serveur soit accessible.

Lorsque `127.0.0.1` est spécifié, le Viewer démarre même lorsque l'appareil est hors ligne.

> **Note**
>
> L'utilisation de `127.0.0.1` nécessite que l'application Viewer soit hébergée via HTTPS.

### auto\_close\_browser

Valeur recommandée :

```text
false
```

### wait\_for\_time\_server

Lorsqu'il est activé, le Viewer démarre uniquement après la fin de la synchronisation de l'heure.

### url

URL de l'application PADS4 Viewer.

Example:

```text
https://<server>/pads4/apps/viewer/
```

### incognito\_mode

Valeur recommandée :

```text
false
```

### autorefresh

Valeur recommandée :

```text
false
```

### ntp\_server

Spécifie le serveur NTP.

Le PADS4 Server peut également être utilisé comme serveur NTP.

Téléchargez le fichier de configuration modifié et redémarrez l'appareil.

## Mise à jour du Viewer

Après l'installation initiale, les mises à jour du Viewer sont installées automatiquement.

Lorsque le PADS4 Server est mis à niveau, le Viewer se met automatiquement à jour vers la version disponible sur le serveur.
