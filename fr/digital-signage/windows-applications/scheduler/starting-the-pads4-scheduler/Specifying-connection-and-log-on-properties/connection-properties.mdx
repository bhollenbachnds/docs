The **Connection Properties** section allows you to configure and manage connections between PADS4 Scheduler and your PADS4 Server. This includes connection methods, login credentials, synchronization settings, and HTTP options.

***

## Opening the PADS4 Server Connections Window

To manage server connections:

1. Navigate to the **Home** tab in the ribbon bar.
2. In the **Connections** section, click the **Manage** button.\
   This opens the **PADS4 Server Connections** window.

***

## Managing Connections

From the PADS4 Server Connections window, you can:

* **Add...** a new PADS4 Server connection
* **Edit...** an existing connection
* **Remove...** a connection no longer needed

***

## Adding or Editing a Server Connection

Whether you're creating a new connection or modifying an existing one, the configuration window is the same.

### Connection Tab Settings

#### **Connection Name**

A unique name used to identify this server connection in the Scheduler interface.

#### **Application Startup Option**

Choose whether to **automatically connect** to this server when PADS4 Scheduler starts.

#### **Server Connection Types**

| Option            | Description                                                                                                                          |
| ----------------- | ------------------------------------------------------------------------------------------------------------------------------------ |
| **Automatic**     | Automatically detects the server. Recommended only when a single PADS4 Server exists in your LAN. Requires UDP port 3702 to be open. |
| **This Computer** | Assumes PADS4 Server runs on the same machine as the Scheduler. Preferred for local setups.                                          |
| **Other**         | Manually specify the serverâ€™s hostname, IP address, or DNS entry. Optionally, click **Search...** to detect servers on the network.  |

> **Advanced Options**: You can define a custom port number if the server is using non-default ports. The port number must match the configuration in **PADS4 System Management**.

***

### HTTP Settings

Used for web-based communication like audit logging or synchronization.

* **Automatic**: Default settings based on the server configuration.
* **Manual**: Specify a custom **Host Path** and **Port**, useful when using routers, proxies, or port forwarding.

***

### Credentials Tab

Specify the login information for your server connection.

* **Username**: The username for server authentication.
* **Password**: (Optional) Stored password. If left blank, the user will be prompted.
* **Automatic Login**: Automatically attempts to log in on connection. If it fails, the user is prompted for credentials.

> **Tip:** Combine **Automatic Login** with **Automatically start this connection** to log in seamlessly when launching the application.

***

### Synchronization Tab

Control how presentation content is synchronized between the Scheduler and the Server.

* **Synchronization Activation**:
  * **Automatic**: Synchronizes on change
  * **Manual**: Synchronizes only when initiated by the user
  * **Never**: Disables synchronization

> If the server and Scheduler are on the same machine, synchronization may not be necessary.

* **PADS4 Content Library Storage Folder**:\
  Define a local folder for content storage if the default location is unsuitable.

* **Domain**:\
  Use this field if the server hosts multiple PADS4 domains, or if you're managing multiple server connections and need to keep them separate. Otherwise, choose **Automatic**.

***

### Saving the Configuration

Click **OK** to save your connection settings and return to the main application window.
