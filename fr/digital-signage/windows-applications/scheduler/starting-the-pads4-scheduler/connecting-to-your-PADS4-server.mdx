Quand **PADS4 Scheduler** démarre, il tente d'établir une connexion à un **PADS4 Server**. Le comportement de l'application au démarrage dépend de la façon dont les **connection properties** sont définies.

***

## Connexion pour la première fois

Si c'est la **first time** que vous démarrez PADS4 Scheduler et qu'aucune propriété de connexion n'a été configurée :

* Vous recevrez un **notification message** vous invitant à configurer la connexion.
* Cliquez sur **OK** pour ouvrir la fenêtre **PADS4 Server Connections**.
* Dans cette fenêtre, vous pouvez spécifier les paramètres requis pour vous connecter à votre PADS4 Server, y compris l'adresse du serveur, le port et les identifiants.

Une fois complété et enregistré, le Scheduler tentera d'établir une connexion.

***

## Connexion manuelle

Si vous avez choisi de **manually manage server connections**, la fenêtre principale restera vide au démarrage.

Pour vous connecter manuellement :

### Option 1 : Utiliser le bouton de connexion réseau

1. Localisez le **network connection button** dans la barre de ruban.
2. Cliquez dessus pour initier une connexion au serveur par défaut ou précédemment utilisé.

### Option 2 : Utiliser le bouton fractionné de connexion au serveur

1. Cliquez sur le **PADS4 Server Connection Split Button** dans le ruban.
2. Une liste déroulante des connexions de serveur disponibles apparaîtra.
3. Passez la souris sur le serveur auquel vous souhaitez vous connecter, et cliquez dessus.
4. PADS4 Scheduler tentera de se connecter au serveur sélectionné.

***

## Connexion automatique

Si vous avez configuré la **automatic server connection**, PADS4 Scheduler va :

* Tenter de se connecter au PADS4 Server spécifié **immediately on launch**
* Charger l'interface une fois la connexion réussie

Aucune interaction supplémentaire n'est requise, sauf si la connexion échoue, auquel cas vous serez invité à vérifier ou à saisir à nouveau les paramètres de connexion.
