Comme indiqué dans la *Recurrence* présentation, PADS4 Scheduler vous permet de définir des **planifications répétitives** à l'aide de modèles de récurrence personnalisables. La récurrence peut être appliquée aux planifications **Simple**, **Playlist** et **Advanced** et garantit que le contenu est affiché de manière répétée à des intervalles définis sans avoir besoin de créer des planifications en double.

***

## Accéder aux paramètres de récurrence

Vous pouvez appliquer la récurrence de deux façons :

* Lors de la création d'une planification dans le **Schedule Wizard** en cliquant sur le bouton **Recurrence**.
* Depuis la **Ribbon Bar** en sélectionnant la fonction **Recurrence** (voir la rubrique *Recurrence* pour plus de détails).

Une fois accessible, la fenêtre **Recurrence Settings** s'affichera.

***

## Fenêtre Recurrence Settings

Cette fenêtre est divisée en quatre sections clés :

1. **Period**
2. **Type**
3. **Within One Day**
4. **Description**

Chacune est décrite ci-dessous.

***

### 1. Period : Définir la plage temporelle

Dans cette section, vous définissez les **conditions de début et de fin** du modèle de récurrence.

* **Start Date** : Indique quand la récurrence commence.
* **End Options** :
  * **By End Date** : Se termine à une date précise.
  * **By Number of Occurrences** : Se termine après un nombre défini de répétitions.
  * **No End Date** : Se répète indéfiniment.

***

### 2. Type : Définir le modèle de récurrence

PADS4 Scheduler propose **quatre types de récurrence**, chacun avec des options de configuration flexibles :

#### **Daily**

* Se répète tous les *X* jours
* Optionnellement, limiter aux jours de semaine uniquement

#### **Weekly**

* Sélectionner des jours spécifiques de la semaine (ex. : lun., mer., ven.)
* Choisir l'intervalle de répétition en semaines (ex. : toutes les 2 semaines)

#### **Monthly**

* Deux options de configuration :
  * **By Date** : ex., le 15 de chaque mois
  * **By Pattern** : ex., *Second Tuesday* de tous les 3 mois
    * Choose: `first`, `second`, `third`, `fourth`, `last`
    * Puis sélectionner : `day`, `weekday`, `weekend day`, ou un jour spécifique (ex. : lundi)

#### **Yearly**

* Similaire à Monthly, avec deux options :
  * **By Date** : ex., le 23 mars chaque année
  * **By Pattern** : ex., *Last Friday* d'avril
    * Définir l'ordinal, le jour de la semaine et le mois

***

### 3. Within One Day : Répétition basée sur l'heure

Cette section définit **à quel moment dans une journée** le contenu doit être affiché :

* **From / Until** : Définir la plage horaire quotidienne (ex. : 08:00–18:00)

Vous pouvez également spécifier une granularité plus fine dans cette plage :

* **Uninterrupted** : Afficher en continu dans la plage horaire
* **Several minutes per hour**
* **Several minutes per X minutes**

> Ceci est utile lorsque vous souhaitez que le contenu apparaisse uniquement à intervalles, par exemple 10 minutes toutes les heures.

***

### 4. Description : Consulter le résumé de récurrence

La **Description** se met à jour dynamiquement au fur et à mesure que vous ajustez les paramètres. Elle fournit un **résumé lisible** du modèle de récurrence pour une vérification facile.

Exemple de sortie :\
*"Se produit chaque lundi et mercredi, de 08h00 à 18h00, à partir du 1er mars, se terminant après 10 occurrences."*
