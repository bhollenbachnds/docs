Lorsque vous modifiez un **recurring schedule**, PADS4 Scheduler vous permet de choisir si vous souhaitez :

* Modifier le **série entière**, ou
* Modifier uniquement une **occurrence unique** de cette série

Cette flexibilité vous permet de maintenir la cohérence dans un planning tout en effectuant des ajustements ponctuels occasionnels selon les besoins.

> **Note:**\
> Si des autorisations d'accès sont en place, certaines options de destination et de priorité peuvent être **restreintes ou masquées** en fonction de vos droits d'utilisateur.

***

## Option 1 : Modifier la **Série Entière**

Cette option vous permet de modifier toutes les occurrences du planning en une seule étape.

### Steps:

1. Dans la vue **Calendar** ou **Table**, **faites un clic droit** sur le planning récurrent que vous souhaitez modifier.
2. Passez la souris sur **Edit**, puis sélectionnez **Series** dans le sous-menu.

La fenêtre **Edit Schedule** s'ouvre, affichant les paramètres généraux du planning.

* Vous pouvez mettre à jour les propriétés standard comme :
  * **Destination**
  * **Priority**
  * **Display Period**
  * **Playlist content**

3. Cliquez sur le bouton **Recurrence** pour ouvrir la fenêtre **Recurrence Settings**.

4. Modifiez le modèle de récurrence selon vos besoins.

5. Cliquez sur **OK** pour appliquer les modifications de récurrence.

6. Cliquez à nouveau sur **OK** pour appliquer toutes les modifications du planning.

> Vos mises à jour s'appliqueront désormais à **toutes les instances futures et passées** du planning récurrent.

***

## Option 2 : Modifier une **Occurrence Unique**

Cette option vous permet d'ajuster uniquement **une instance spécifique** d'un planning récurrent.

### Steps:

1. Dans la vue **Calendar** ou **Table**, **faites un clic droit** sur l'occurrence spécifique que vous souhaitez modifier.
2. Passez la souris sur **Edit**, puis sélectionnez **Occurrence** dans le sous-menu.

La fenêtre **Edit Occurrence** s'ouvrira.

* Vous pouvez ajuster la **période d'affichage** et d'autres détails de base pour cette instance.

3. Cliquez sur **OK** pour appliquer la modification uniquement à cette occurrence.

> Le planning affichera maintenant une **Recurrence Exception**. Cela signifie que l'occurrence fait toujours partie de la série mais possède une configuration modifiée.

***

## Important: Managing Exceptions

**Les modifications apportées à la série affecteront toujours les exceptions, sauf si elles sont explicitement supprimées.**

Si vous souhaitez **déconnecter complètement une occurrence** du planning récurrent :

1. **Supprimez** l'occurrence.
2. **Créez manuellement** un nouveau **Simple Schedule** pour la remplacer.

Cela garantit que la nouvelle entrée est complètement **indépendante** de la série récurrente d'origine.
