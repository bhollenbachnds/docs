Lorsque des planifications **se chevauchent dans le temps**, celle ayant la **priorité plus élevée** est affichée. Si plusieurs planifications qui se chevauchent partagent le même niveau de priorité, le système utilise la **sous-priorité** pour déterminer quelle planification prend la priorité.

Pour en savoir plus sur le fonctionnement des priorités, consultez la section **Priorités**.

> **Note:** Si des autorisations d'accès sont configurées dans votre système, certaines fonctions liées aux priorités peuvent être **masquées ou désactivées** selon votre rôle d'utilisateur.

***

## Modification du niveau de priorité principal

Vous pouvez modifier la priorité d'une planification en utilisant soit la **barre de ruban** soit le **menu contextuel**. L'exemple ci-dessous montre comment utiliser le menu contextuel.

### Steps:

1. Dans le **Destinations tree view** (panneau de gauche), sélectionnez le Viewer ou le Viewer Group contenant le planning.

2. Dans la vue **Calendar** ou **Table**, localisez le planning que vous souhaitez ajuster.

3. **Cliquez avec le bouton droit** sur le planning, puis passez la souris sur l'élément de menu **Priority**.

4. Dans le sous-menu, choisissez :
   * **Increase** – pour augmenter le niveau de priorité
   * **Decrease** – pour diminuer le niveau de priorité

Une fois sélectionnée, la **priority of the schedule** est mise à jour immédiatement.

***

## Modification de la sous-priorité

Si deux plannings ou plus ont le **same priority level**, vous pouvez ajuster leur **sub priority** pour contrôler lequel prend la priorité. La sous-priorité est basée sur l'**creation time** du planning mais peut être ajustée manuellement.

### Steps:

1. Dans le **Destinations tree view**, sélectionnez le Viewer ou le Viewer Group.

2. Dans la vue **Calendar** ou **Table**, localisez le planning que vous souhaitez ajuster.

3. **Cliquez avec le bouton droit** sur le planning, passez la souris sur le menu **Priority**, puis allez au sous-menu **Importance**.

4. Choose:
   * **Increase** – pour augmenter la sous-priorité (rend plus probable son affichage)
   * **Decrease** – pour diminuer la sous-priorité

La **sub priority of the schedule** est ajustée en conséquence.

***

Ce mécanisme vous donne un **fine-tuned control** sur quelle présentation est affichée lorsque les plannings se chevauchent, garantissant que le contenu le plus pertinent est priorisé.
