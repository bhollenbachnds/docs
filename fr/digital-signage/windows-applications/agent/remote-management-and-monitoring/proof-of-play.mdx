Grâce aux fonctions de preuve de diffusion, vous pouvez consulter les entrées du journal de lecture stockées sur votre PADS4 Server ou PADS4 Viewer. Sur votre serveur, les informations de preuve de diffusion peuvent être exportées ou supprimées.

Lors de la conception de votre présentation dans PADS4 Designer, vous pouvez spécifier que vous souhaitez collecter des informations de preuve de diffusion concernant votre présentation, page ou élément de page. Pendant la lecture, chaque PADS4 Viewer collecte et met en cache localement les informations de preuve de diffusion. Toutes les heures pleines, ces informations sont transmises au PADS4 Server principal où elles sont fusionnées dans une base de données centrale de preuve de diffusion (Microsoft SQL Compact Edition). Si l'application PADS4 Viewer est arrêtée normalement, toutes les informations de preuve de diffusion qui n'ont pas encore été transmises seront envoyées au serveur.

 

#### **Pour afficher les informations de preuve de diffusion stockées sur votre PADS4 Server**

1. Cliquez sur l'onglet **Server** dans le volet de navigation.

   ![EN Agent Navigationpane Servertab Pn](/images/digital-signage/windows-applications/agent/remote-management-and-monitoring/EN-Agent-Navigationpane-Servertab.png)
2. Cliquez sur **Proof of play** dans la liste des catégories d'informations serveur disponibles.

Après cela, le système récupérera automatiquement les informations demandées depuis le PADS4 Server connecté et affichera les résultats dans la fenêtre de preuve de diffusion. Si votre installation PADS4 comprend des sous-serveurs, vous devez d'abord sélectionner un serveur avant de sélectionner la catégorie d'informations.

 

#### **Pour afficher les informations de preuve de diffusion mises en cache localement sur un PADS4 Viewer**

1. Cliquez sur l'onglet **Viewer** dans le volet de navigation.

   ![EN Agent Navigationpane Viewertab Pn](/images/digital-signage/windows-applications/agent/remote-management-and-monitoring/EN-Agent-Navigationpane-Viewertab.png)
2. Sélectionnez un Viewer dans la liste des destinations disponibles.
3. Cliquez sur **Proof of play** dans la liste des catégories d'informations viewer disponibles.

Après cela, le système se connectera automatiquement au PADS4 Server connecté pour récupérer les informations demandées depuis le Viewer sélectionné et afficher les résultats dans la fenêtre de preuve de diffusion.

<Warning>
  L'application PADS4 ne pourra commencer à récupérer les informations du viewer demandées que lorsque vous aurez sélectionné <u>à la fois</u> un Viewer et une Catégorie dans le volet de navigation.
</Warning>

 

#### **Fenêtre de preuve de diffusion**

La fenêtre de preuve de diffusion affiche les entrées du journal de lecture qui ont été stockées sur votre PADS4 Server ou PADS4 Viewer. Cela vous permet de vérifier si, quand, où et pendant combien de temps un contenu spécifique a été diffusé.

![EN Agent Proof Of Play Pn](/images/digital-signage/windows-applications/agent/remote-management-and-monitoring/EN-Agent-ProofOfPlay.png)

Dans la fenêtre de preuve de diffusion de votre PADS4 Server, vous pouvez consulter les informations de preuve de diffusion sur une base quotidienne. Sélectionnez la **Date** pour laquelle vous souhaitez afficher les données. En raison du volume potentiellement important de données, cela est limité à une base quotidienne. Les options d'**exportation** permettent à l'utilisateur de spécifier une période. Il est également possible d'utiliser l'Agent SDK ou une Web API pour lire les informations de preuve de diffusion depuis le système PADS4. Consultez la documentation du SDK.

Les entrées de preuve de diffusion transmises par les PADS4 Viewers connectés sont stockées dans la base de données de preuve de diffusion sur votre PADS4 Server. Dans la fenêtre de preuve de diffusion d'un PADS4 Viewer (onglet Viewer), vous ne voyez que les entrées qui ont été mises en cache localement sur votre Viewer et qui n'ont pas encore été transmises à votre PADS4 Server. Toutes les heures pleines ou lors de la fermeture du PADS4 Viewer, l'application transmet ses informations de preuve de diffusion à votre PADS4 Server.

<Tip>
  La fenêtre de preuve de diffusion d'un PADS4 Viewer est destinée à suivre le processus de journalisation en cours d'un PADS4 Viewer, tandis que la fenêtre de preuve de diffusion de votre PADS4 Server est destinée à vérifier les informations de preuve de diffusion d'une période clôturée a posteriori.
</Tip>

**Heure de début**\
Date et heure auxquelles la lecture du contenu a commencé. L'heure affichée utilise le fuseau horaire local de votre système PADS4.

**Décalage UTC**\
La différence de temps (en minutes) entre l'heure de Greenwich (UTC) et l'heure locale du PADS4 Viewer. Cette information est généralement utilisée lorsque votre installation PADS4 comprend des PADS4 Viewers fonctionnant dans différents fuseaux horaires.

**Nom du Viewer**\
Nom unique du PADS4 Viewer qui a lu le contenu.

**Niveau**\
Niveau de l'élément qui a lu le contenu. Les niveaux possibles sont présentation, page et élément.

**Élément**\
Identification de l'élément qui a lu le contenu.

**Durée**\
Durée pendant laquelle le contenu a été lu. Dans l'aperçu, la durée est arrondie aux secondes, tandis que l'information est stockée avec une précision en millisecondes. Lorsque vous exportez les informations de preuve de lecture, la valeur en millisecondes est incluse.

**Terminé**\
Indique si le contenu a été entièrement lu ou s'il a été interrompu pendant la lecture. Dans ce contexte, entièrement lu signifie que le contenu a été lu pendant toute la durée pour laquelle il était programmé. Certains éléments (comme la vidéo en streaming) n'ont pas de durée programmée et ne prennent donc pas en charge cette option.

**État de l'affichage**\
Affiche l'état du moniteur pendant la lecture du contenu. Cela peut être utilisé comme preuve que le contenu a réellement été visible pour le public. Les informations sur l'état de l'affichage ne peuvent être collectées que lorsque le PADS4 Viewer a été correctement configuré et prend en charge la collecte de ces informations.

**Contenu**\
Contenu qui a été lu par l'élément. Pour les éléments texte, il s'agira du texte qui a été affiché, tandis que pour d'autres éléments comme les images et les vidéos, il s'agira du nom du fichier contenant le contenu.

 

#### **Vues**

Sur votre PADS4 Server, les informations de preuve de lecture sont stockées dans une base de données de preuve de lecture. Le système crée automatiquement une nouvelle base de données de preuve de lecture au début d'un nouveau mois. Lors de la consultation des informations de preuve de lecture de votre PADS4 Server, vous devez sélectionner la**Date de preuve de lecture** (et donc implicitement la base de données de preuve de lecture) pour laquelle vous souhaitez voir les informations. Si une certaine période n'est pas répertoriée, cela signifie qu'il n'y a pas d'informations de preuve de lecture disponibles pour cette période.

 

#### **Exportation des informations de preuve de lecture**

Pour traiter les informations de preuve de lecture dans une application externe (comme Microsoft Excel), les informations peuvent être exportées dans différents formats. Cela inclut le format XML (suivant les définitions officielles de formatage de preuve de lecture de l'organisation POPAI) et le format Microsoft Excel.

Pour exporter les informations de preuve de lecture :

1. Cliquez sur **Exporter** dans le **Accueil** menu.
2. Saisissez la **Début** et la **Fin** de la période pour laquelle vous souhaitez exporter les informations de preuve de lecture.

   ![EN Agent Proof Of Play Export Pn](/images/digital-signage/windows-applications/agent/remote-management-and-monitoring/EN-Agent-ProofOfPlayExport.png)

   <Tip>
     Lors de la spécification de la période d'exportation ici, vous n'êtes pas limité aux périodes mensuelles de vos bases de données physiques de preuve de lecture. Si nécessaire, le système PADS4 collectera automatiquement les informations de preuve de lecture à partir de plusieurs bases de données de preuve de lecture lors de l'exécution de la fonction d'exportation.
   </Tip>
3. Sélectionnez le type de sortie, **XML** ou **Excel**.
4. Cliquez sur **Parcourir** pour spécifier le **Nom du fichier** du fichier dans lequel vous souhaitez enregistrer les résultats de l'exportation.
5. Dans la boîte de dialogue Enregistrer qui apparaît, spécifiez le nom du fichier et sélectionnez le format de fichier dans lequel vous souhaitez que les informations soient enregistrées, puis cliquez sur **Enregistrer**.
6. Cliquez sur **Exporter** pour commencer à enregistrer toutes les informations de preuve de lecture pour la période spécifiée dans le fichier spécifié.

 

#### **Suppression des informations de preuve de lecture**

* Après avoir exporté ou traité vos informations de preuve de lecture, vous devez supprimer les informations de PADS4.

  <Warning>
    Les informations de preuve de lecture ne sont pas automatiquement supprimées par le système PADS4, vous devez donc toujours les supprimer vous-même. Si vous ne supprimez pas les informations de preuve de lecture, vous risquez que celles-ci consomment tout l'espace de votre disque dur, entraînant des problèmes imprévisibles.
  </Warning>

Dans PADS4 Agent, vous avez la possibilité de supprimer soit des entrées individuelles de preuve de lecture, soit tous les éléments d'une base de données de preuve de lecture entière.

Comme alternative, vous pouvez accéder au dossier de preuve de lecture PADS4 sur votre PADS4 Server et supprimer manuellement les fichiers de base de données de preuve de lecture qui s'y trouvent. Enfin, vous pouvez également utiliser le PADS4 SDK pour développer votre propre solution personnalisée de traitement et de suppression des informations de preuve de lecture.

 

#### **Pour supprimer une entrée de journal de preuve de lecture**

1. Sélectionnez l'élément de preuve de lecture dans la fenêtre de preuve de lecture de votre PADS4 Server.\
   Optionnellement, vous pouvez sélectionner plusieurs éléments en appuyant sur la touche \[Ctrl] ou \[Shift] et en la maintenant enfoncée tout en cliquant sur les éléments.
2. Cliquez sur **Supprimer** dans le **Accueil** menu.

   ![EN Agent Ribbon Proof Of Play Remove Pn](/images/digital-signage/windows-applications/agent/remote-management-and-monitoring/EN-Agent-Ribbon-ProofOfPlayRemove.png)
3. Cliquez sur **Oui** pour confirmer que vous souhaitez supprimer l'entrée de journal de preuve de lecture.

Après cela, le système supprimera l'élément de preuve de lecture de la base de données de preuve de lecture sur votre PADS4 Server.

<Tip>
  La suppression des éléments de preuve de lecture est irréversible et ne doit normalement être effectuée qu'après que l'élément a été traité ou exporté. Étant donné que tous les éléments de preuve de lecture ont un numéro de séquence unique, vous pouvez toujours voir dans les informations exportées quand un ou plusieurs éléments ont été supprimés.
</Tip>

 

#### **Pour supprimer toutes les informations de preuve de lecture d'une période entière**

1. Sélectionnez la **Période de preuve de lecture** dans la fenêtre de preuve de lecture de votre PADS4 Server.
2. Cliquez sur **Tout supprimer** dans le **Accueil** menu.

   ![EN Agent Ribbon Proof Of Play Remove All Pn](/images/digital-signage/windows-applications/agent/remote-management-and-monitoring/EN-Agent-Ribbon-ProofOfPlayRemoveAll.png)
3. Cliquez sur **Oui** pour confirmer que vous souhaitez supprimer toutes les informations de preuve de lecture.

Après cela, le système supprimera toutes les informations de preuve de lecture pour la période spécifiée. Étant donné que chaque période correspond exactement à une base de données de preuve de lecture, cela signifie en pratique que la base de données de preuve de lecture correspondante sera supprimée du dossier de preuve de lecture de votre système PADS4.

Comme pour la suppression d'éléments individuels de preuve de lecture, la suppression d'une période complète de preuve de lecture est irréversible et ne doit normalement être effectuée qu'après que les éléments ont été traités ou exportés.
