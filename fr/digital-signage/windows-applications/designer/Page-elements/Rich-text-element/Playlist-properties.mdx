Ces propriétés définissent le **contenu textuel** affiché par l'élément **Rich Text**. Vous pouvez saisir un **élément de texte unique** ou créer une **playlist d'éléments textuels** qui seront lus séquentiellement.

***

### **Éditeur de texte enrichi avancé**

* **Formatage de base** (gras, italique, souligné, etc.) est disponible dans la barre d'outils.
* Pour accéder aux **fonctionnalités avancées** (intégration de fournisseur de données, règles, et plus), cliquez sur l'icône **Advanced Editor**.

***

### **Gestion des éléments de playlist**

* **Ajouter** de nouveaux éléments de texte.
* **Modifier** les éléments existants.
* **Changer l'ordre** des éléments de texte.
* **Supprimer** des éléments de texte.
* **Naviguer** à travers les éléments de la playlist à l'aide des boutons en bas.
* **Durée totale** de la playlist est affichée.

**Important:** Pour la gestion générale des playlists, consultez **Working with Playlists**.

***

### **Propriétés des éléments de playlist basés sur le texte**

| **Option**                                  | **Description**                                                                                                                                     |
| ------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Texte**                                   | Texte statique directement saisi dans l'élément.                                                                                                    |
| **Texte depuis la bibliothèque de contenu** | Utilise un fichier texte stocké dans la **Content Library** de PADS4 (recommandé).                                                                  |
| **Texte depuis un fichier**                 | Fait référence à un fichier texte externe en dehors de la Content Library. **Non synchronisé automatiquement**—nécessite une distribution manuelle. |
| **No Text**                                 | Insère une entrée vide, créant une pause dans la playlist.                                                                                          |

***

### **Propriété de contenu**

* Affiche le **contenu texte** ou **référence de fichier texte** selon le type sélectionné.
* **Parcourir la Content Library** ou **système de fichiers local** pour sélectionner des fichiers.
* Si **Text from File** est choisi, assurez-vous que le **chemin complet du fichier** est inclus.

***

### **Propriétés supplémentaires de la playlist**

| **Propriété** | **Description**                                                              |
| ------------- | ---------------------------------------------------------------------------- |
| **Durée**     | Spécifie la durée d'affichage du texte (en secondes).                        |
| **Effet**     | Applique un **effet de transition d'entrée** et définit la durée de l'effet. |

***

### **Remarques**

* **Texte multi-lignes :** Utilisez la touche **\[Enter]** pour séparer les lignes. Le retour à la ligne automatique dans l'éditeur **ne signifie pas automatiquement** un affichage multi-lignes.
* **Fichiers texte brut :** Les paramètres **Data Source Text Conversion** déterminent comment le texte multi-lignes est géré.
* **Formats pris en charge :** L'élément Rich Text peut lire **Plain Text (TXT), Rich Text Format (RTF), and HTML**.
* **Références de fichiers uniquement :** Si vous utilisez des fichiers texte externes, seules les **références** sont stockées. Si le fichier texte est mis à jour, la **dernière version** sera utilisée pendant la lecture.
* **Contenu piloté par les données :** Lorsqu'il est lié à une **source de données**, les propriétés de la playlist deviennent **désactivées**, car les données en temps réel déterminent dynamiquement le contenu.
