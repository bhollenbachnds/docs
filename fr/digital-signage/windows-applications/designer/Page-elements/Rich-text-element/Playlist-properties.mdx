Ces propriétés définissent le **contenu textuel** affiché par l'élément **Rich Text**. Vous pouvez saisir un **élément de texte unique** ou créer une **playlist d'éléments textuels** qui seront lus séquentiellement.

***

### **Éditeur de texte enrichi avancé**

* **Formatage de base** (gras, italique, souligné, etc.) est disponible dans la barre d'outils.
* Pour accéder aux **fonctionnalités avancées** (intégration de fournisseur de données, règles, et plus), cliquez sur l'icône **Advanced Editor**.

***

### **Gestion des éléments de playlist**

* **Ajouter** de nouveaux éléments de texte.
* **Modifier** les éléments existants.
* **Changer l'ordre** des éléments de texte.
* **Supprimer** des éléments de texte.
* **Naviguer** à travers les éléments de la playlist à l'aide des boutons en bas.
* **Durée totale** de la playlist est affichée.

**Important:** Pour la gestion générale des playlists, consultez **Working with Playlists**.

***

### **Propriétés des éléments de playlist basés sur le texte**

| **Option**                              | **Description**                                                                                                                                     |
| --------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Texte**                               | Texte statique directement saisi dans l'élément.                                                                                                    |
| **Texte de la bibliothèque de contenu** | Utilise un fichier texte stocké dans la **Content Library** de PADS4 (recommandé).                                                                  |
| **Texte à partir d'un fichier**         | Fait référence à un fichier texte externe en dehors de la Content Library. **Non synchronisé automatiquement**—nécessite une distribution manuelle. |
| **Pas de texte**                        | Insère une entrée vide, créant une pause dans la playlist.                                                                                          |

***

### **Propriété de contenu**

* Affiche le **contenu textuel** ou **la référence au fichier texte** selon le type sélectionné.
* **Parcourir la Content Library** ou **le système de fichiers local** pour sélectionner des fichiers.
* Si **Text from File** est choisi, assurez-vous que le **full file path** est inclus.

***

### **Propriétés supplémentaires de la playlist**

| **Propriété** | **Description**                                                            |
| ------------- | -------------------------------------------------------------------------- |
| **Duration**  | Spécifie la durée d'affichage du texte (en secondes).                      |
| **Effect**    | Applique un **incoming transition effect** et définit la durée de l'effet. |

***

### **Remarques**

* **Texte multi-lignes :** Utilisez la touche **\[Enter] key** pour séparer les lignes. Le retour à la ligne automatique dans l'éditeur **does not automatically mean** un affichage multi-lignes.
* **Fichiers texte brut :** Les paramètres **Data Source Text Conversion** déterminent comment le texte multi-lignes est traité.
* **Formats pris en charge :** L'élément Rich Text peut lire **Plain Text (TXT), Rich Text Format (RTF), and HTML**.
* **Références de fichiers uniquement :** Si vous utilisez des fichiers texte externes, seules les **references** sont stockées. Si le fichier texte est mis à jour, la **latest version** sera utilisée pendant la lecture.
* **Contenu piloté par les données :** Lorsqu'il est lié à une **data source**, les propriétés de la playlist deviennent **disabled**, car les données en temps réel déterminent dynamiquement le contenu.
