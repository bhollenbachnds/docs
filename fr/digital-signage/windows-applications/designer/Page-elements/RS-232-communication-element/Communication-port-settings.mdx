Ces propriétés définissent les **paramètres du port de communication RS-232**, garantissant une transmission de données réussie entre votre **système de présentation** et un **périphérique externe**.

***

## **Paramètres du port de communication**

### **Port de communication**

* Sélectionnez le **COM (série)** ou **LPT (parallèle)** port utilisé pour la **communication RS-232**.

### **Baud Rate**

* Définit la **vitesse de transmission** en **symboles par seconde (bps)**.
* Valeurs courantes : **1200, 2400, 4800, 9600, 19200, 38400, 57600, 115200**.
* Doit correspondre aux **paramètres du périphérique externe**.

### **Parity**

Détermine comment le **contrôle de parité** est appliqué pour la **détection d'erreurs**.

| **Option** | **Description**                                               |
| ---------- | ------------------------------------------------------------- |
| **None**   | Aucun contrôle de parité (par défaut).                        |
| **Even**   | Garantit un nombre pair de **1s** dans la trame de données.   |
| **Odd**    | Garantit un nombre impair de **1s** dans la trame de données. |
| **Mark**   | Définit toujours le bit de parité à **1**.                    |
| **Space**  | Définit toujours le bit de parité à **0**.                    |

### **Data Bits**

* Définit le nombre de **bits de données par trame**.
* Valeurs standard : **5, 6, 7 ou 8 bits** (par défaut : **8**).
* Doit correspondre aux **paramètres du périphérique externe**.

### **Start Bits**

* Spécifie les **bits de début** avant la transmission effective des données.
* Valeur standard : **1 bit**.

### **Stop Bits**

* Spécifie les **bits de fin de transmission**.
* Valeurs standard : **1, 1,5 ou 2 bits** (par défaut : **1**).
* Utilisé pour **séparer les paquets de données**.

### **Handshake**

Définit le **contrôle de flux** entre l'émetteur et le récepteur pour éviter la **perte de données**.

| **Option**   | **Description**                                                                                 |
| ------------ | ----------------------------------------------------------------------------------------------- |
| **None**     | Aucune négociation (par défaut).                                                                |
| **XOn/XOff** | Négociation logicielle. Utilise des caractères spéciaux pour contrôler le flux de données.      |
| **RTS/CTS**  | Négociation matérielle utilisant les signaux Ready-to-Send (RTS) et Clear-to-Send (CTS).        |
| **DTR/DSR**  | Négociation matérielle utilisant les signaux Data Terminal Ready (DTR) et Data Set Ready (DSR). |

### **Read Timeout**

* **Délai limite (en millisecondes)** pour recevoir une **réponse du périphérique externe**.
* Si aucune réponse n'est reçue dans ce délai, une **erreur de délai d'attente se produit**.

### **Write Timeout**

* **Délai limite (en millisecondes)** pour **envoyer une commande avec succès**.
* Si les données ne peuvent pas être transmises dans ce délai, une **erreur de délai d'attente se produit**.

***

## **Remarques**

* Les **paramètres RS-232 corrects** pour la communication avec votre **périphérique externe** se trouvent dans son **manuel d'utilisation**.
* **Des paramètres de port incorrects** peuvent entraîner un **échec de communication** ou un **comportement inattendu du périphérique**.

### **Important**

* Les paramètres du port de communication **doivent correspondre** à ceux du **périphérique externe**.
* Ces paramètres s'appliquent **uniquement pendant la lecture**, donc **les commandes RS-232 ne seront envoyées que si le port spécifié est disponible sur le PC de lecture**.
