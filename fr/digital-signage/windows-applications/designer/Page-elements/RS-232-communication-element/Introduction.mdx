L'**élément de communication RS-232** vous permet d'envoyer des **commandes RS-232** depuis votre présentation pour contrôler des périphériques externes, tels que des écrans, des projecteurs ou d'autres équipements à commande série. Ces commandes peuvent régler la luminosité, changer les canaux d'entrée ou basculer les états d'alimentation.

Pour une utilisation correcte, consultez la **documentation RS-232** de votre périphérique externe pour déterminer les commandes et configurations prises en charge.

Étant donné que l'élément de communication RS-232 **ne nécessite pas de présence visuelle**, il est généralement défini sur **invisible** dans les présentations. Cependant, il peut être connecté à une **source de données externe** pour déterminer, en temps réel, quelles commandes doivent être envoyées.

***

## **Aperçu des propriétés**

| **Section de propriété**      | **Description**                                                                                                             |
| ----------------------------- | --------------------------------------------------------------------------------------------------------------------------- |
| **Options alternatives**      | Définit comment l'élément basculera entre les éléments de la play list.                                                     |
| **Couleur d'arrière-plan**    | Définit la couleur d'arrière-plan de l'élément.                                                                             |
| **Bordure**                   | Ajuste la couleur, la largeur et le style de bordure de l'élément.                                                          |
| **Data Provider**             | Connecte l'élément à une **source de données** externe pour des mises à jour de commandes en temps réel.                    |
| **Data Scrolling**            | Détermine si et comment l'élément fait défiler les données en temps réel.                                                   |
| **Default Duration**          | Spécifie la durée par défaut pour les éléments de la **play list**.                                                         |
| **Default Transition Effect** | Définit l'**effet de transition** lors de la mise à jour du contenu.                                                        |
| **Flip**                      | Permet **le retournement horizontal ou vertical** du contenu.                                                               |
| **Police**                    | Définit les paramètres de police pour les **affichages de commandes RS-232 basés sur du texte**.                            |
| **Identification**            | Attribue un nom unique pour identifier l'élément dans la page.                                                              |
| **Play List**                 | Définit une **séquence de commandes RS-232** à envoyer.                                                                     |
| **Play List Options**         | Détermine comment les éléments de la play list sont exécutés.                                                               |
| **Port Settings**             | Définit les **paramètres du port RS-232**, notamment le débit en bauds, la parité, les bits de données et les bits d'arrêt. |
| **Position**                  | Détermine la position de l'élément dans la page.                                                                            |
| **Proof of Play**             | Enregistre les données de lecture à des fins de **rapport et de vérification**.                                             |
| **Property Bindings**         | Lie des propriétés spécifiques de l'élément à une source de données externe.                                                |
| **Reflection**                | Active un **effet de reflet** sur l'élément.                                                                                |
| **Rotation**                  | Vous permet de **faire pivoter** l'élément selon un certain angle.                                                          |
| **Rules**                     | Définit des **règles d'élément** pour un comportement dynamique.                                                            |
| **Shadow**                    | Ajoute un effet d'ombre à l'élément.                                                                                        |
| **Taille**                    | Définit la largeur et la hauteur de l'élément.                                                                              |
| **Alignement du texte**       | Ajuste l'**alignement horizontal et vertical** des commandes basées sur du texte.                                           |
| **Transparence**              | Définit le niveau de transparence de l'élément.                                                                             |
| **Visibilité**                | Contrôle si l'élément est visible pendant la lecture.                                                                       |

***

### **Remarques**

* **Assurez-vous que les paramètres RS-232 corrects** (débit en bauds, parité, bits de données, bits d'arrêt) correspondent aux spécifications de votre **périphérique externe**.
* **Les éléments RS-232 restent généralement invisibles**, car ils jouent un rôle fonctionnel plutôt que visuel.
* Si connecté à une **source de données externe**, les commandes peuvent changer dynamiquement en temps réel.
* **La journalisation Proof of Play** peut être utile pour surveiller quand et où les commandes RS-232 ont été exécutées.
