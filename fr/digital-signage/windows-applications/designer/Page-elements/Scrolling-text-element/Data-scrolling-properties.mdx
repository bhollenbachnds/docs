Lorsqu'un **scrolling text element** est connecté à une **data source**, ces propriétés définissent comment **real-time data** défile. **Data scrolling** parcourt les lignes de données, tandis que **visual scrolling** détermine comment le texte se déplace à l'écran.

***

## **Data Scrolling Settings**

### **Use Data Scrolling**

* Active ou désactive **scrolling through real-time data** dans l'élément.

### **Scrolling Step Size**

* Définit combien de **rows** l'élément avance chaque fois que les données sont actualisées.
* Example:
  * Si **Row = 2** et **Scrolling Step Size = 3**, l'élément affichera **Row 2, Row 5, Row 8**, et ainsi de suite.
  * Une fois que toutes les lignes disponibles ont été affichées, l'élément **loops back** à Row 2.

**Important:**

* Lorsque plusieurs éléments avec **data scrolling** sont placés dans une **table row**, ils **must** avoir le **same Scrolling Step Size** pour rester synchronisés.

***

## **Alternative Data Scrolling (For Table Elements Only)**

Lorsqu'un **scrolling text element** fait partie d'une **table row**, des options supplémentaires sont disponibles :

### **Number of Rows**

* Limite le défilement aux **first N rows** de la source de données.
* Utile lors de l'affichage uniquement des **most recent news items** d'un **RSS feed** ou d'une base de données.

**Tip:**

* Si vous devez afficher uniquement les **top news items**, utilisez **Number of Rows** pour limiter le défilement aux dernières entrées.

***

## **Additional Notes**

* **Data scrolling** les propriétés sont uniquement disponibles si l'élément est **connected to a data source**.
* Les données affichées sont déterminées par :
  * **Row property** (position de départ).
  * **Scroll Step Size** (combien de lignes à sauter par actualisation).
* Le processus de défilement **loops** à travers toutes les lignes disponibles et **restarts** lorsqu'il atteint la fin.
