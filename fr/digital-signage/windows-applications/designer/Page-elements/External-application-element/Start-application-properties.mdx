Ces propriétés définissent **quelle application externe** sera lancée par l'élément Application Externe et comment elle se comportera lors de son lancement.

***

### **Démarrer l'Application**

* **Ne pas démarrer cette Application si elle est déjà en cours d'exécution**\
  Si activé, PADS4 **ne démarrera pas une nouvelle instance** de l'application externe si elle est déjà en cours d'exécution.\
  Au lieu de cela, il **activera** l'instance en cours d'exécution.

***

### **Application**

Options pour spécifier quelle application doit être lancée :

| Option            | Description                                                                                  |
| ----------------- | -------------------------------------------------------------------------------------------- |
| **Manuel**        | Saisir manuellement le **chemin complet** vers l'exécutable de l'application.                |
| **Data Provider** | Définir dynamiquement le chemin de l'application via un **data provider**, colonne et ligne. |

* **Application**:\
  Le **chemin complet du fichier** de l'exécutable à lancer. Vous pouvez utiliser le **bouton parcourir** pour localiser le fichier.

> **Note**: Le chemin est **absolu** et doit être valide sur l'appareil de lecture.

***

### **Data Provider (Alternative optionnelle au mode Manuel)**

Un **Data Provider** peut être utilisé pour déterminer dynamiquement quelle application démarrer :

* **Data Provider**: La source de données externe (table, requête, XML, RSS, etc.).
* **Column**: La colonne spécifique dans la source de données contenant le chemin de l'application.
* **Row**: La ligne à partir de laquelle lire la valeur de l'application.

***

### **Arguments**

Certaines applications acceptent des **arguments de ligne de commande** pour des options de lancement avancées (par exemple, ouvrir un fichier, démarrer en mode silencieux).

| Option            | Description                                                                     |
| ----------------- | ------------------------------------------------------------------------------- |
| **Manuel**        | Spécifier manuellement les arguments utilisés pour lancer l'application.        |
| **Data Provider** | Définir dynamiquement les arguments via un **data provider**, colonne et ligne. |

* **Arguments**:\
  La chaîne complète d'arguments de ligne de commande à utiliser lors du lancement de l'application.

***

### **Remarques et conseils**

* Si des **espaces** sont présents dans le chemin de l'Application ou des Arguments, PADS4 **ajoute automatiquement** des guillemets `""`, donc **vous n'avez pas besoin de les ajouter manuellement**, sauf si nécessaire **à l'intérieur** de la commande.
* **Exemple** pour exécuter une présentation PowerPoint sans afficher la fenêtre PowerPoint :
  ```
  Application: C:\Program Files\Microsoft Office\Powerpoint Viewer\PPTView.exe  
  Arguments: /s sample.ppt
  ```
* Si le data provider contient **moins de lignes** que spécifié dans le champ **Row**, **aucune application ne sera démarrée**.
* **Les guillemets** sont ajoutés automatiquement lorsque nécessaire (par exemple, pour les chemins de fichiers contenant des espaces).
* Utile lors de l'intégration d'outils externes, de fichiers ou de processus dans votre flux de travail de présentation PADS4.
