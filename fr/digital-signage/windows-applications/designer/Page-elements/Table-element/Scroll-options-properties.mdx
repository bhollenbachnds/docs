Les **options de défilement** définissent la façon dont le **défilement des données** est géré dans l'élément tableau, notamment **les lignes sans défilement, l'affichage des données groupées et les transitions de page automatiques**.

***

## **Faire défiler les données une fois et afficher la page suivante**

* **Enabled:** Le tableau fait défiler toutes les **lignes de données disponibles une fois**. Après l'affichage de la dernière ligne, la présentation **passe automatiquement à la page suivante**.
* **Disabled:** Le tableau continue de défiler **indéfiniment**, en bouclant sur les données disponibles.

> **Meilleur cas d'utilisation :**
>
> * Lorsque le nombre de **lignes de données est inconnu**, mais que vous devez vous assurer que **chaque ligne est affichée exactement une fois** avant de passer à la page suivante.

***

## **Ne pas faire défiler les X premières lignes**

* **Définissez un nombre fixe de lignes** qui **resteront toujours visibles** pendant que les autres lignes défilent.
* Utile pour **afficher des informations critiques ou prioritaires** en haut du tableau.

> **Exemple de cas d'utilisation :**
>
> * **Aéroports** : Les **cinq premières lignes** restent statiques pour afficher les vols au départ prochainement, tandis que les autres vols défilent en dessous.
> * **Marchés boursiers** : Les **principales entreprises** restent visibles, tandis que les **actions moins importantes défilent**.

***

## **Faire défiler les données, regrouper par**

* Affiche **des ensembles de données groupées** un à la fois avant de passer au groupe suivant.
* **Sélectionnez une colonne** contenant les informations de regroupement.
* Le **Data Provider doit déjà être trié** selon la colonne de groupe sélectionnée.

> **Exemple de cas d'utilisation :**
>
> * **Réservations de chambres d'hôtel** : Affiche d'abord les **réservations de la chambre 101**, puis passe à la **chambre 102**, puis à la **chambre 103**, etc.
> * **Rapports de ventes d'entreprise** : Affiche les données de ventes **regroupées par département**, en veillant à ce que les données de chaque département soient affichées séparément.

***

## **Remarques importantes**

* **Le défilement des données doit être activé pour chaque élément d'une ligne** pour qu'il défile dans le tableau.
* **Le tri doit être configuré dans les paramètres du Data Provider** — l'élément tableau ne **pas** trier les données automatiquement.
