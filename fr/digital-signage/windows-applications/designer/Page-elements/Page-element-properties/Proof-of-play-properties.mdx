Les **Proof of Play** propriétés vous permettent de **consigner des informations détaillées** sur la lecture d'éléments spécifiques dans votre présentation. Ce journal peut être utilisé, par exemple, pour **démontrer aux annonceurs** que leur contenu a été diffusé, notamment quand, où et pendant combien de temps.

### Proof of Play

**Enregistrer les statistiques de l'élément**\
Lorsque cette option est activée, le PADS4 Viewer enregistre des informations détaillées chaque fois que l'élément est lu. Ces informations sont automatiquement envoyées au **PADS4 Server** pour une collecte et un stockage centralisés.

### Remarques

* Les informations de lecture sont stockées dans une **base de données dédiée au proof of play** sur le **PADS4 Server PC**.
* Cette base de données est accessible et consultable via :
  * **PADS4 Agent**, qui permet également d'exporter le journal en XML pour une analyse approfondie (par exemple, dans Microsoft Excel).
  * **PADS4 Software Development Kit (SDK)** pour créer des outils personnalisés permettant de traiter et d'analyser les données enregistrées.

### Important

Si vous activez **Proof of Play** la journalisation, il est essentiel d'établir une **procédure pour gérer et nettoyer régulièrement** la ou les bases de données proof of play.\
Sans maintenance appropriée, la base de données peut **croître indéfiniment**, finissant par consommer tout l'espace disque disponible, car **la suppression automatique des anciens enregistrements n'est pas effectuée par le système**.
