À la **deuxième étape** de l'**Assistant Fournisseur de données**, vous définissez un **nom unique** pour votre fournisseur de données et configurez les paramètres pour la **récupération de données en temps réel**.

***

## **Étapes pour configurer les paramètres généraux du fournisseur**

1. **Démarrer l'Assistant Fournisseur de données**
   * Suivez les étapes dans **Ajout d'un fournisseur de données** ou **Modification d'un fournisseur de données**
2. **Saisir un nom unique pour le fournisseur**
   * Attribuer un **Nom du fournisseur** (doit être unique dans votre présentation)
   * Facultativement, saisir une **Description**
3. **Décider d'utiliser ou non le serveur de données PADS4**
   * **Activer ou désactiver** l'option :\
     **"Utiliser le serveur de données pour la récupération des données"**
   * Si **activé**, les données en temps réel sont récupérées de manière centralisée via le **PADS4 Data Server**
   * Si **désactivé**, chaque **PADS4 Viewer** récupère les données **individuellement**
4. **Activer la mise en cache (facultatif)**
   * **Utiliser le cache de conception** : Stocke les données de la source pour la **prévisualisation en mode conception**
   * **Générer des valeurs d'exemple** : Crée des **valeurs de remplacement** si la source de données est indisponible
5. **Passer à l'étape suivante**

***

## **Utilisation du PADS4 Data Server**

Le **PADS4 Data Server** récupère et distribue les **données en temps réel** à tous les Viewers connectés.

### **Comparaison des méthodes de récupération de données**

| **Fonctionnalité**                 | **Utilisation du PADS4 Data Server**                  | **Récupération individuelle des données**                         |
| ---------------------------------- | ----------------------------------------------------- | ----------------------------------------------------------------- |
| **Récupération des données**       | Requête unique pour tous les Viewers                  | Chaque Viewer récupère les données séparément                     |
| **Performance**                    | Optimisée (réduit les requêtes de base de données)    | Charge plus importante sur la source de données                   |
| **Cas d'utilisation**              | Réseaux à grande échelle, sources de données lentes   | Requêtes spécifiques au Viewer (ex. : filtrage par nom de Viewer) |
| **Prise en charge du basculement** | Bascule vers un PADS4 Server de secours si nécessaire | Aucune redondance intégrée                                        |

### **Quand activer le PADS4 Data Server**

✔ **Recommandé si :**

* **Plusieurs Viewers** ont besoin des **mêmes données** en même temps
* Votre **source de données est lente ou externe** (ex. : API, bases de données distantes)
* Vous souhaitez **réduire le trafic réseau et la charge sur la base de données**

✘ **Non recommandé si :**

* Chaque **Viewer nécessite des données différentes** (ex. : filtré par emplacement)
* La **base de données optimise déjà les requêtes** pour plusieurs connexions

***

## **Options de mise en cache dans PADS4**

La mise en cache peut **améliorer les performances** et **permettre la conception hors ligne** lorsque les données en direct ne sont pas disponibles.

| **Option de mise en cache**         | **Fonction**                                                                 |
| ----------------------------------- | ---------------------------------------------------------------------------- |
| **Utiliser le cache de conception** | Enregistre **jusqu'à 999 lignes** de données pour une utilisation hors ligne |
| **Générer des valeurs d'exemple**   | Remplace les données manquantes par un **texte de remplacement**             |
| **Idéal pour**                      | Concevoir des présentations **sans accès aux données en direct**             |

### **Exemple de cas d'utilisation**

Un **concepteur** doit créer une **mise en page pilotée par les données** mais n'a pas **accès à la base de données**. En utilisant des **données en cache**, il peut quand même **concevoir et tester** la mise en page.

***

## **Considérations relatives au basculement et à la sauvegarde**

* Si le **PADS4 Data Server** tombe en panne, un **PADS4 Server de sauvegarde** (si configuré) prend le relais.
* Si le **serveur de données est désactivé**, assurez-vous que **chaque Viewer** peut **accéder de manière indépendante** à la source de données.
