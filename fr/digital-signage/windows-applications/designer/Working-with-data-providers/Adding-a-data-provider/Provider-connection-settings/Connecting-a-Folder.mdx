Le **Folder** provider type vous permet de vous connecter à un **dossier spécifique** pour récupérer et afficher son **contenu de fichier**. Vous pouvez inclure **tous les fichiers** ou les filtrer en fonction des **extensions**. **Subfolders** peuvent également être inclus si nécessaire.

***

## **Étapes pour ajouter/modifier une connexion de dossier dans PADS4**

### **1. Démarrer le Data Provider Wizard**

* Ouvrez le **Data Provider Wizard** en suivant les instructions dans **Adding a Data Provider** ou **Editing a Data Provider**.
* Sélectionnez **Folder** comme **provider type**.

### **2. Spécifier les paramètres de connexion du dossier**

* Choisissez l'une des **options de source** suivantes :
  * **Content Library:** Utilise un dossier de la **PADS4 Content Library** (Recommandé).
  * **Folder:** Utilise un **dossier externe** à la content library.

* **Entrez le chemin du dossier** :
  * Si vous utilisez **Content Library**, fournissez le **chemin relatif**.
  * Si vous utilisez un **dossier externe**, fournissez le **chemin absolu** (par exemple, `C:\MyFolder`).
  * Utilisez le bouton **Browse** pour localiser le dossier.

* **Include Subfolders** *(Optionnel)*
  * Activez cette option pour inclure les fichiers **des sous-dossiers** du dossier sélectionné.

* **Définir les filtres de fichiers**
  * Utilisez **Search Extension Restriction** pour **filtrer des types de fichiers spécifiques** (par exemple, `.jpg`, `.png`, `.mp4`).

* **Trier les fichiers**
  * Utilisez les paramètres **Sorting File List** pour **définir l'ordre de tri**.

### **3. Finaliser la connexion**

* Cliquez sur **Next** pour continuer.
* Complétez l'assistant comme décrit dans **Adding a Data Provider**.

***

## **Meilleures pratiques**

* **Utiliser la PADS4 Content Library**
  * Les fichiers dans la **Content Library** sont **automatiquement synchronisés** sur **tous les systèmes PADS4**.
  * Si vous utilisez un **dossier externe**, vous devez **distribuer manuellement** les fichiers sur tous les systèmes.

* **Éviter les dossiers réseau partagés**
  * **Le trafic réseau peut augmenter** lors de l'utilisation d'un **lecteur mappé** ou d'un **chemin UNC** (`\\server\folder`).
  * Cela peut **ralentir** les **performances du Viewer** lors de la récupération des listes de fichiers ou du chargement du contenu.

* **Règles de tri**
  * **Le tri spécifique au dossier** s'applique en premier.
  * **Le tri générique** (défini dans les étapes ultérieures de l'assistant) **remplace le tri du dossier**.

***

## **Considérations importantes**

* **Accès au Data Server**
  * Si vous utilisez un **PADS4 Data Server**, assurez-vous que le **dossier est accessible** du **point de vue du serveur**.
  * Si vous **n'utilisez pas de Data Server**, chaque **PADS4 Designer et Viewer** doit avoir un **accès direct** au dossier.

En suivant ces étapes, vous pouvez réussir à **connecter un dossier** et récupérer son contenu pour l'utiliser dans des **présentations d'affichage dynamique**.
