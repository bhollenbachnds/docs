JSON (**JavaScript Object Notation**) est un format largement utilisé pour **stocker et échanger des données structurées**. Dans **PADS4**, les fichiers JSON peuvent être utilisés comme **sources de données** pour le contenu d'affichage dynamique en temps réel.

***

### **Étapes pour ajouter ou modifier une connexion de fichier JSON**

#### **1. Démarrer l'assistant de fournisseur de données**

* Ouvrir l'**assistant de fournisseur de données** en suivant les instructions dans **Ajout d'un fournisseur de données** ou **Modification d'un fournisseur de données**.
* Sélectionner **JSON File** comme type de fournisseur.

***

#### **2. Spécifier l'emplacement du fichier JSON**

* **Content Library** → Sélectionner cette option pour utiliser un fichier JSON stocké dans la **PADS4 Content Library**.
* **File Name** → Choisir cette option pour utiliser un fichier JSON situé **en dehors de la bibliothèque de contenu**.
* Utiliser la fonction de navigation pour localiser et sélectionner le fichier JSON souhaité.

**Recommandation** :\
Si **vous n'utilisez pas de PADS4 Data Server**, il est préférable de stocker les fichiers JSON dans la **PADS4 Content Library**. Cela garantit une **synchronisation automatique** sur tous les ordinateurs connectés. Si un fichier JSON est **en dehors de la bibliothèque**, la distribution des fichiers doit être gérée manuellement.

***

#### **3. Configurer l'encodage du texte**

* Si le fichier JSON **nécessite un format d'encodage de texte spécifique**, sélectionner l'option appropriée.

***

#### **4. Configurer le formatage des données**

* **Flatten Hierarchical Data** → Si la structure JSON contient plusieurs niveaux, cette option **aplatit les données imbriquées** en un format tabulaire.
* **Include Parent Levels Column Names** → Ajoute les noms des niveaux parents pour une **représentation hiérarchique plus claire**.
* **Create Additional Columns Without HTML Tags** → Supprime les balises HTML des champs de texte et les **convertit en texte brut**.

Passer à l'étape suivante.

***

#### **5. Finalisation de la configuration du fichier JSON**

* Terminez la configuration du fournisseur de données.

***

## **Considérations importantes**

* **Utilisation du PADS4 Data Server**
  * Si un **PADS4 Data Server** est utilisé, assurez-vous que le fichier JSON **est accessible** depuis le **point de vue du serveur**.
  * Si **aucun Data Server n'est utilisé**, confirmez que **PADS4 Designer et les PADS4 Viewers** peuvent accéder à l'emplacement du fichier JSON spécifié.

* **Gestion des données JSON hiérarchiques**
  * Si la structure JSON contient plusieurs niveaux, l'activation de **Flatten Hierarchical Data** garantit une meilleure compatibilité.

En suivant ces étapes, **PADS4 peut intégrer et afficher de manière transparente des données au format JSON** dans des présentations d'affichage dynamique.
