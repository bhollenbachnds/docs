**JSON (JavaScript Object Notation)** est un format de données structuré et léger largement utilisé pour **l'échange de données entre les services web et les applications**. PADS4 permet l'intégration avec **des flux JSON (depuis Internet) ou des fichiers JSON (stockage local)** en tant que fournisseurs de données.

***

## **Étapes pour ajouter ou modifier une connexion JSON dans PADS4**

### **1. Démarrer l'assistant de fournisseur de données**

* Ouvrez l'**Assistant de fournisseur de données** en suivant les étapes dans **Ajout d'un fournisseur de données** ou **Modification d'un fournisseur de données**.
* Choisissez **Flux JSON** (pour une source en ligne) ou **Fichier JSON** (pour une source locale).

***

### **2. Spécifier la source JSON**

* **Pour les flux JSON (en ligne)**
  * Entrez l'**URL (URI)** du flux JSON.
  * Assurez-vous que l'URL est **accessible publiquement** ou dispose des paramètres d'authentification appropriés.

* **Pour les fichiers JSON (locaux)**
  * Spécifiez le **chemin du fichier**.
  * Le fichier peut être situé dans la **Bibliothèque de contenu PADS4** ou dans un **emplacement externe**.
  * S'il est stocké **en dehors de la Bibliothèque de contenu PADS4**, il **ne se synchronisera pas automatiquement** avec le viewer.

Cliquez sur **Suivant** pour continuer.

***

### **3. Sélectionner la source de données**

* **Les structures JSON** peuvent contenir plusieurs **tables et données imbriquées**.
* Cliquez sur **Sélectionner** pour ouvrir une **vue arborescente** de la structure JSON.
* Parcourez les **différents niveaux** pour trouver les données appropriées.
* Une fois trouvées, cliquez sur **OK** pour confirmer.

#### **Sélection avancée (pour les utilisateurs techniques)**

* Modifiez l'**expression de chemin JSON** manuellement.
* Utilisez la **syntaxe JSONPath** pour un filtrage et une sélection avancés.
* Consultez la **documentation JSONPath** pour plus de détails.

***

### **4. Configurer les paramètres de gestion des données**

* **Inclure le nom de colonne du niveau parent** → Ajoute des identifiants de niveau parent pour les relations hiérarchiques.
* **Créer des colonnes supplémentaires sans balises HTML5** → Supprime le **formatage HTML** et extrait le texte brut.
* **Aplatir les données hiérarchiques** → Convertit les **structures JSON imbriquées** en un tableau de données plat.

Cliquez sur **Suivant** pour continuer.

***

### **5. Configurer la méthode de requête**

* **GET (par défaut)** → Récupère les données sans les modifier.
* **POST** → Utilisé pour envoyer des paramètres supplémentaires (remplissez le **corps de la requête** si nécessaire).
* **Spécifier l'encodage** → Choisissez un format d'encodage de texte si nécessaire.

***

### **6. Configurer la sécurité et l'authentification**

* **Autorisation (si requise)**
  * Activez **Autorisation requise**.
  * Entrez **l'identifiant utilisateur, le mot de passe et le domaine**.

* **Serveur proxy (le cas échéant)**
  * Configurez les paramètres du proxy si l'accès à Internet nécessite une authentification.

* **Identification de l'agent utilisateur**
  * Certains serveurs nécessitent une **chaîne User-Agent spécifique** (par exemple, pour l'accès à une API).

Cliquez sur **Suivant** pour continuer.

***

### **7. Finalisation de la configuration du fournisseur JSON**

* Cliquez sur **Terminer** pour finaliser la configuration.

***

## **Considérations importantes**

* **Serveur de données PADS4**
  * Si vous utilisez un **Serveur de données**, assurez-vous que la source JSON est **accessible depuis le serveur**.
  * Si vous n'utilisez pas de Serveur de données, le **Designer et les Viewers** doivent avoir **un accès direct** au flux ou au fichier JSON.

* **Joindre plusieurs tables de données**
  * Si vous avez besoin de **combiner des données** provenant de plusieurs sources JSON, utilisez le **Join Data Provider**.

En suivant ces étapes, **PADS4 peut intégrer de manière transparente des sources de données JSON** dans **des présentations d'affichage dynamique en temps réel**.
