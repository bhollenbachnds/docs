Le **Microsoft Access provider type** permet **PADS4** de se connecter aux **Microsoft Access databases** (**Office 2010 or earlier**).

***

### **Exigences**

* **Office System Drivers**
  * Si vous utilisez **Windows 7** ou certaines autres versions de Windows, vous pourriez avoir besoin d'**install the Office System drivers** avant de vous connecter à une **Microsoft Access database**.
  * Ces pilotes sont disponibles **gratuitement** sur le **Microsoft website**.

***

### **Étapes pour se connecter à une Microsoft Access Database**

#### **1. Démarrer le Data Provider Wizard**

* Ouvrez le **Data Provider Wizard** en suivant les instructions dans **Adding a Data Provider** ou **Editing a Data Provider**.
* Sélectionnez **Microsoft Access** comme **provider type**.

***

#### **2. Spécifier l'emplacement du fichier Access Database**

* Choisissez entre :
  * **Content Library** (Recommandé) → Utilise une base de données stockée dans la **PADS4 Content Library**.
  * **File Name** → Utilise une base de données **outside the Content Library** (doit spécifier le chemin complet du fichier).
* Cliquez sur **Browse** pour localiser et sélectionner le fichier **.mdb** ou **.accdb**.

**Pour les bases de données protégées :**

* Si votre base de données nécessite une authentification, saisissez le **Login Name** et le **Password**.
* Cliquez sur **Test** pour vérifier la connexion.

***

#### **3. Sélectionner la source de données**

* Choisissez le type de données à récupérer :
  * **Tables** – Sélectionnez une table dans la liste déroulante.
  * **Queries** – Sélectionnez une requête qui existe déjà dans la base de données.
  * **Parameter Queries** – Si votre requête nécessite des paramètres, configurez-les en conséquence.
  * **SQL Statement** – Saisissez manuellement une **SQL query** (À utiliser uniquement si aucune table ou requête prédéfinie ne répond à vos besoins).

**Tip:**\
L'utilisation de **SQL Statements** peut réduire les performances car elles doivent être **validated every time** que les données sont récupérées.

***

#### **4. Terminer la configuration**

* Cliquez sur **Next** pour continuer.
* Terminez la configuration en complétant le **Data Provider Wizard**.

***

### **Considérations importantes**

* **Utilisation d'un PADS4 Data Server**
  * Si un **PADS4 Data Server** est utilisé, assurez-vous que les **Office System drivers** sont **installed on the server**.
  * La **Access database must be accessible** du **server's point of view**.
* **Sans PADS4 Data Server**
  * Assurez-vous que **PADS4 Designer** (pour la prévisualisation) et **PADS4 Viewers** ont les **Office System drivers installed**.
  * Confirmez que **each Viewer can access the database**.
* **Évitez d'utiliser des dossiers réseau partagés**
  * **L'utilisation d'un dossier réseau partagé** (via un **mapped drive** ou un **UNC path**) pour une **Access database** peut provoquer un **high network traffic** chaque fois que les données sont actualisées.
  * Il est **recommandé** de **store the database in the PADS4 Content Library** ou d'utiliser **a centralized SQL database** pour de meilleures performances.

***

En suivant ces étapes, PADS4 peut se connecter avec succès à une **Microsoft Access database** et récupérer des données en temps réel.
