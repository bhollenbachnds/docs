Le **Office 365 Microsoft Exchange** fournisseur dans **PADS4** permet des connexions à **Exchange Online** (basé sur le cloud). Cela permet de récupérer des données telles que **calendriers, messages de boîte de réception et disponibilité des utilisateurs**.

***

### **Étapes pour ajouter ou modifier une connexion Office 365 Microsoft Exchange**

#### **1. Démarrer l'Assistant Fournisseur de données**

* Ouvrez l'**Assistant Fournisseur de données** dans **PADS4**.
* Suivez les étapes pour **Ajouter un fournisseur de données** ou **Modifier un fournisseur de données**.
* Sélectionnez **Office 365 Microsoft Exchange** comme **type de fournisseur**.

***

#### **2. Configurer les paramètres de connexion**

* **Spécifier les paramètres de proxy (si nécessaire)**
  * Si votre organisation utilise un **serveur proxy**, saisissez les paramètres requis.
  * Contactez votre **administrateur système** pour plus de détails.

* **S'authentifier auprès de Microsoft Exchange**
  * Cliquez sur le bouton **Authentifier**.
  * Une **page de connexion Microsoft** s'ouvrira.
  * Connectez-vous avec vos **identifiants Exchange Online** et autorisez **PADS4** pour accéder à votre environnement.
  * Une fois l'authentification réussie, le **status** affichera **successful**.

* **Reauthentication**
  * Chaque fois que vous modifiez ce **data provider**, vous devrez vous réauthentifier.

Cliquez sur **Next** pour continuer.

***

#### **3. Choisir la Source de Données à Récupérer**

Sélectionnez le type de **Exchange data** à récupérer :

| **Option**            | **Description**                                                                                                                                      |
| --------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------- |
| **My Calendar**       | Récupère les événements du calendrier de votre **personal** Exchange calendar.                                                                       |
| **Shared Calendar**   | Récupère les **shared** calendars en spécifiant des adresses e-mail. Utile pour les **meeting rooms or group events**.                               |
| **User Availability** | Récupère le **free/busy status** pour les utilisateurs spécifiés. Utile pour les **scheduling applications**.                                        |
| **My Inbox**          | Récupère les **emails** de votre **personal** inbox. Spécifiez éventuellement un dossier et si vous souhaitez inclure les **subfolders**.            |
| **Shared Inbox**      | Récupère les emails d'une **shared** inbox en spécifiant des adresses e-mail. Fusionnez éventuellement les données de plusieurs boîtes de réception. |

Cliquez sur **Next** pour continuer.

***

#### **4. Configurer les Options de Filtrage des Données**

* **Include Message Text**
  * Cochez cette option pour **retrieve the email body** avec les autres métadonnées.

* **Specify Date Range (From / Until)**
  * Utilisez une période **static** ou **dynamic** pour filtrer les résultats.
  * Réduit **memory usage** et **increases performance**.

**Note:**\
Si vous récupérez une **large period of data**, cela peut entraîner une **high memory usage**. Utilisez des filtres ou définissez un **maximum number of rows** pour éviter des charges de données excessives.

Cliquez sur **Next** pour continuer.

***

#### **5. Finaliser la Configuration**

* Complétez les étapes restantes dans le **Data Provider Wizard**.
* Cliquez sur **Finish** pour finaliser la configuration.

***

### **Notes Supplémentaires**

* Si **authentication fails**, assurez-vous que votre **Microsoft 365 account** dispose des **permissions** nécessaires pour accéder à **Exchange Online**.
* Si des **large data sets** causent des problèmes de performance, **limit the time range or the number of rows retrieved**.
* Si vous utilisez un **proxy server**, assurez-vous que les **proxy settings** sont correctement configurés.

Cette configuration permet à **PADS4** de récupérer et d'afficher en temps réel les **Exchange data** pour les **emails, calendars, and availability**.
