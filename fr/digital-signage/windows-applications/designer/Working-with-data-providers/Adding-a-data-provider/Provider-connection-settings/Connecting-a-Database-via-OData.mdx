**OData (Open Data Protocol)** est un protocole web utilisé pour **interroger et récupérer des données structurées** depuis diverses sources telles que des bases de données relationnelles, des systèmes de fichiers et des systèmes de gestion de contenu. Il s'appuie sur **HTTP, JSON et AtomPub** pour exposer les données en tant que service web.

***

## **Étapes pour se connecter à une source de données OData**

### **1. Démarrer l'assistant Data Provider**

* Suivez les étapes décrites dans **Ajout d'un Data Provider** ou **Modification d'un Data Provider**.
* Sélectionnez **OData** comme type de fournisseur.

### **2. Spécifier l'URL du service web OData**

* Entrez l'**URL** où la **source de données OData** est située.
* Le système n'acceptera que les données conformes aux **spécifications officielles OData** de [www.odata.org](http://www.odata.org).

#### **Exemple d'URL**

Un exemple couramment utilisé est la **base de données exemple Northwind**:

```plaintext
http://services.odata.org/Northwind/Northwind.svc/
```

* Si votre **source OData est sécurisée**, activez **Authorization Required** et entrez :
  * **User ID**
  * **Password**
  * **Domain** (optionnel)

* Cliquez sur le bouton **Test** pour vérifier la connexion.

* Cliquez sur **Next** pour continuer.

***

### **3. Sélectionner la source de données**

* Si la **source de données est détectée automatiquement**, sélectionnez **Predefined** et choisissez la table ou l'entité appropriée dans la liste.

* Si **aucune source de données prédéfinie n'est détectée**, sélectionnez **Manual** et entrez manuellement le **nom de la table, la requête ou le nom de l'entité**.

* Optionnellement, utilisez **Skip first number of rows** pour supprimer un nombre spécifié de lignes du jeu de données récupéré.

* Cliquez sur **Next** pour continuer.

***

### **4. Finaliser la configuration**

* Suivez les étapes restantes dans le guide **Ajout d'un Data Provider**.
* Cliquez sur **Finish** pour enregistrer les paramètres de connexion.

***

## **Considérations importantes**

* **Assurer l'accès réseau :**
  * Si vous utilisez un **PADS4 Data Server**, vérifiez que la **source OData est accessible** du point de vue du serveur.
  * Si **vous n'utilisez pas de Data Server**, assurez-vous que **chaque PADS4 Viewer et le Designer peuvent accéder directement à la source OData**.

* **Sécurité et authentification :**
  * Si le **service OData nécessite une authentification**, fournissez des **identifiants de connexion valides**.

* **Test de la connexion :**
  * Utilisez le bouton **Test** avant de continuer pour vous assurer que les informations saisies sont correctes.

* **Ignorer des lignes :**
  * Si votre **source OData contient des en-têtes** ou des premières lignes non pertinentes, utilisez l'option **Skip first number of rows**.

***

En intégrant **OData avec PADS4**, les entreprises peuvent se connecter efficacement à **des bases de données cloud, des API REST et des applications d'entreprise** pour récupérer **des données structurées en temps réel** pour **l'affichage dynamique de contenu et la signalétique numérique**.
