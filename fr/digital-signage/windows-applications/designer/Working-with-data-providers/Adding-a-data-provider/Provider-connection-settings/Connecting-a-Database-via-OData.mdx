**OData (Open Data Protocol)** est un protocole web utilisé pour **interroger et récupérer des données structurées** depuis diverses sources telles que des bases de données relationnelles, des systèmes de fichiers et des systèmes de gestion de contenu. Il s'appuie sur **HTTP, JSON et AtomPub** pour exposer les données en tant que service web.

***

## **Étapes pour se connecter à une source de données OData**

### **1. Démarrer l'assistant de fournisseur de données**

* Suivez les étapes décrites dans **Ajout d'un fournisseur de données** ou **Modification d'un fournisseur de données**.
* Sélectionnez **OData** comme type de fournisseur.

### **2. Spécifier l'URL du service web OData**

* Saisissez l'**URL** où la **source de données OData** est située.
* Le système n'acceptera que les données conformes aux **spécifications officielles OData** de [www.odata.org](http://www.odata.org).

#### **Exemple d'URL**

Un exemple couramment utilisé est la **base de données exemple Northwind**:

```plaintext
http://services.odata.org/Northwind/Northwind.svc/
```

* Si votre **source OData est sécurisée**, activez **Autorisation requise** et saisissez :
  * **Identifiant utilisateur**
  * **Mot de passe**
  * **Domaine** (facultatif)

* Cliquez sur le bouton **Test** pour vérifier la connexion.

* Cliquez sur **Suivant** pour continuer.

***

### **3. Sélectionner la source de données**

* Si la **source de données est détectée automatiquement**, sélectionnez **Prédéfini** et choisissez la table ou l'entité appropriée dans la liste.

* Si **aucune source de données prédéfinie n'est détectée**, sélectionnez **Manuel** et saisissez manuellement le **nom de la table, la requête ou le nom de l'entité**.

* Optionnellement, utilisez **Ignorer le premier nombre de lignes** pour supprimer un nombre spécifié de lignes du jeu de données récupéré.

* Cliquez sur **Suivant** pour continuer.

***

### **4. Finaliser la configuration**

* Suivez les étapes restantes dans le guide **Ajout d'un fournisseur de données**.
* Cliquez sur **Terminer** pour enregistrer les paramètres de connexion.

***

## **Considérations importantes**

* **Assurer l'accès réseau :**
  * Si vous utilisez un **PADS4 Data Server**, vérifiez que la **source OData est accessible** depuis le point de vue du serveur.
  * Si **vous n'utilisez pas de Data Server**, assurez-vous que **chaque PADS4 Viewer et le Designer peuvent accéder directement à la source OData**.

* **Sécurité et authentification :**
  * Si le **service OData nécessite une authentification**, fournissez des **identifiants de connexion valides**.

* **Test de la connexion :**
  * Utilisez le bouton **Test** avant de continuer pour vous assurer que les informations saisies sont correctes.

* **Ignorer des lignes :**
  * Si votre **source OData contient des en-têtes** ou des premières lignes non pertinentes, utilisez l'option **Ignorer le premier nombre de lignes**.

***

En intégrant **OData avec PADS4**, les entreprises peuvent se connecter efficacement à des **bases de données cloud, des API REST et des applications d'entreprise** pour récupérer des **données en temps réel et structurées** pour **l'affichage dynamique et la signalétique numérique**.
