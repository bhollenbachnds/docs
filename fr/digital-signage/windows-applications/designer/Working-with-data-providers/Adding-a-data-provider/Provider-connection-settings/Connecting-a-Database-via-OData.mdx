**OData (Open Data Protocol)** est un protocole basé sur le web utilisé pour **interroger et récupérer des données structurées** à partir de diverses sources telles que des bases de données relationnelles, des systèmes de fichiers et des systèmes de gestion de contenu. Il utilise **HTTP, JSON, et AtomPub** pour exposer les données sous forme de service web.

***

## **Étapes pour se connecter à une source de données OData**

### **1. Démarrer l'Assistant Fournisseur de données**

* Suivez les étapes décrites dans **Adding a Data Provider** ou **Editing a Data Provider**.
* Sélectionnez **OData** comme type de fournisseur.

### **2. Spécifier l'URL du service Web OData**

* Entrez l'**URL** où se trouve la **source de données OData**.
* Le système n'acceptera que les données qui suivent les **spécifications officielles OData** de [www.odata.org](http://www.odata.org).

#### **Exemple d'URL**

Un exemple couramment utilisé est la **base de données exemple Northwind**:

```plaintext
http://services.odata.org/Northwind/Northwind.svc/
```

* Si votre **La source OData est sécurisée**, activez **Autorisation requise** et saisissez le :
  * **ID utilisateur**
  * **Mot de passe**
  * **Domaine** (facultatif)

* Cliquez sur le bouton **Test** pour vérifier la connexion.

* Cliquez sur **Suivant** pour continuer.

***

### **3. Sélectionnez la source de données**

* Si la **source de données est automatiquement détectée**, sélectionnez **Prédéfini** et choisissez la table ou l'entité appropriée dans la liste.

* Si **aucune source de données prédéfinie n'est détectée**, sélectionnez **Manuel** et saisissez manuellement le **nom de la table, la requête ou le nom de l'entité**.

* Optionnellement, utilisez **Ignorer les premières lignes** pour supprimer un nombre spécifié de lignes de l'ensemble de données récupéré.

* Cliquez sur **Suivant** pour continuer.

***

### **4. Terminez la configuration**

* Suivez les étapes restantes dans le guide **Adding a Data Provider**.
* Cliquez sur **Terminer** pour enregistrer les paramètres de connexion.

***

## **Considérations importantes**

* **Assurer l'accès réseau :**
  * Si vous utilisez un **PADS4 Data Server**, vérifiez que la **source OData est accessible** du point de vue du serveur.
  * Si **vous n'utilisez pas de Data Server**, assurez-vous que **chaque PADS4 Viewer et le Designer peuvent accéder directement à la source OData**.

* **Sécurité et authentification :**
  * Si le **service OData nécessite une authentification**, fournissez des **identifiants de connexion valides**.

* **Test de la connexion :**
  * Utilisez le bouton **Test** avant de continuer pour vous assurer que les détails saisis sont corrects.

* **Ignorer des lignes :**
  * Si votre **source OData contient des en-têtes** ou des premières lignes non pertinentes, utilisez l'option **Ignorer les premières lignes**.

***

En intégrant **OData with PADS4**, les entreprises peuvent se connecter efficacement aux **bases de données cloud, API REST et applications d'entreprise** pour récupérer des **données structurées en temps réel** pour **l'affichage dynamique et la signalisation numérique**.
