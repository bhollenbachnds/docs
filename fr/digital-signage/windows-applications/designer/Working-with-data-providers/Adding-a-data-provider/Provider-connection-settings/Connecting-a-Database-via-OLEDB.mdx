**OLEDB (Object Linking and Embedding Database)** est un **modèle générique** pour accéder à diverses **bases de données standard de l'industrie**. Par rapport à **ODBC**, OLEDB offre **plus de fonctionnalités** mais s'accompagne d'une **surcharge plus élevée**.

***

## **Prérequis**

Avant de configurer une **connexion OLEDB** dans **PADS4**, assurez-vous que :

* Le **pilote OLEDB** pour la base de données requise **est installé** sur votre système.
* La **base de données est accessible** depuis le **PADS4 Data Server** ou la **machine locale**.

***

## **Étapes pour ajouter/modifier une connexion OLEDB dans PADS4**

### **1. Démarrer l'Assistant fournisseur de données**

* Suivez les étapes décrites dans **Ajout d'un fournisseur de données** ou **Modification d'un fournisseur de données**.
* Sélectionnez **OLEDB** comme type de fournisseur.

### **2. Saisissez la chaîne de connexion OLEDB**

* Fournissez la **chaîne de connexion OLEDB** pour établir un lien avec votre base de données.
* Le format de cette chaîne dépend du **pilote OLEDB** utilisé.
* Visitez [www.connectionstrings.com](https://www.connectionstrings.com) pour référence.

#### **Exemple de chaîne de connexion pour Microsoft Access**

```plaintext
Provider=Microsoft.ACE.OLEDB.12.0;Data Source=C:\MyDatabase.accdb;Persist Security Info=True
```

* Cliquez sur **Test** pour valider la connexion.
* Cliquez sur **Next** pour continuer.

***

### **3. Sélectionnez la source de données**

* Choisissez l'une des sources de données détectées :
  * **Tables**
  * **Views**
  * **Stored Procedures**
* Si aucune source de données prédéfinie n'est disponible, choisissez **SQL Statement** et saisissez manuellement une **requête SQL**.

#### **Exemple de requête SQL**

```sql
SELECT * FROM Customers
```

**Tip:**\
N'utilisez les **instructions SQL** que lorsque c'est nécessaire. Les requêtes écrites manuellement doivent être validées pendant l'exécution, ce qui peut **affecter les performances**.

* Cliquez sur **Next** pour continuer.

***

### **4. Terminez la configuration**

* Suivez les étapes restantes dans le guide **Adding a Data Provider**.
* Cliquez sur **Finish** pour enregistrer les paramètres de connexion.

***

## **Considérations importantes**

* **Assurer l'accès réseau :**
  * Si vous utilisez un **PADS4 Data Server**, le **pilote OLEDB doit être installé** sur le **serveur**, et la base de données **doit être accessible**.
  * Si **vous n'utilisez pas de Data Server**, assurez-vous que **chaque PADS4 Viewer et Designer** dispose du **pilote OLEDB correct installé**.

* **Test de la connexion :**
  * Utilisez le bouton **Test** pour vérifier la connexion **avant de continuer**.

* **Considérations de performance :**
  * Utilisez des **Tables, Views ou Stored Procedures prédéfinis** pour **de meilleures performances** au lieu de **requêtes SQL** brutes.

En suivant ces étapes, vous pouvez **connecter avec succès une base de données OLEDB à PADS4** et récupérer des **données en temps réel** pour **l'affichage dynamique et les mises à jour automatiques de contenu**.
