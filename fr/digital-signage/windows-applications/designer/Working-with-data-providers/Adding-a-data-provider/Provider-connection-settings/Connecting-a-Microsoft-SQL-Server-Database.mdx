Le fournisseur **Microsoft SQL Server** dans **PADS4** vous permet de vous connecter à une base de données **SQL Server**, de récupérer des données et de les utiliser dans votre présentation.

***

### **Étapes pour ajouter ou modifier une connexion Microsoft SQL Server**

#### **1. Démarrer l'Assistant de fournisseur de données**

* Ouvrez **PADS4** et accédez à **Data Provider Wizard**.
* Sélectionnez **Microsoft SQL Server** comme **type de fournisseur**.
* Cliquez sur **Next** pour continuer.

#### **2. Spécifier les paramètres de connexion SQL Server**

* Entrez le **Nom du SQL Server** :
  * Tapez manuellement le **nom du serveur**.
  * Cliquez sur le **bouton déroulant** pour scanner et sélectionner un serveur disponible sur votre réseau.
* Dans la section **Logon to the Server**, choisissez l'une des méthodes d'authentification :
  * **Windows Authentication** *(utilise les identifiants de l'utilisateur actuel)*.
  * **SQL Server Authentication** *(nécessite un nom d'utilisateur et un mot de passe)*.
* Cliquez **Suivant** pour continuer.

#### **3. Sélectionnez la Base de données et la Source de données**

* Choisissez la **base de données** dans la liste déroulante.

* Sélectionnez la **source de données**:
  * **Tables** – Récupérer des données à partir d'une table spécifique.
  * **Views** – Utiliser une vue SQL Server prédéfinie.
  * **Stored Procedures** – Exécuter une procédure stockée.
  * **SQL Statement** – Saisir manuellement une requête SQL personnalisée.

* Si vous utilisez **SQL Statement**, assurez-vous qu'elle est optimisée pour les performances.

* Cliquez **Suivant** pour continuer.

#### **4. Terminez la configuration**

* **Vérifiez** les paramètres de connexion.
* Cliquez **Terminer** pour enregistrer le fournisseur de données.

***

### **Remarques importantes**

* **SQL Server Authentication**:
  * Si vous utilisez **Windows Authentication**, assurez-vous que le **PADS4 service** dispose des **autorisations appropriées**.
  * Si vous utilisez **SQL Server Authentication**, utilisez un **nom d'utilisateur et mot de passe sécurisés**.
* **Optimisation des requêtes**:
  * L'utilisation de **Views** ou **Stored Procedures** est recommandée pour l'efficacité.
  * Évitez les **requêtes SQL complexes** dans **SQL Statements** lorsque c'est possible.
* **Considérations de performance**:
  * **SQL Server est optimisé** pour traiter les requêtes en interne.
  * L'utilisation d'un **PADS4 Data Server** peut ne pas apporter d'avantages supplémentaires.

Cette configuration garantit que **PADS4** peut récupérer efficacement **des données SQL Server en temps réel**, ce qui est utile pour **les présentations dynamiques, les tableaux de bord et l'automatisation**.
