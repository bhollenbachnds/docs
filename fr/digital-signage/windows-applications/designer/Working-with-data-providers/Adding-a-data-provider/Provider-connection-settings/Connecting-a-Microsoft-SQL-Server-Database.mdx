Le **Microsoft SQL Server** fournisseur dans **PADS4** vous permet de vous connecter à une **base de données SQL Server**, de récupérer des données et de les utiliser dans votre présentation.

***

### **Étapes pour ajouter ou modifier une connexion Microsoft SQL Server**

#### **1. Démarrer l'assistant de fournisseur de données**

* Ouvrez **PADS4** et accédez à l'**Assistant de fournisseur de données**.
* Sélectionnez **Microsoft SQL Server** comme **type de fournisseur**.
* Cliquez sur **Suivant** pour continuer.

#### **2. Spécifier les paramètres de connexion SQL Server**

* Entrez le **nom du serveur SQL** :
  * Saisissez manuellement le **nom du serveur**.
  * Cliquez sur le **bouton déroulant** pour rechercher et sélectionner un serveur disponible sur votre réseau.
* Dans la section **Connexion au serveur**, choisissez l'une des méthodes d'authentification :
  * **Authentification Windows** *(utilise les informations d'identification de l'utilisateur actuel)*.
  * **Authentification SQL Server** *(nécessite un nom d'utilisateur et un mot de passe)*.
* Cliquez sur **Suivant** pour continuer.

#### **3. Sélectionner la base de données et la source de données**

* Choisissez la **base de données** dans la liste déroulante.

* Sélectionnez la **source de données** :
  * **Tables** – Récupérer des données d'une table spécifique.
  * **Vues** – Utiliser une vue SQL Server prédéfinie.
  * **Procédures stockées** – Exécuter une procédure stockée.
  * **Instruction SQL** – Saisir manuellement une requête SQL personnalisée.

* Si vous utilisez une **Instruction SQL**, assurez-vous qu'elle est optimisée pour les performances.

* Cliquez sur **Suivant** pour continuer.

#### **4. Finaliser la configuration**

* **Vérifiez** les paramètres de connexion.
* Cliquez sur **Terminer** pour enregistrer le fournisseur de données.

***

### **Remarques importantes**

* **Authentification SQL Server** :
  * Si vous utilisez l'**Authentification Windows**, assurez-vous que le **service PADS4** dispose des **autorisations appropriées**.
  * Si vous utilisez l'**Authentification SQL Server**, utilisez un **nom d'utilisateur et un mot de passe sécurisés**.
* **Optimisation des requêtes** :
  * L'utilisation de **Vues** ou de **Procédures stockées** est recommandée pour l'efficacité.
  * Évitez les **requêtes SQL complexes** dans les **Instructions SQL** dans la mesure du possible.
* **Considérations de performance** :
  * **SQL Server est optimisé** pour traiter les requêtes en interne.
  * L'utilisation d'un **PADS4 Data Server** peut ne pas apporter d'avantages supplémentaires.

Cette configuration garantit que **PADS4** peut récupérer efficacement les **données SQL Server en temps réel**, ce qui le rend utile pour les **présentations dynamiques, les tableaux de bord et l'automatisation**.
