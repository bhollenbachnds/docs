Le **fournisseur .NET framework** permet des **connexions haute performance** vers **MySQL, Oracle, SQL Server** et d'autres bases de données. Il réduit la surcharge tout en maintenant toutes les fonctionnalités.

***

## **Prérequis**

* Installez le **pilote .NET** pour votre base de données avant de continuer.
* Assurez-vous que votre base de données est **accessible** depuis le **PADS4 Data Server** ou les Viewers individuels.

***

## **Étapes pour connecter une base de données via .NET Framework**

### **1. Démarrer l'assistant de fournisseur de données**

* Suivez les étapes dans **Ajout d'un fournisseur de données** ou **Modification d'un fournisseur de données**.
* Choisissez **.NET Framework** comme type de fournisseur.

### **2. Configurer les paramètres de connexion .NET**

* Sélectionnez le **fournisseur de données .NET** dans la liste des fournisseurs installés.
  * Si votre fournisseur est manquant, assurez-vous que le bon **pilote .NET** est installé.
* Saisissez la **chaîne de connexion** pour vous connecter à votre base de données.
  * Référez-vous à la **documentation de votre fournisseur .NET** ou consultez **[www.connectionstrings.com](http://www.connectionstrings.com)** pour des exemples.
* Sécuriser la chaîne de connexion (Optionnel)
  * Utilisez **"Mot de passe optionnel"** pour masquer la chaîne de connexion lors de la modification.

**Exemple de chaîne de connexion pour MySQL :**

```
Server=192.168.1.10;Port=3306;Database=mysampledb;Uid=root;Pwd=mypassword;pooling=false;
```

### **3. Saisir une requête SQL pour récupérer les données**

* Définissez une **instruction SQL** spécifiant les données à récupérer.

**Exemple de requête pour sélectionner toutes les données :**

```sql
SELECT * FROM MyTable
```

* Personnalisez la requête en fonction des filtres, du tri ou des jointures.

### **4. Tester la connexion**

* Cliquez sur **Tester** pour vérifier la connexion.

### **5. Terminer l'assistant**

* Cliquez sur **Suivant** pour continuer.
* Terminez l'assistant comme décrit dans **Ajout d'un fournisseur de données**.

***

## **Considérations importantes**

* **Si vous utilisez PADS4 Data Server :**
  * Assurez-vous que le **pilote .NET est installé** sur le **serveur**.
  * La **base de données doit être accessible** depuis le **réseau du PADS4 Data Server**.

* **Si vous n'utilisez pas PADS4 Data Server :**
  * Installez le **pilote .NET** sur **tous les PADS4 Viewers** et les machines **PADS4 Designer**.
  * Vérifiez que **chaque Viewer** peut accéder à la **base de données** directement.

***

## **Dépannage**

* **La connexion à la base de données échoue ?**
  * Vérifiez les **paramètres du pare-feu** et les **identifiants de la base de données**.
  * Assurez-vous de l'**accès réseau** au **serveur de base de données**.
  * Vérifiez que le **pilote .NET** est **correctement installé**.

* **La requête ne renvoie aucune donnée ?**
  * Exécutez la **requête SQL manuellement** dans un outil de base de données pour vérifier les erreurs.
  * Vérifiez que les **noms de table et de colonne** sont corrects.

***

Votre base de données est maintenant connectée. Si vous avez besoin d'un filtrage avancé, d'un tri ou de mises à jour en temps réel, explorez les **options du fournisseur de données** dans PADS4.
