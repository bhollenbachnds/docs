**JSON (JavaScript Object Notation)** est un format léger largement utilisé pour **data exchange**. PADS4 prend en charge **legacy JSON feed integration**, permettant la récupération de données en temps réel pour les présentations d'affichage dynamique.

***

## **Étapes pour ajouter ou modifier une connexion JSON Feed**

### **1. Démarrer l'assistant Data Provider**

* Ouvrez le **Data Provider Wizard** en suivant les instructions dans **Adding a Data Provider** ou **Editing a Data Provider**.
* Sélectionnez **JSON Feed** comme type de fournisseur.

***

### **2. Spécifier l'URL du JSON Feed**

* Saisissez l'**JSON Feed URI** (URL) pointant vers la source de données.
* Cliquez sur **Preview** pour afficher un échantillon des données récupérées.

Cliquez sur **Next** pour continuer.

***

### **3. Choisir la méthode de requête**

* **GET (default)** → Récupère des données sans les modifier.
* **POST** → Utilisé pour envoyer des paramètres supplémentaires.
  * Si **POST** est sélectionné, saisissez le **request body** avec les paramètres requis.

***

### **4. Configurer le formatage des données**

* **Flatten Hierarchical Data** → Convertit les **nested JSON structures** en format de tableau plat.
* **Include Parent Levels Column Names** → Ajoute des **parent-level identifiers** pour les relations hiérarchiques.
* **Create Additional Columns Without HTML Tags** → Supprime le **HTML formatting** et extrait le **plain text**.

Cliquez sur **Next** pour continuer.

***

### **5. Configurer la sécurité et l'authentification**

* **Authorization (si nécessaire)**
  * Activez **Authorization Required**.
  * Saisissez **UserID, Password, and Domain**.

* **Proxy Server (si applicable)**
  * Configurez les paramètres du proxy si votre réseau nécessite une authentification.

* **User-Agent Identification**
  * Si nécessaire, spécifiez une **custom User-Agent string** (utile pour les restrictions d'accès API).

Cliquez sur **Next** pour continuer.

***

### **6. Finalisation de la configuration JSON Feed**

* Cliquez sur **Finish** pour terminer la configuration.

***

## **Considérations importantes**

* **PADS4 Data Server**
  * Si vous utilisez un **PADS4 Data Server**, assurez-vous que le **flux JSON est accessible depuis le serveur**.
  * Si vous n'utilisez pas de Data Server, assurez-vous que **PADS4 Designer (pour la prévisualisation) et les Viewers** peuvent accéder au flux JSON.

* **Gestion des données JSON hiérarchiques**
  * Si votre structure JSON comporte **plusieurs niveaux**, assurez-vous que **Flatten Hierarchical Data** est activé pour une sortie structurée.

En suivant ces étapes, **PADS4 peut récupérer et afficher efficacement des données en temps réel basées sur JSON** dans les présentations d'affichage dynamique.
