Le **Cognimatics feed provider** permet l'intégration avec les **Cognimatics devices** pour les **audience measurement data**, tels que le **people counting**, le **queue monitoring**, et l'**demographics analysis**.

***

## **Étapes pour ajouter ou modifier un Cognimatics Data Provider**

### **1. Démarrer l'Assistant Data Provider**

* Suivez les étapes dans **Adding a Data Provider** ou **Editing a Data Provider**.
* Choisissez **Cognimatics** comme type de fournisseur.

### **2. Configurer les paramètres de connexion Cognimatics**

* Entrez l'**HTTP Address (IP Address)** de l'**Cognimatics application**.
* Alternativement, spécifiez une **manual URL** avec tous les paramètres.

### **3. Sélectionner la source active**

* Choisissez la **installed and active source** dans la liste.
* Seule **one source** peut être active à la fois.
* La sélection d'une **inactive source** entraînera une **404 error** lors des tests.

### **4. Sélectionner le type de données : temps réel ou période**

#### **Real-Time Data**

* Affiche le **current people count** entrant ou sortant d'un lieu.

#### **Period-Based Data**

* Récupère les **statistical results** pour une période spécifique :

| Period Option | Description                                   |
| ------------- | --------------------------------------------- |
| **None**      | Aucune plage de temps n'est appliquée.        |
| **Today**     | Données pour **today** uniquement.            |
| **Yesterday** | Données pour **yesterday** uniquement.        |
| **Manual**    | Sélection personnalisée de la plage de dates. |

***

## **5. Champs de données renvoyés par le Cognimatics Feed**

### **People Counter**

| Field             | Type      | Description                                 |
| ----------------- | --------- | ------------------------------------------- |
| **Name**          | String    | Nom du lecteur.                             |
| **Serial**        | String    | Numéro de série du logiciel.                |
| **In**            | Integer   | Nombre de personnes qui sont entrées.       |
| **Out**           | Integer   | Nombre de personnes qui sont sorties.       |
| **People Inside** | Integer   | Différence entre les valeurs `In`In`Out`Out |
| **Timestamp**     | Date/Time | Heure d'enregistrement des données.         |

### **One-Way Flow Monitoring**

| Field                       | Type       | Description                             |
| --------------------------- | ---------- | --------------------------------------- |
| **Name**                    | String     | Nom du lecteur.                         |
| **Serial**                  | String     | Numéro de série du logiciel.            |
| **Entrée**                  | Entier     | Nombre de personnes qui sont entrées.   |
| **Sortie**                  | Entier     | Nombre de personnes qui sont sorties.   |
| **Personnes à l'intérieur** | Entier     | Différence entre `In` et `Out` valeurs. |
| **Horodatage**              | Date/Heure | Heure d'enregistrement des données.     |

### **Surveillance de file d'attente**

| Champ               | Type       | Description                                        |
| ------------------- | ---------- | -------------------------------------------------- |
| **Serial**          | Chaîne     | Numéro de série du logiciel.                       |
| **Name**            | Chaîne     | Nom de l'appareil.                                 |
| **Region 1 Name**   | Chaîne     | Nom de **Region 1**.                               |
| **Region 1 People** | Entier     | Personnes dans **Region 1**.                       |
| **People Inside**   | Entier     | Nombre total de personnes dans toutes les régions. |
| **Region 2 Name**   | Chaîne     | Nom de **Region 2**.                               |
| **Region 2 People** | Entier     | Personnes dans **Region 2**.                       |
| **Region 3 Name**   | Chaîne     | Nom de **Region 3**.                               |
| **Region 3 People** | Entier     | Personnes dans **Region 3**.                       |
| **Timestamp**       | Date/Heure | Heure d'enregistrement des données.                |

### **Surveillance d'occupation**

| Champ             | Type       | Description                             |
| ----------------- | ---------- | --------------------------------------- |
| **Name**          | Chaîne     | Nom du lecteur.                         |
| **Serial**        | Chaîne     | Numéro de série du logiciel.            |
| **In**            | Entier     | Nombre de personnes qui sont entrées.   |
| **Out**           | Entier     | Nombre de personnes qui sont sorties.   |
| **People Inside** | Entier     | Différence entre `In` et `Out` valeurs. |

### **Analyse démographique**

| ------------------- | ---------- | -------------------------------------------- |

## **6. Terminer la configuration**

* Cliquez sur **Next** pour continuer.

## **Considérations importantes**

* **Assurez-vous que la source correcte est sélectionnée :**
  * Si une **source inactive** est sélectionnée, le bouton **Preview** renverra une erreur **404**.

* **Vérifiez l'accès réseau :**
  * Si vous utilisez **PADS4 Data Server**, assurez-vous que le **Cognimatics feed** est **accessible** depuis le **réseau du serveur**.
  * Si vous **n'utilisez pas** de **Data Server**, chaque **PADS4 Viewer** doit avoir un **accès direct** au flux Cognimatics.

* **Consultez la documentation Cognimatics :**
  * Pour les **descriptions détaillées des champs** et les **configurations des appareils**, consultez la **documentation officielle de Cognimatics**.

***

## **Dépannage**

* **Aucune donnée n'apparaît ?**
  * Vérifiez que l'**adresse HTTP/IP** est correcte.
  * Assurez-vous que l'**appareil Cognimatics** est allumé et actif.
  * Vérifiez si la **source correcte** est sélectionnée.

* **Valeurs de données inattendues ?**
  * Confirmez que la **plage de temps sélectionnée** correspond aux données que vous attendez.
  * Si vous utilisez le **Mode Temps Réel**, assurez-vous que le flux de données est mis à jour.

***

En configurant un **flux de données Cognimatics** dans **PADS4**, les entreprises peuvent **analyser le flux de clients, les temps d'attente et les informations démographiques** en **temps réel**, optimisant ainsi leur **contenu d'affichage dynamique et leurs opérations**.
