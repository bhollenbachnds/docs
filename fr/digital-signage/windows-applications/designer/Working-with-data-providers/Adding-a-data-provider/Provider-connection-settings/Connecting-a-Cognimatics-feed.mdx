Le **fournisseur de flux Cognimatics** permet l'intégration avec **les appareils Cognimatics** pour **les données de mesure d'audience**, telles que **le comptage de personnes**, **la surveillance des files d'attente**, et **l'analyse démographique**.

***

## **Étapes pour ajouter ou modifier un fournisseur de données Cognimatics**

### **1. Démarrer l'assistant de fournisseur de données**

* Suivez les étapes dans **Ajout d'un fournisseur de données** ou **Modification d'un fournisseur de données**.
* Choisissez **Cognimatics** comme type de fournisseur.

### **2. Configurer les paramètres de connexion Cognimatics**

* Entrez l'**adresse HTTP (adresse IP)** de l'**application Cognimatics**.
* Vous pouvez également spécifier une **URL manuelle** avec tous les paramètres.

### **3. Sélectionner la source active**

* Choisissez la **source installée et active** dans la liste.
* Seulement **une source** peut être active à la fois.
* La sélection d'une **source inactive** entraînera une **erreur 404** lors du test.

### **4. Sélectionner le type de données : Temps réel ou Période**

#### **Données en temps réel**

* Affiche **le nombre de personnes actuel** entrant ou sortant d'un lieu.

#### **Données basées sur une période**

* Récupère **les résultats statistiques** pour une période spécifique :

| Option de période | Description                                |
| ----------------- | ------------------------------------------ |
| **Aucune**        | Aucune plage de temps n'est appliquée.     |
| **Aujourd'hui**   | Données pour **aujourd'hui** uniquement.   |
| **Hier**          | Données pour **hier** uniquement.          |
| **Manuel**        | Sélection de plage de dates personnalisée. |

## **Considérations importantes**

* **Assurez-vous que la source correcte est sélectionnée :**
  * Si une **source inactive** est sélectionnée, le bouton **Aperçu** renverra une **erreur 404**.

* **Vérifiez l'accès réseau :**
  * Si vous utilisez **PADS4 Data Server**, assurez-vous que le **flux Cognimatics** est **accessible** depuis le **réseau du serveur**.
  * Si vous n'utilisez **pas** de **Data Server**, chaque **PADS4 Viewer** doit avoir un **accès direct** au flux Cognimatics.

* **Consultez la documentation Cognimatics :**
  * Pour **des descriptions détaillées des champs** et **les configurations des appareils**, référez-vous à la **documentation officielle de Cognimatics**.

***

## **Dépannage**

* **Aucune donnée n'apparaît ?**
  * Vérifiez que l'**adresse HTTP/IP** est correcte.
  * Assurez-vous que l'**appareil Cognimatics** est allumé et actif.
  * Vérifiez si la **source correcte** est sélectionnée.

* **Valeurs de données inattendues ?**
  * Confirmez que la **plage de temps sélectionnée** correspond aux données attendues.
  * Si vous utilisez le **Mode temps réel**, assurez-vous que le flux de données est mis à jour.

***

En configurant un **flux de données Cognimatics** dans **PADS4**, les entreprises peuvent **analyser le flux de clients, les temps d'attente et les informations démographiques** en **temps réel**, optimisant ainsi leur **contenu d'affichage dynamique et leurs opérations**.
