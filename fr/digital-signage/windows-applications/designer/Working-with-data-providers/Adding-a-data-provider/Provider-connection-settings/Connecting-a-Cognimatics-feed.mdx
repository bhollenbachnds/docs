Le **Cognimatics feed provider** permet l'intégration avec les **Cognimatics devices** pour les **audience measurement data**, telles que le **people counting**, le **queue monitoring**, et l'**demographics analysis**.

***

## **Étapes pour ajouter ou modifier un Cognimatics Data Provider**

### **1. Démarrer l'assistant Data Provider**

* Suivez les étapes dans **Adding a Data Provider** ou **Editing a Data Provider**.
* Choisissez **Cognimatics** comme type de fournisseur.

### **2. Configurer les paramètres de connexion Cognimatics**

* Entrez l'**HTTP Address (IP Address)** de l'**Cognimatics application**.
* Alternativement, spécifiez une **manual URL** avec tous les paramètres.

### **3. Sélectionner la source active**

* Choisissez la **installed and active source** dans la liste.
* Seulement **one source** peut être active à la fois.
* La sélection d'une **inactive source** entraînera une **404 error** lors des tests.

### **4. Sélectionner le type de données : temps réel ou période**

#### **Real-Time Data**

* Affiche le **current people count** entrant ou sortant d'un lieu.

#### **Period-Based Data**

* Récupère les **statistical results** pour une période spécifique :

| Period Option | Description                                   |
| ------------- | --------------------------------------------- |
| **None**      | Aucune plage de temps n'est appliquée.        |
| **Today**     | Données pour **today** uniquement.            |
| **Yesterday** | Données pour **yesterday** uniquement.        |
| **Manual**    | Sélection personnalisée de la plage de dates. |

## **Considérations importantes**

* **Assurez-vous que la source correcte est sélectionnée :**
  * Si une **inactive source** est sélectionnée, le bouton **Preview** renverra une erreur **404 error**.

* **Vérifiez l'accès réseau :**
  * Si vous utilisez **PADS4 Data Server**, assurez-vous que le flux **Cognimatics feed** est **accessible** depuis le **server's network**.
  * Si vous **not** utilisez pas un **Data Server**, chaque **PADS4 Viewer** doit avoir un **direct access** au flux Cognimatics.

* **Consultez la documentation Cognimatics :**
  * Pour les **detailed field descriptions** et les **device configurations**, référez-vous à la **Cognimatics official documentation**.

***

## **Dépannage**

* **Aucune donnée n'apparaît ?**
  * Vérifiez que l'**HTTP Address/IP** est correcte.
  * Assurez-vous que l'**Cognimatics device** est allumé et actif.
  * Vérifiez si la **correct source** est sélectionnée.

* **Valeurs de données inattendues ?**
  * Confirmez que la **selected time range** correspond aux données que vous attendez.
  * Si vous utilisez le **Real-Time Mode**, assurez-vous que le flux de données est mis à jour.

***
