**Viewer Variables** permettent une personnalisation dynamique du contenu en remplaçant les espaces réservés par des valeurs en temps réel pour chaque **Viewer**. Ces variables peuvent être utilisées dans les **text fields, data providers, rules, filters, and other elements**, garantissant que le contenu affiché est spécifique à chaque Viewer sans nécessiter de présentations séparées.

***

## **Gestion des Viewer Variables**

### **System Management (Administrator Level)**

* Les variables de Viewer sont créées dans **PADS4 System Management**.
* Chaque variable a une **valeur par défaut** assignée.
* Ces variables s'appliquent à tous les Viewers sauf si elles sont remplacées.
* Les utilisateurs **ne peuvent pas ajouter ou supprimer des variables** via **PADS4 Agent**.

### **PADS4 Agent (Niveau Utilisateur)**

* Les variables de Viewer peuvent être **modifiées** via **PADS4 Agent (onglet Serveur)**.
* Les modifications effectuées ici **écrasent** les valeurs par défaut définies dans **System Management**.

### **Options du Viewer (Ajustement Local)**

* La **fenêtre Options du Viewer** dans l'application **PADS4 Viewer** permet des **ajustements locaux**.
* Ces modifications **remplacent à la fois les paramètres de System Management et de PADS4 Agent**.

***

## **Utilisation des variables de Viewer**

### **Comment les variables de Viewer sont identifiées**

* Une variable de Viewer est écrite comme `#VariableName#`.
* Example: `#Store Name#`.

### **Utilisation des variables de Viewer dans les éléments de texte**

* Example:
  * **System Management :** `#Store Name# = New York Health`
  * **Élément de texte :** `Welcome to #Store Name#!`
  * **Sortie affichée :** `Welcome to New York Health!`

Cette méthode permet **d'utiliser une présentation pour plusieurs emplacements** tout en ajustant dynamiquement le texte.

***

## **Où les variables de Viewer peuvent-elles être utilisées ?**

### **Champs basés sur du texte**

* Contenu textuel de playlist.
* Conditions de règles.
* Liaisons de propriétés.
* Connexions de fournisseurs de données.

### **Fournisseurs de données**

* Utilisées dans **le tri, le filtrage, les expressions, les références de fichiers, les paramètres, les requêtes SQL**.

### **Cas d'utilisation pour l'automatisation**

* **Filtrage par taille de magasin :**
  * Si un magasin a une **variable de Viewer pour la taille** (`L, XL, XXL`), seuls les produits pertinents pour cette taille sont affichés.
* **Contenu basé sur la localisation :**
  * Une **variable de Viewer basée sur le GPS** permet **d'afficher des cartes ou des publicités spécifiques à l'emplacement**.
* **Une seule présentation pour plusieurs Viewers :**
  * Au lieu de créer plusieurs présentations, **une seule présentation peut être utilisée avec des variables de Viewer** pour modifier **dynamiquement les noms de magasins, les adresses ou les promotions**.

***

## **Quand NE PAS utiliser les variables de Viewer**

* **Pour des mises à jour continues en temps réel** (par exemple, des cours boursiers qui changent rapidement).
* Dans ces cas, un **fournisseur de données** devrait être utilisé à la place.

***

## **Quand les variables de Viewer sont-elles mises à jour ?**

### **Dans la source de données**

* Lorsque le **Viewer récupère des données**, les **variables sont remplacées immédiatement**.
* Si le **fournisseur de données se met à jour**, les **nouvelles valeurs sont appliquées**.

### **Dans les fournisseurs de données**

* Si une **variable de Viewer** est utilisée dans **des filtres, des tris ou des requêtes**, elle est **remplacée au début de la récupération des données**.
* Si un fournisseur de données **se met à jour une fois par heure**, **les variables de Viewer ne se mettent à jour qu'après le prochain cycle de lecture**.

### **Dans les présentations**

* Lorsqu'une **présentation se charge**, **les variables de Viewer dans les éléments (texte, images, vidéos) sont remplacées**.

### **Dans les règles**

* **Les variables de Viewer dans les règles** sont remplacées **uniquement lorsque la règle est vérifiée**.
* **Les vérifications de règles ne se déclenchent pas automatiquement** si une variable de Viewer change.

### **Dans les liaisons de propriétés**

* Si un **fournisseur de données se met à jour à intervalle défini**, les **nouvelles valeurs des variables de Viewer remplacent les anciennes**.
* Cependant, modifier une variable de Viewer **ne force pas le fournisseur de données à se rafraîchir**.

### **Dans les playlists**

* Si la **playlist ne contient qu'un seul élément**, la **variable de Viewer n'est remplacée qu'une seule fois** (elle ne se met pas à jour).
* Si **l'alternance de playlist est activée**, la **variable de Viewer est remplacée par de nouvelles valeurs après chaque intervalle**.

***

## **Principaux avantages des variables de Viewer**

* **Une présentation peut être utilisée dans plusieurs emplacements**, réduisant la duplication.
* **Personnalisation automatisée du contenu**, comme les noms et emplacements des magasins.
* **Fonctionne parfaitement avec les fournisseurs de données, les filtres et les règles**.
* **Mises à jour efficaces sans reconcevoir les présentations**.
