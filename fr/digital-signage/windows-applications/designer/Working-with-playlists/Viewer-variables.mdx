**Viewer Variables**   activer la personnalisation dynamique du contenu en remplaçant les espaces réservés par des valeurs en temps réel pour chaque **Viewer**. Ces variables peuvent être utilisées dans **champs de texte, fournisseurs de données, règles, filtres et autres éléments**, garantissant que le contenu affiché est spécifique à chaque Viewer sans nécessiter de présentations séparées.

***

## **Gestion des Viewer Variables**

### **System Management (Niveau Administrateur)**

* Les Viewer Variables sont créées dans **PADS4 System Management**.
* Chaque variable a une **valeur par défaut** attribuée.
* Ces variables s'appliquent à tous les Viewers sauf si elles sont remplacées.
* Les utilisateurs **ne peuvent pas ajouter ni supprimer des variables** via **PADS4 Agent**.

### **PADS4 Agent (Niveau Utilisateur)**

* Les Viewer Variables peuvent être **modifiées** via **PADS4 Agent (onglet Serveur)**.
* Les modifications effectuées ici **écrasent** les valeurs par défaut définies dans **System Management**.

### **Viewer Options (Ajustement Local)**

* La **fenêtre Viewer Options** dans l'application **PADS4 Viewer** permet des **ajustements locaux**.
* Ces modifications **remplacent à la fois les paramètres de System Management et de PADS4 Agent**.

***

## **Utilisation des Viewer Variables**

### **Comment les Viewer Variables sont identifiées**

* Une Viewer Variable s'écrit comme `#VariableName#`.
* Example: `#Store Name#`.

### **Utilisation des Viewer Variables dans les éléments de texte**

* Example:
  * **System Management :** `#Store Name# = New York Health`
  * **Élément de texte :** `Welcome to #Store Name#!`
  * **Sortie affichée :** `Welcome to New York Health!`

Cette méthode permet **d'utiliser une seule présentation sur plusieurs emplacements** tout en ajustant dynamiquement le texte.

***

## **Où les Viewer Variables peuvent-elles être utilisées ?**

### **Champs basés sur le texte**

* Contenu textuel de la playlist.
* Conditions des règles.
* Liaisons de propriétés.
* Connexions aux fournisseurs de données.

### **Data Providers**

* Utilisé dans **le tri, le filtrage, les expressions, les références de fichiers, les paramètres, les requêtes SQL**.

### **Cas d'utilisation d'automatisation**

* **Filtrage par taille de magasin :**
  * Si un magasin a une **Viewer Variable pour la taille** (`L, XL, XXL`), seuls les produits pertinents pour cette taille sont affichés.
* **Contenu basé sur la localisation :**
  * Une **Viewer Variable basée sur le GPS** permet **d'afficher des cartes ou des publicités spécifiques à l'emplacement**.
* **Une seule présentation pour plusieurs Viewers :**
  * Au lieu de créer plusieurs présentations, **une seule présentation peut être utilisée avec des Viewer Variables** pour modifier **les noms de magasins, adresses ou promotions de manière dynamique**.

***

## **Quand NE PAS utiliser les Viewer Variables**

* **Pour les mises à jour continues en temps réel** (par exemple, des cours boursiers changeant rapidement).
* Dans ces cas, un **fournisseur de données** doit être utilisé à la place.

***

## **Quand les Viewer Variables sont-elles mises à jour ?**

### **Dans la source de données**

* Lorsque le **Viewer récupère des données**, les **variables sont remplacées immédiatement**.
* Si le **fournisseur de données se met à jour**, les **nouvelles valeurs sont appliquées**.

### **Dans les Data Providers**

* Si une **Viewer Variable** est utilisée dans **des filtres, du tri ou des requêtes**, elle est **remplacée au début de la récupération des données**.
* Si un fournisseur de données **se met à jour une fois par heure**, **les Viewer Variables ne se mettent à jour qu'après le prochain cycle de lecture**.

### **Dans les présentations**

* Lorsqu'une **présentation se charge**, **les Viewer Variables dans les éléments (texte, images, vidéos) sont remplacées**.

### **Dans les règles**

* **Les Viewer Variables dans les règles** sont remplacées **uniquement lorsque la règle est vérifiée**.
* **Les vérifications de règles ne se déclenchent pas automatiquement** si une Viewer Variable change.

### **Dans les liaisons de propriétés**

* Si un **fournisseur de données se met à jour à un intervalle défini**, les **nouvelles valeurs de Viewer Variable remplacent les anciennes**.
* Cependant, modifier une Viewer Variable **ne force pas le fournisseur de données à se rafraîchir**.

### **Dans les playlists**

* Si la **playlist ne contient qu'un seul élément**, la **Viewer Variable n'est remplacée qu'une seule fois** (elle ne se met pas à jour).
* Si **l'alternance de playlist est activée**, la **Viewer Variable est remplacée par de nouvelles valeurs après chaque intervalle**.

***

## **Principaux avantages des Viewer Variables**

* **Une seule présentation peut être utilisée sur plusieurs emplacements**, réduisant la duplication.
* **Personnalisation automatisée du contenu**, comme les noms et emplacements de magasins.
* **Fonctionne de manière transparente avec les fournisseurs de données, les filtres et les règles**.
* **Mises à jour efficaces sans reconcevoir les présentations**.
