L'**PADS4 Designer application** vous permet de **prévisualiser les présentations** pour voir exactement comment elles apparaîtront et se comporteront lorsqu'elles seront diffusées dans l'**PADS4 Viewer application**. Cela permet de vérifier la mise en page, le contenu, les connexions de données et l'interactivité avant la publication.

***

## **Comment démarrer une prévisualisation**

### **Prévisualiser la présentation entière**

1. **Cliquez** sur l'onglet **View** dans la barre de ruban.
2. Dans la **Preview section**, cliquez sur **Presentation**.
3. La **Presentation Preview window** s'ouvrira et **démarrera automatiquement la présentation**.

**Tip:**

* Démarrez rapidement une prévisualisation en appuyant sur **F2**.
* Appuyez sur **F12** pendant la prévisualisation pour le **web element debugging** (lors de l'utilisation du moteur de navigateur Chromium).

***

### **Prévisualiser une page spécifique**

1. Naviguez jusqu'à la page que vous souhaitez prévisualiser en utilisant la section **Go To** dans le menu **View**.
2. Suivez les étapes ci-dessus pour démarrer la prévisualisation de la présentation.
3. Alternativement, dans la **Presentation Preview window** :
   * Utilisez le **Page selector dropdown** pour choisir une page de départ.
   * Cliquez sur **Start** pour commencer la prévisualisation à partir de cette page.

***

## **Aperçu de la fenêtre de prévisualisation de présentation**

### **Groupe Presentation**

* **Close** : Ferme la fenêtre de prévisualisation.
* **Start** : Démarre la prévisualisation à partir de la page sélectionnée.
* **Stop** : Arrête la présentation en cours.
* **Page & Total** : Affiche le **current page number**, le **time on current page** et le **total running time**.

***

### **Groupe Zoom**

* **Stretch/Zoom Options** : Ajustez le niveau de zoom pour adapter la fenêtre de prévisualisation.\
  Note: Some content like **web elements** peuvent ne pas s'adapter parfaitement.

***

### **Groupe Options**

* **Viewer Name** : Entrez un nom de Viewer existant pour **simulate** comment ce Viewer spécifique exécuterait la présentation. Utile pour tester les **Viewer Variables** comme `#ViewerName#`.
* **Page** : Sélectionnez la page de départ pour la prévisualisation.
* **Viewer Type** : Choisissez un type de Viewer à **simulate** et supprimez les éléments **not supported** par ce type de Viewer.

***

### **Groupe Data Source**

(Ces options ne peuvent être ajustées que **before** de démarrer la prévisualisation)

* **Use Data Cache** :
  * Utilise les **cached data** pour une prévisualisation plus rapide.
  * Évite les appels chronophages aux fournisseurs de données.
  * Essentiel pour travailler hors ligne ou concevoir avec des sources de données lentes/distantes.
  * En savoir plus : **Data Caching in PADS4 Designer**.
* **Use Backup** :
  * Force PADS4 à utiliser les **backup Data Providers** (si définis), permettant de simuler des scénarios de repli.

***

### **Groupe Show**

* **Viewer Variables** : Ouvre une fenêtre listant **all Viewer Variables** pour cette session de prévisualisation. Voir la page **Viewer Variables** pour plus de détails.
* **Data Preview** : Affiche tous les **Data Providers** connectés et leurs **retrieved data** pour cette présentation.
* **Log Information** :
  * Affiche les journaux avec les **errors/warnings** pendant la prévisualisation.
  * Se met à jour automatiquement en temps réel.
  * Le bouton devient **red** si un problème est détecté.
  * Visualisez directement les 4 derniers messages du journal.

***

## **Remarques importantes**

* **Hidden pages** peuvent toujours être forcées à s'afficher pour la prévisualisation si elles sont choisies comme page de départ.
* Si vous utilisez des **web elements** avec **interactivity enabled**, appuyez sur **F12** pour ouvrir les **Chromium browser debug tools**.\
  (Remarque : Ceci n'est pas pris en charge pour le moteur Internet Explorer).

***
