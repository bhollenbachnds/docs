L'**application PADS4 Designer** vous permet de **prévisualiser les présentations** pour voir exactement comment elles apparaîtront et se comporteront lorsqu'elles seront lues dans l'**application PADS4 Viewer**. Cela permet de vérifier la mise en page, le contenu, les connexions de données et l'interactivité avant la publication.

***

## **Comment démarrer une prévisualisation**

### **Prévisualiser la présentation entière**

1. **Cliquez** sur l'onglet **View** dans la barre de ruban.
2. Dans la **section Preview**, cliquez sur **Presentation**.
3. La **fenêtre Presentation Preview** s'ouvrira et **démarrera automatiquement la présentation**.

**Tip:**

* Démarrez rapidement une prévisualisation en appuyant sur **F2**.
* Appuyez sur **F12** pendant la prévisualisation pour le **débogage des éléments web** (lors de l'utilisation du moteur de navigateur Chromium).

***

### **Prévisualiser une page spécifique**

1. Naviguez jusqu'à la page que vous souhaitez prévisualiser en utilisant la section **Go To** dans le menu **View**.
2. Suivez les étapes ci-dessus pour démarrer la prévisualisation de la présentation.
3. Alternativement, dans la **fenêtre Presentation Preview**:
   * Utilisez le **menu déroulant Page selector** pour choisir une page de départ.
   * Cliquez sur **Start** pour commencer la prévisualisation à partir de cette page.

***

## **Aperçu de la fenêtre de prévisualisation de présentation**

### **Groupe Presentation**

* **Close** : Ferme la fenêtre de prévisualisation.
* **Start** : Démarre la prévisualisation à partir de la page sélectionnée.
* **Stop** : Arrête la présentation en cours.
* **Page & Total** : Affiche le **numéro de page actuel**, le **temps sur la page actuelle** et le **temps d'exécution total**.

***

### **Groupe Zoom**

* **Options d'étirement/zoom** : Ajustez le niveau de zoom pour adapter la fenêtre de prévisualisation.\
  Note: Some content like **web elements** peuvent ne pas s'adapter parfaitement.

***

### **Groupe Options**

* **Viewer Name** : Entrez un nom de Viewer existant pour **simuler** comment ce Viewer spécifique exécuterait la présentation. Utile pour tester les **Viewer Variables** comme `#ViewerName#`.
* **Page** : Sélectionnez la page de départ pour la prévisualisation.
* **Viewer Type**: Choisissez un type de visionneuse pour **simuler** et supprimer les éléments **non pris en charge** par ce type de visionneuse.

***

### **Groupe de source de données**

(Ceux-ci ne peuvent être ajustés que **avant** de démarrer l'aperçu)

* **Utiliser le cache de données**:
  * Utilise **les données mises en cache** pour un aperçu plus rapide.
  * Évite les appels chronophages aux fournisseurs de données.
  * Essentiel pour travailler hors ligne ou concevoir avec des sources de données lentes/distantes.
  * En savoir plus : **Data Caching in PADS4 Designer**.
* **Utiliser la sauvegarde**:
  * Force PADS4 à utiliser **les fournisseurs de données de sauvegarde** (si définis), permettant de simuler des scénarios de repli.

***

### **Groupe d'affichage**

* **Variables de visionneuse**: Ouvre une fenêtre listant **toutes les variables de visionneuse** pour cette session d'aperçu. Voir la page **Viewer Variables** pour plus de détails.
* **Aperçu des données**: Affiche tous les **Data Providers** connectés et leurs **données récupérées** pour cette présentation.
* **Informations du journal**:
  * Affiche les journaux avec **erreurs/avertissements** pendant l'aperçu.
  * Se met à jour automatiquement en temps réel.
  * Le bouton devient **rouge** si un problème est détecté.
  * Visualisez directement les 4 derniers messages du journal.

***

## **Remarques importantes**

* **Les pages masquées** peuvent toujours être forcées à s'afficher pour l'aperçu si elles sont choisies comme page de démarrage.
* Si vous utilisez **des éléments web** avec **l'interactivité activée**, appuyez sur **F12** pour ouvrir les **outils de débogage du navigateur Chromium**.\
  (Remarque : Ceci n'est pas pris en charge pour le moteur Internet Explorer).

***
