<Info>
  **Prérequis**: Veuillez installer Node.js (version 19 ou supérieure) avant de continuer.<br />
  Veuillez mettre à niveau vers `docs.json` avant de continuer et supprimer le fichier obsolète `mint.json`.
</Info>

Suivez ces étapes pour installer et exécuter Mintlify sur votre système d'exploitation :

**Étape 1**: Installez Mintlify :

<CodeGroup>
  ```bash npm
  npm i -g mintlify
  ```

  ```bash yarn
  yarn global add mintlify
  ```
</CodeGroup>

**Étape 2**: Naviguez vers le répertoire docs (où se trouve le fichier `docs.json`) et exécutez la commande suivante :

```bash
mintlify dev
```

Un aperçu local de votre documentation sera disponible à `http://localhost:3000`.

### Ports personnalisés

Par défaut, Mintlify utilise le port 3000. Vous pouvez personnaliser le port sur lequel Mintlify s'exécute en utilisant l'option `--port`. Pour exécuter Mintlify sur le port 3333, par exemple, utilisez cette commande :

```bash
mintlify dev --port 3333
```

Si vous tentez d'exécuter Mintlify sur un port déjà utilisé, il utilisera le prochain port disponible :

```md
Port 3000 is already in use. Trying 3001 instead.
```

## Versions de Mintlify

Veuillez noter que chaque version de CLI est associée à une version spécifique de Mintlify. Si votre site web local ne correspond pas à la version de production, veuillez mettre à jour le CLI :

<CodeGroup>
  ```bash npm
  npm i -g mintlify@latest
  ```

  ```bash yarn
  yarn global upgrade mintlify
  ```
</CodeGroup>

## Validation des liens

Le CLI peut vous aider à valider les liens de référence dans votre documentation. Pour identifier les liens brisés, utilisez la commande suivante :

```bash
mintlify broken-links
```

## Formatage du code

Nous suggérons d'utiliser des extensions sur votre IDE pour reconnaître et formater MDX. Si vous utilisez VSCode, envisagez l'extension [MDX VSCode extension](https://marketplace.visualstudio.com/items?itemName=unifiedjs.vscode-mdx) pour la coloration syntaxique, et [Prettier](https://marketplace.visualstudio.com/items?itemName=esbenp.prettier-vscode) pour le formatage du code.

## Dépannage

<AccordionGroup>
  <Accordion title="Error: Could not load the &#x22;sharp&#x22; module using the darwin-arm64 runtime">
    Cela peut être dû à une version obsolète de node. Essayez ce qui suit :

    1. Supprimez la version actuellement installée de mintlify : `npm remove -g mintlify`
    2. Mettez à niveau vers Node v19 ou supérieur.
    3. Réinstallez mintlify : `npm install -g mintlify`
  </Accordion>

  <Accordion title="Issue: Encountering an unknown error">
    Solution: Go to the root of your device and delete the \~/.mintlify folder. Afterwards, run `mintlify dev` à nouveau.
  </Accordion>
</AccordionGroup>

Curieux de savoir ce qui a changé dans la version CLI ? [Consultez le changelog du CLI.](https://www.npmjs.com/package/mintlify?activeTab=versions)
