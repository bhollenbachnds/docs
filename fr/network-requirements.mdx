- title: Exigences réseau et pare-feu de PADS4
- description: Référence des ports TCP, UDP et HTTP utilisés par les composants
    serveur et client de PADS4, incluant le port par défaut 7237 et les règles
    de pare-feu entrant ou sortant requises.
---

Cette page explique l'utilisation des ports réseau par PADS4. Veuillez noter que les numéros de port mentionnés sont les paramètres par défaut. Certains ports peuvent être modifiés.

La communication réseau entre les composants du système PADS4 utilise TCP/IP. PADS4 utilise désormais un seul numéro de port pour communiquer et par défaut il s'agit du port 7237.

Si vous utilisez un pare-feu sur le serveur PADS4, cela signifie qu'un ou plusieurs des numéros de port ci-dessus devront être ouverts pour que PADS4 puisse communiquer sur le réseau.

Du côté client, il n'est pas nécessaire d'ouvrir des numéros de port dans le pare-feu.

## Ports du serveur PADS4 utilisés avec les connexions externes

| Port         | Protocole | Règles de pare-feu | Description                                                                                    |
| ------------ | --------- | ------------------ | ---------------------------------------------------------------------------------------------- |
| 7237         | TCP       | Entrant            | Port par défaut utilisé pour la communication PADS4                                            |
| 3702         | UDP       | Entrant            | Requis pour la fonction de recherche du serveur PADS4 (fonctionnalité de découverte du réseau) |
| 4525 ou 443  | HTTP      | Sortant            | Activation en ligne (un seul port doit être disponible)                                        |
| 80 et/ou 443 | HTTP      | Entrant / Sortant  | Mises à jour logicielles et réplication lors de l'utilisation de plusieurs serveurs            |
| 7            | UDP       | Sortant            | Fonction Wake On LAN (port WOL standard)                                                       |
| 9            | UDP       | Sortant            | Fonction Wake On LAN (port WOL standard)                                                       |
| 123          | UDP       | Entrant / Sortant  | Requis pour NTP (Network Time Protocol pour la synchronisation de l'heure du Viewer)           |

## Ports locaux utilisés par le serveur PADS4

*Notez que ces ports sont utilisés localement uniquement !*

| Port  | Protocole | Description                                                       |
| ----- | --------- | ----------------------------------------------------------------- |
| 15672 | HTTP      | utilisé par RabbitMQ pour la connexion à l'API HTTP               |
| 15671 | HTTP      | utilisé par RabbitMQ https pour la connexion à l'API HTTP via TLS |
| 5672  | TCP       | utilisé par RabbitMQ AMQP                                         |
| 5671  | TCP       | utilisé par RabbitMQ AMQP via TLS                                 |

## Ports du PADS4 Viewer

| Port | Protocole         | Règles de pare-feu                                                                   | Description                                                                                    |
| ---- | ----------------- | ------------------------------------------------------------------------------------ | ---------------------------------------------------------------------------------------------- |
| 7237 | TCP               | Sortant                                                                              | Port par défaut utilisé pour la communication PADS4                                            |
| 3702 | UDP               | Sortant                                                                              | Requis pour la fonction de recherche de serveur PADS4 (fonctionnalité de découverte du réseau) |
| 7    | UDP               | Entrant                                                                              | Fonction Wake On LAN (port WOL standard)                                                       |
| 9    | UDP               | Entrant                                                                              | Fonction Wake On LAN (port WOL standard)                                                       |
| 123  | Entrant / Sortant | Requis pour NTP (Network Time Protocol pour la synchronisation de l'heure du Viewer) |                                                                                                |

## Ports du PADS4 HTML5 Viewer

Port 80 et/ou 443 (HTTP)

### Remarque

<Note>
  PADS4 dispose de fonctionnalités intégrées pour détecter automatiquement un serveur PADS4 sur votre réseau. Cependant, un serveur PADS4 ne peut être détecté automatiquement que si le port UDP numéro 3702 n'est pas bloqué par un pare-feu sur le PC serveur ou ailleurs dans le réseau.
</Note>

<Note>
  Avec la version 2021.3, un nouveau mécanisme est introduit qui réplique dans les deux sens. Les modifications apportées sur le serveur Sub/Backup sont synchronisées avec le serveur principal, et les modifications du serveur principal sont synchronisées avec les serveurs Sub/Backup.
</Note>
