---
- title: Paramètres globaux
- description: Mintlify vous donne un contrôle total sur l'apparence de votre
    documentation grâce au fichier docs.json
- icon: gear
---

Chaque site Mintlify nécessite un`docs.json` fichier avec les paramètres de configuration de base. En savoir plus sur les[propriétés](#properties) ci-dessous.

## Propriétés

<ResponseField name="name" type="string" required>
  Nom de votre projet. Utilisé pour le titre global.

  Example: `mintlify`
</ResponseField>

<ResponseField name="navigation" type="Navigation[]" required>
  Un tableau de groupes contenant toutes les pages de ce groupe

  <Expandable title="Navigation">
    <ResponseField name="group" type="string">
      Le nom du groupe.

      Example: `Settings`
    </ResponseField>

    <ResponseField name="pages" type="string[]">
      Les chemins relatifs vers les fichiers markdown qui serviront de pages.

      Example: `["customization", "page"]`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="logo" type="string or object">
  Chemin vers l'image du logo ou objet avec le chemin vers les images du logo en mode "light" et "dark"

  <Expandable title="Logo">
    <ResponseField name="light" type="string">
      Chemin vers le logo en mode clair
    </ResponseField>

    <ResponseField name="dark" type="string">
      Chemin vers le logo en mode sombre
    </ResponseField>

    <ResponseField name="href" type="string" default="/">
      Vers où le clic sur le logo vous redirige
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="favicon" type="string">
  Chemin vers l'image du favicon
</ResponseField>

<ResponseField name="colors" type="Colors">
  Codes de couleur hexadécimaux pour votre thème global

  <Expandable title="Couleurs">
    <ResponseField name="primary" type="string" required>
      La couleur principale. Utilisée le plus souvent pour le contenu mis en évidence, les en-têtes de section, les accents, en mode clair
    </ResponseField>

    <ResponseField name="light" type="string">
      La couleur principale pour le mode sombre. Utilisée le plus souvent pour le contenu mis en évidence, les en-têtes de section, les accents, en mode sombre
    </ResponseField>

    <ResponseField name="dark" type="string">
      La couleur principale pour les boutons importants
    </ResponseField>

    <ResponseField name="background" type="object">
      La couleur de l'arrière-plan en mode clair et en mode sombre

      <Expandable title="Objet">
        <ResponseField name="light" type="string" required>
          Le code de couleur hexadécimal de l'arrière-plan en mode clair
        </ResponseField>

        <ResponseField name="dark" type="string" required>
          Le code de couleur hexadécimal de l'arrière-plan en mode sombre
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="topbarLinks" type="TopbarLink[]">
  Tableau de`name`s et`url`s de liens que vous souhaitez inclure dans la barre supérieure

  <Expandable title="TopbarLink">
    <ResponseField name="name" type="string">
      Le nom du bouton.

      Example: `Contact us`
    </ResponseField>

    <ResponseField name="url" type="string">
      L'URL une fois que vous cliquez sur le bouton. Exemple :`https://mintlify.com/docs`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="topbarCtaButton" type="Call to Action">
  <Expandable title="Appel à l'action de la barre supérieure">
    <ResponseField name="type" type={'"link" or "github"'} default="link">
      Link affiche un bouton. GitHub affiche les informations du dépôt à l'URL fournie, y compris le nombre d'étoiles GitHub.
    </ResponseField>

    <ResponseField name="url" type="string">
      Si`link`: Ce à quoi le bouton renvoie.

      Si`github`: Lien vers le dépôt à partir duquel charger les informations GitHub.
    </ResponseField>

    <ResponseField name="name" type="string">
      Texte à l'intérieur du bouton. Requis uniquement si`type` est un`link`.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="versions" type="string[]">
  Tableau de noms de versions. À utiliser uniquement si vous souhaitez afficher différentes versions de la documentation avec un menu déroulant dans la barre de navigation.
</ResponseField>

<ResponseField name="anchors" type="Anchor[]">
  Un tableau des ancres, incluant le`icon`,`color`, et`url`.

  <Expandable title="Ancre">
    <ResponseField name="icon" type="string">
      Le[Font Awesome](https://fontawesome.com/search?q=heart) icône utilisée pour mettre en avant l'ancre.

      Example: `comments`
    </ResponseField>

    <ResponseField name="name" type="string">
      Le nom de l'étiquette de l'ancre.

      Example: `Community`
    </ResponseField>

    <ResponseField name="url" type="string">
      Le début de l'URL qui détermine quelles pages appartiennent à l'ancre. En général, il s'agit du nom du dossier dans lequel vous placez vos pages.
    </ResponseField>

    <ResponseField name="color" type="string">
      La couleur hexadécimale de l'arrière-plan de l'icône d'ancre. Peut également être un dégradé si vous passez un objet avec les propriétés`from` et`to` qui sont chacune une couleur hexadécimale.
    </ResponseField>

    <ResponseField name="version" type="string">
      Utilisé si vous souhaitez masquer une ancre jusqu'à ce que la version correcte de la documentation soit sélectionnée.
    </ResponseField>

    <ResponseField name="isDefaultHidden" type="boolean" default="false">
      Passer`true` si vous souhaitez masquer l'ancre jusqu'à ce que vous redirigiez directement quelqu'un vers la documentation qu'elle contient.
    </ResponseField>

    <ResponseField name="iconType" default="duotone" type="string">
      L'une des valeurs suivantes : "brands", "duotone", "light", "sharp-solid", "solid", ou "thin"
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="topAnchor" type="Object">
  Remplacer les configurations par défaut pour l'ancre la plus haute.

  <Expandable title="Objet">
    <ResponseField name="name" default="Documentation" type="string">
      Le nom de l'ancre la plus haute
    </ResponseField>

    <ResponseField name="icon" default="book-open" type="string">
      Icône Font Awesome.
    </ResponseField>

    <ResponseField name="iconType" default="duotone" type="string">
      L'une des valeurs suivantes : "brands", "duotone", "light", "sharp-solid", "solid", ou "thin"
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="tabs" type="Tabs[]">
  Un tableau d'onglets de navigation.

  <Expandable title="Onglets">
    <ResponseField name="name" type="string">
      Le nom de l'étiquette de l'onglet.
    </ResponseField>

    <ResponseField name="url" type="string">
      Le début de l'URL qui détermine quelles pages appartiennent à l'onglet. En général, il s'agit du nom du dossier dans lequel vous placez vos pages.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="api" type="API">
  Configuration des paramètres API. En savoir plus sur les pages API sur[Composants API](https://mintlify.com/docs/api-playground/overview).

  <Expandable title="API">
    <ResponseField name="baseUrl" type="string">
      L'URL de base pour tous les endpoints API. Si`baseUrl` est un tableau, cela activera plusieurs options d'URL de base que l'utilisateur peut basculer.
    </ResponseField>

    <ResponseField name="auth" type="Auth">
      <Expandable title="Auth">
        <ResponseField name="method" type="&#x22;bearer&#x22; | &#x22;basic&#x22; | &#x22;key&#x22;">
          La stratégie d'authentification utilisée pour tous les endpoints API.
        </ResponseField>

        <ResponseField name="name" type="string">
          Le nom du paramètre d'authentification utilisé dans le playground API.

          Si la méthode est`basic`, le format doit être`[usernameName]:[passwordName]`
        </ResponseField>

        <ResponseField name="inputPrefix" type="string">
          La valeur par défaut conçue pour être un préfixe pour le champ de saisie d'authentification.

          Par ex. Si un`inputPrefix` de`AuthKey` hériterait du résultat de saisie par défaut du champ d'authentification comme`AuthKey`.
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="playground" type="Playground">
      Configurations pour le playground API

      <Expandable title="Playground">
        <ResponseField name="mode" default="show" type="&#x22;show&#x22; | &#x22;simple&#x22; | &#x22;hide&#x22;">
          Indique si le playground est affiché, masqué, ou affiche uniquement l'endpoint sans interactivité utilisateur supplémentaire`simple`

          En savoir plus sur le[guides du playground](https://mintlify.com/docs/api-playground/overview)
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="maintainOrder" type="boolean">
      L'activation de cet indicateur garantit que l'ordre des clés dans les pages OpenAPI correspond à l'ordre des clés défini dans le fichier OpenAPI.

      <Warning>Ce comportement sera bientôt activé par défaut, auquel cas ce champ sera déprécié.</Warning>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="openapi" type="string | string[]">
  Une chaîne ou un tableau de chaînes d'URL ou de chemins relatifs pointant vers votre fichier OpenAPI.

  Examples:

  <CodeGroup>
    ```json Absolute
    "openapi": "https://example.com/openapi.json"
    ```

    ```json Relative
    "openapi": "/openapi.json"
    ```

    ```json Multiple
    "openapi": ["https://example.com/openapi1.json", "/openapi2.json", "/openapi3.json"]
    ```
  </CodeGroup>
</ResponseField>

<ResponseField name="footerSocials" type="FooterSocials">
  Un objet de comptes de réseaux sociaux où la paire clé:propriété représente la plateforme de réseau social et l'URL du compte.

  Example:

  ```json
  {
    "x": "https://x.com/mintlify",
    "website": "https://mintlify.com"
  }
  ```

  <Expandable title="FooterSocials">
    <ResponseField name="[key]" type="string">
      L'une des valeurs suivantes`website`,`facebook`,`x`,`discord`,`slack`,`github`,`linkedin`,`instagram`,`hacker-news`

      Example: `x`
    </ResponseField>

    <ResponseField name="property" type="string">
      L'URL vers la plateforme sociale.

      Example: `https://x.com/mintlify`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="feedback" type="Feedback">
  Configurations pour activer les boutons de retour

  <Expandable title="Retour">
    <ResponseField name="suggestEdit" type="boolean" default="false">
      Active un bouton permettant aux utilisateurs de suggérer des modifications via des pull requests
    </ResponseField>

    <ResponseField name="raiseIssue" type="boolean" default="false">
      Active un bouton permettant aux utilisateurs de signaler un problème concernant la documentation
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="modeToggle" type="ModeToggle">
  Personnalisez le bouton de basculement du mode sombre.

  <Expandable title="ModeToggle">
    <ResponseField name="default" type={'"light" or "dark"'}>
      Définissez si vous souhaitez toujours afficher le mode clair ou sombre pour les nouveaux utilisateurs. Si non défini, nous utilisons par défaut le même mode que le système d'exploitation de l'utilisateur.
    </ResponseField>

    <ResponseField name="isHidden" type="boolean" default="false">
      Définissez sur true pour masquer le bouton de basculement mode sombre/clair. Vous pouvez combiner `isHidden` avec `default` pour forcer votre documentation à utiliser uniquement le mode clair ou sombre. Par exemple :

      <CodeGroup>
        ```json Only Dark Mode
        "modeToggle": {
          "default": "dark",
          "isHidden": true
        }
        ```

        ```json Only Light Mode
        "modeToggle": {
          "default": "light",
          "isHidden": true
        }
        ```
      </CodeGroup>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="backgroundImage" type="string">
  Une image d'arrière-plan à afficher derrière chaque page. Voir l'exemple avec
  [Infisical](https://infisical.com/docs) et [FRPC](https://frpc.io).
</ResponseField>
