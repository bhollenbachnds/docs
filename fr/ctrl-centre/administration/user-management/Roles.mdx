Dans notre dernière version, 2025.1, nous introduisons notre nouveau système de gestion des utilisateurs.

Ce changement fondamental déplace les paramètres de gestion des utilisateurs de l'application System Management vers le PADS4 Web Portal.

Avec ce changement, nous avons également introduit les Rôles pour affiner davantage la partie gestion des utilisateurs de PADS4.

Les Rôles vous permettront d'attribuer rapidement un certain ensemble de licences et de permissions à un utilisateur.

<Warning>
  Le nouveau système de gestion des utilisateurs n'est pas compatible avec Legacy CMS et Workspace dans PADS4 version 2025.1, cela a été ajouté dans PADS4 version 2025.2.
</Warning>

## Comment utiliser les rôles dans PADS4

<Info>
  Les licences System Management et License Management sont requises pour administrer vos utilisateurs et rôles
</Info>

* Vous pouvez y accéder en vous connectant à votre PADS4 Web Portal avec votre compte Administrateur. Allez dans le Dashboard, et en bas à gauche, accédez à **Users**.

<img src="/images/ctrl-centre/administration/user-management/Users.png" alt="Users Pn" style={{ width:"50%" }} />

* Sur le côté gauche, vous pouvez basculer entre les Users et les Roles

<img src="/images/ctrl-centre/administration/user-management/Users-overview.png" alt="Users Overview Pn" style={{ width:"51%" }} />

* Dans l'aperçu des utilisateurs, vous verrez les utilisateurs et leurs informations correspondantes telles que :
  * Name
  * Username
  * Roles
  * License

![Single User Summary Pn](/images/ctrl-centre/administration/user-management/single-user-summary.png)

* Dans la section Roles, vous obtiendrez l'aperçu suivant des rôles disponibles, y compris les rôles par défaut, qui sont expliqués ci-dessous.

![Default Roles](/images/ctrl-centre/administration/user-management/default-roles.png)

### Rechercher des utilisateurs

Utilisez la fonction de recherche pour localiser rapidement des utilisateurs spécifiques dans le système. Elle effectue des recherches dans les champs clés : Name, Username, Roles et License. C'est idéal pour les administrateurs gérant PADS4.

**Comment rechercher**

1. Saisissez des mots-clés dans la zone de recherche (par ex., "john.doe", "admin role" ou "Cms users group").
2. Les résultats se filtrent dynamiquement au fur et à mesure que vous tapez — pas besoin d'appuyer sur Entrée.

**Exporter les résultats**

Cliquez sur le **Export** bouton pour télécharger les résultats en Excel. Cela développe la vue pour une analyse en masse ou des rapports. Les données exportées incluent toutes les colonnes visibles pour une importation facile ailleurs.

## Rôles par défaut

<Note>
  Dans PADS4 version 2025.1, il y a cinq rôles par défaut ; depuis PADS4 version 2025.2, cinq rôles supplémentaires ont été ajoutés, qui sont listés plus loin dans l'article.
</Note>

### CMS Advanced users group

| Application  | Access            | Inheritance         |
| ------------ | ----------------- | ------------------- |
| Web Designer | Read/Write/Delete |                     |
| CMS          | Read              |                     |
| Content      | Read/Write/Delete | Content users group |

### CMS Manager group

| Application | Access            | Inheritance         |
| ----------- | ----------------- | ------------------- |
| CMS         | Read/Write/Delete |                     |
| Users       | Read/Write/Delete |                     |
| Content     | Read/Write/Delete | Content users group |

### CMS User group

| Application | Access            | Inheritance         |
| ----------- | ----------------- | ------------------- |
| CMS         | Read              |                     |
| Content     | Read/Write/Delete | Content users group |

### Content Managers group

| Application | Access                   | Inheritance |
| ----------- | ------------------------ | ----------- |
| Content     | Read/Write/Delete/Modify |             |

### Content User group

| Application | Access            | Inheritance |
| ----------- | ----------------- | ----------- |
| Content     | Read/Write/Delete |             |

<Note>
  Avec la sortie de PADS4 version 2025.2, les rôles par défaut ci-dessous ont été ajoutés.
</Note>

<Danger>
  Notez que les rôles de viewer nécessitent une licence Agent ou System Management (uniquement pour le statut en ligne/hors ligne / suppression des viewers) pour être efficaces en dehors de CMS2.0, car les rôles servent uniquement à définir des permissions une fois que l'accès a été accordé par la licence attribuée à l'utilisateur.

  Vous pouvez choisir d'ajuster les rôles de viewer par défaut pour ajouter automatiquement cette licence à l'utilisateur auquel ce rôle est attribué.
</Danger>

## Personnalisation des rôles

* Vous pouvez modifier ces rôles selon votre configuration préférée, ou vous pouvez créer vos propres Rôles dans lesquels vous pouvez spécifier d'autres rôles dont il est membre, ou attribuer certaines licences.

<img src="/images/ctrl-centre/administration/user-management/new-roles.png" alt="New Roles Pn" style={{ width:"65%" }} />

<img src="/images/ctrl-centre/administration/user-management/create-role.png" alt="Create Role Pn" style={{ width:"78%" }} />

* Avec ces rôles ou les rôles créés par vous-même, vous pouvez ensuite préciser davantage l'accès que vous accordez aux utilisateurs ayant un certain rôle aux applications disponibles dans le PADS4 Web Portal.

### Bibliothèque de contenu

* Faites un clic droit sur le dossier pour lequel vous souhaitez modifier/consulter l'accès et appuyez sur **"Edit access"**

<img src="/images/ctrl-centre/administration/user-management/content-access.png" alt="Content Access Pn" style={{ width:"71%" }} />

* Un aperçu des rôles et des utilisateurs avec leurs droits sur ce dossier particulier s'affichera et vous permettra d'ajouter ou de supprimer des rôles et des utilisateurs.

![Content Access 2 Pn](/images/ctrl-centre/administration/user-management/content-access-2.png)

### Catégories Web designer

* Cliquez sur les 3 points à droite de votre catégorie et cliquez sur **"Edit access"**

<img src="/images/ctrl-centre/administration/user-management/Webdesigner-category-access.png" alt="Webdesigner Category Access Pn" style={{ width:"57%" }} />

### Modèles Web designer

* Survolez un modèle avec votre souris et cliquez sur les trois points, puis cliquez sur **"Edit access"**

![Webdesigner Template Access Pn](/images/ctrl-centre/administration/user-management/Webdesigner-template-access.png)

### Viewers & Viewer groups

* Dans la vue Arborescence, faites un clic droit sur le groupe de visualiseurs et cliquez sur **"Modifier l'accès"**

<img src="/images/ctrl-centre/administration/user-management/treeview-perm.png" alt="Treeview Perm" style={{ width:"61%" }} />

* Dans la vue Détail, survolez le groupe de visualiseurs ou le visualiseur avec votre souris et appuyez sur les trois points pour "**Modifier l'accès"**.

<img src="/images/ctrl-centre/administration/user-management/detailviewer-perm.png" alt="Detailviewer Perm" style={{ width:"65%" }} />
