Dans notre dernière version, 2025.1, nous introduisons notre nouveau système de gestion des utilisateurs.

Ce changement fondamental déplace les paramètres de Gestion des Utilisateurs de l'application System Management vers le PADS4 Web Portal.

Avec ce changement, nous avons également introduit des Rôles pour affiner davantage la partie gestion des utilisateurs de PADS4.

Les Rôles vous permettront d'attribuer rapidement un certain ensemble de licences et de permissions à un utilisateur.

<Warning>
  Le nouveau système de gestion des utilisateurs n'est pas compatible avec Legacy CMS et Workspace dans PADS4 version 2025.1,**cela a été ajouté dans PADS4 version 2025.2.**
</Warning>

## Comment utiliser les rôles dans PADS4

<Info>
  Les licences System Management et License Management sont nécessaires pour administrer vos utilisateurs et rôles
</Info>

* Vous pouvez y accéder en vous connectant à votre PADS4 Web Portal avec votre compte Administrateur. Allez au Dashboard, et en bas à gauche, allez à **Users**.

![Users Pn](/images/ctrl-centre/administration/user-management/Users.png)

* Sur le côté gauche, vous pouvez basculer entre les Users et Roles

![Users Overview Pn](/images/ctrl-centre/administration/user-management/Users-overview.png)

* Dans l'aperçu des utilisateurs, vous verrez les utilisateurs et leurs informations correspondantes comme :
  * Name
  * Username
  * Roles
  * License

![Single User Summary Pn](/images/ctrl-centre/administration/user-management/single-user-summary.png)

* Dans la section Roles, vous obtiendrez l'aperçu suivant des rôles disponibles, y compris les rôles par défaut, qui sont expliqués ci-dessous.

![Default Roles](/images/ctrl-centre/administration/user-management/default-roles.png)

## Rôles par défaut

<Note>
  Dans PADS4 version 2025.1, il existe cinq rôles par défaut, depuis PADS4 version 2025.2, cinq rôles supplémentaires ont été ajoutés qui sont listés plus loin dans l'article.
</Note>

| Application  | Access            | Inheritance         |
| ------------ | ----------------- | ------------------- |
| Web Designer | Read/Write/Delete |                     |

| Content      | Read/Write/Delete | Content users group |

### CMS User group

| Application | Access            | Inheritance         |
| ----------- | ----------------- | ------------------- |
| CMS         | Read              |                     |
| Content     | Read/Write/Delete | Content users group |

### Content Managers group

| Application | Access                   | Inheritance |
| ----------- | ------------------------ | ----------- |
| Content     | Read/Write/Delete/Modify |             |

### Content User group

| Application | Access            | Inheritance |
| ----------- | ----------------- | ----------- |
| Content     | Read/Write/Delete |             |

<Note>
  Avec la sortie de PADS4 version 2025.2, les rôles par défaut ci-dessous ont été ajoutés.
</Note>

<Danger>
  Notez que les **viewer roles** nécessitent une **Agent** ou **System Management ( Only for on-offline status / removal of viewers )** licence pour être efficaces en dehors de CMS2.0, car les rôles servent uniquement à définir les permissions une fois que l'accès a été accordé par la licence attribuée à l'utilisateur.

  Vous pouvez choisir d'ajuster les rôles de viewer par défaut pour ajouter automatiquement cette licence à l'utilisateur auquel ce rôle est attribué.
</Danger>

## Personnalisation des rôles

* Vous pouvez modifier ces rôles selon votre configuration préférée, ou vous pouvez créer vos propres Rôles où vous pouvez spécifier d'autres rôles dont il est membre, ou attribuer certaines licences.

![New Roles Pn](/images/ctrl-centre/administration/user-management/new-roles.png)

![Create Role Pn](/images/ctrl-centre/administration/user-management/create-role.png)

* Avec ces rôles ou les rôles que vous avez créés, vous pouvez ensuite préciser davantage l'accès que vous accordez aux utilisateurs ayant un certain rôle aux applications disponibles dans le PADS4 Web Portal.

### Content library

* Faites un clic droit sur le dossier pour lequel vous souhaitez modifier/examiner l'accès et appuyez sur **"Edit access"**

![Content Access Pn](/images/ctrl-centre/administration/user-management/content-access.png)

* Un aperçu des rôles et des utilisateurs avec leurs droits sur ce dossier particulier s'affichera et vous permettra d'ajouter ou de supprimer des rôles et des utilisateurs.

![Content Access 2 Pn](/images/ctrl-centre/administration/user-management/content-access-2.png)

### Web designer category's

* Cliquez sur les 3 points à droite de votre catégorie et cliquez sur **"Edit access"**

![Webdesigner Category Access Pn](/images/ctrl-centre/administration/user-management/Webdesigner-category-access.png)

### Web designer templates

* Passez votre souris sur un modèle et cliquez sur les trois points, cliquez sur **"Edit access"**

![Webdesigner Template Access Pn](/images/ctrl-centre/administration/user-management/Webdesigner-template-access.png)

### Viewers & Viewer groups

* En vue arborescente, faites un clic droit sur le groupe de viewers et cliquez sur **"Edit Access"**

![Treeview Perm](/images/ctrl-centre/administration/user-management/treeview-perm.png)

* En vue détaillée, passez votre souris sur le groupe de viewers ou le viewer et appuyez sur les trois points pour "**Edit access"**.

![Detailviewer Perm](/images/ctrl-centre/administration/user-management/detailviewer-perm.png)
