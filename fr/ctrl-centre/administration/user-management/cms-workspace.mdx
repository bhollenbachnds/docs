<Warning>
  Depuis la version PADS4 2025.2, la gestion des informations personnelles des utilisateurs, des attributions de rôles et des attributions de licences s'effectue désormais dans le PADS4 Web Portal. Les informations spécifiques à CMS 1.0 ou Workspace, comme l'accès aux salles, sont toujours gérées dans les pages d'administration Crystal correspondantes.
</Warning>

## PADS4 Web Portal

* Une fois que vous avez attribué des licences (PADS4 CMS 1.0 User et/ou PADS4 Workspace User) à un utilisateur, celui-ci sera ajouté à la liste des utilisateurs CMS 1.0 et/ou Workspace. À partir de là, l'accès à des parties spécifiques de ces applications est géré dans les profils utilisateurs (plus d'informations à ce sujet dans Profils utilisateurs (Workspace & CMS 1.0).

![Cms Ws1](/images/ctrl-centre/administration/user-management/cms-ws1.png)

* Attribuez la licence "PADS4 Workspace User" pour donner à l'utilisateur l'accès à Workspace. Cette licence est suffisante en elle-même pour permettre l'accès à l'application Workspace. L'accès aux salles et les autres paramètres liés à Workspace sont toujours gérés via Workspace Administration.
* Attribuez la licence "**PADS4 CMS 1.0 User**" pour donner à l'utilisateur l'accès à Crystal CMS/CMS (legacy)/CMS 1.0 et/ou aux pages d'administration (pour CMS 1.0, Workspace ou les deux).

  <Note>
    Cette licence n'est PAS suffisante en elle-même pour permettre l'accès aux modules Crystal ou à ses pages d'administration. L'utilisateur a besoin d'un profil utilisateur qui autorise l'accès à certaines parties de la plateforme Crystal : CMS 1.0, Page d'administration CMS 1.0, Page d'administration Workspace, Wayfinding, etc.).
  </Note>
* Les utilisateurs disposant de licences/droits suffisants verront les icônes d'application correspondantes dans le sélecteur d'applications sur la page de connexion.

![Cms Ws2](/images/ctrl-centre/administration/user-management/cms-ws2.png)

* Ainsi que dans la barre latérale une fois que vous êtes sur la page d'accueil.

![Cms Ws3](/images/ctrl-centre/administration/user-management/cms-ws3.png)

## Informations principales sur les utilisateurs et création

* Auparavant, la création d'utilisateurs était effectuée séparément dans chaque application. Désormais, elle commence toujours par la création de l'utilisateur via PADS4 Ctrl Center. Ensuite, en attribuant à l'utilisateur une licence CMS 1.0 et/ou Workspace, il sera automatiquement visible dans la page d'administration CMS 1.0 et/ou la page d'administration Workspace (selon la licence).

<Note>
  Les informations principales telles que le nom d'utilisateur, le prénom, le nom de famille, le nfc et l'adresse e-mail sont désormais gérées uniquement dans PADS4. Il n'est plus possible de les mettre à jour via une page d'administration CMS 1.0.
</Note>

![Cms Ws4](/images/ctrl-centre/administration/user-management/cms-ws4.png)

* La création d'un utilisateur via une page d'administration Crystal reste cependant possible. Il vous suffit de sélectionner l'utilisateur PADS4. Il existe également une option pour attribuer automatiquement le profil utilisateur. Plus d'informations sur cette attribution de profil utilisateur sont expliquées plus loin dans le document.

![Cms Ws5](/images/ctrl-centre/administration/user-management/cms-ws5.png)

* En supposant que l'utilisateur actuel dispose d'un accès suffisant pour gérer les utilisateurs dans PADS4 Ctrl Center, vous pouvez cliquer sur le lien situé sous le menu déroulant des utilisateurs pour être redirigé vers la page de gestion des utilisateurs dans PADS4 Ctrl Center afin de gérer les (nouveaux) utilisateurs.

## Profils utilisateurs (CMS 1.0 & Workspace)

* Les profils utilisateurs CMS et Workspace – et donc la gestion des accès utilisateurs – sont toujours gérés dans la page d'administration correspondante, mais leur attribution est désormais gérée automatiquement par les paramètres PADS4 Role Mapping dans la page des plugins d'administration CMS 1.0. Cette attribution automatique peut être désactivée par utilisateur si souhaité.

![Cms Ws6](/images/ctrl-centre/administration/user-management/cms-ws6.png)

* Le profil attribué est prévisualisé sous la case à cocher « Auto assigned profile ». Le profil sélectionné est basé sur le(s) PADS4 Role(s) attribué(s) à l'utilisateur PADS4 sélectionné et les paramètres PADS4 Role Mapping que vous avez configurés. Lorsque cette case est décochée, vous êtes autorisé à mettre à jour le profil selon vos souhaits.

<Note>
  Veuillez noter que cet utilisateur ne mettra PAS à jour son profil en fonction des PADS4 Roles attribués à l'utilisateur PADS4 lié lorsque ce paramètre est désactivé. Vous pouvez mettre à jour le paramètre « Auto assigned profile » de l'utilisateur à tout moment.
</Note>

## PADS4 Role Mapping

* Ces paramètres se trouvent dans CMS 1.0 Administration -> Plugins -> PADS4 Role Mapping et constituent une part importante de la gestion correcte des utilisateurs. Ils déterminent quel profil utilisateur doit être attribué à chaque utilisateur :
* PADS4 Role Mapping attribue automatiquement un profil à un utilisateur lors de la synchronisation avec un utilisateur PADS4. Étant donné qu'un utilisateur PADS4 peut avoir plusieurs rôles, vous pouvez utiliser '-' comme valeur de profil dans un mapping. Cela indique à la synchronisation de collecter tous les mappings liés aux rôles de l'utilisateur et de sélectionner le profil en fonction de l'ordre défini dans le tableau ci-dessous (de haut en bas). Si aucun mapping ne correspond à l'un des rôles de l'utilisateur, le profil par défaut sera appliqué.

![Cms Ws7](/images/ctrl-centre/administration/user-management/cms-ws7.png)

* En supposant que vous disposez du tableau ci-dessus, cela donnera les profils suivants dans différentes situations d'attribution de PADS4 Role.

| **Lorsque l'utilisateur PADS4 a le(s) rôle(s)** | **Profil utilisateur CMS 1.0** | **Profil utilisateur Workspace**    |
| :---------------------------------------------- | :----------------------------- | :---------------------------------- |
| \<No Roles>                                     | Default                        | Default                             |
| CMS 1.0 administrators group  - Cms users group | Administration                 | Profile with all rooms access       |
| Cms managers group  - Viewer managers group     | Wayfinding profile             | Profile with restricted room access |
| Cms advanced user group                         | Digital Signage profile        | Default                             |
| Cms users group                                 | Default                        | Default                             |

Quelques points à noter dans cet exemple :

* Les profils Default sont utilisés comme solution de repli et peuvent être remplacés par un autre profil. Lorsque l'utilisateur PADS4 n'a pas de rôle attribué ou qu'aucun de ses rôles n'est mappé dans ce tableau, le profil par défaut sera utilisé.
* Le '-' signifie qu'aucun profil utilisateur CMS ou Workspace n'est mappé et que la recherche continuera parmi les autres rôles mappés. Cela signifie que le rôle mappé pourrait finir par utiliser le profil par défaut.
* Le mapping est facultatif, vous n'êtes pas obligé de mapper tous les PADS4 Roles et vous n'êtes pas obligé de mapper un profil à un PADS4 Role.
* Le tableau est ordonné. Cela signifie que lorsqu'un utilisateur PADS4 a plusieurs rôles et que plus d'un de ces rôles sont mappés, il prendra toujours le rôle le plus haut dans le tableau. Lorsque celui-ci est un '-', il continuera à chercher un rôle d'ordre inférieur dans la liste jusqu'à ce qu'il trouve un profil mappé ou le profil par défaut (de repli).

## CMS 1.0 Admin Plugins

* La plupart des informations utilisateurs sont désormais gérées dans PADS4. Cela inclut LDAP, User Provisioning/SCIM, SMTP et Single Sign-On. Ces paramètres sont désormais gérés par PADS4 et ont donc été déplacés de la page Plugins vers [PADS4 Identity providers.](https://docs.pads4.com/ctrl-centre/administration/credentials/active-directory)
