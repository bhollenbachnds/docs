- title: Microsoft Teams
---

Ce guide fournit des instructions étape par étape pour configurer **PADS4 Teams integration** dans **Microsoft Azure**. Il suppose une familiarité avec **Azure App Registrations** et **Azure Communication Services**.

***

## **1. Enregistrement d'application dans Azure**

Pour intégrer **Microsoft Teams** avec PADS4, créez un **nouvel enregistrement d'application** dans **Azure Active Directory (AAD)**.

### **Étape 1 : Créer un nouvel enregistrement d'application**

1. Ouvrir **Microsoft Azure Portal**.
2. Accéder à **Azure Active Directory** > **App Registrations**.
3. Cliquer sur **New Registration**.
4. Configurer les paramètres suivants :
   * **Name** : Tout nom de référence.
   * **Supported account types** :\
     Sélectionner **Single-tenant** ou **multi-tenant** selon vos besoins.\
     Les comptes Microsoft personnels ne sont **pas** pris en charge.
   * **Redirect URI** :
     * Sélectionner **Single Page Application (SPA)** comme type de plateforme.
     * Saisir l'URI de redirection :

       ```
       https://<host>:<port>/credentials/oauth2/authorize/callback
       ```

       * **Host** : Nom DNS ou **adresse IP** du **PADS4 CTRL Center**.
       * **Port** : Facultatif, à spécifier uniquement si PADS4 CTRL Center n'utilise pas les ports HTTP/HTTPS par défaut.

***

### **Étape 2 : Ajouter la revendication facultative Login Hint**

Cette revendication permet la **ré-authentification** dans PADS4 CTRL Center pour le **renouvellement manuel du jeton d'accès**.

1. Ouvrir **App Registration**.
2. Dans le menu de gauche, accéder à **Token Configuration**.
3. Cliquer sur **Add Optional Claim**.
4. Sélectionner **Token type: ID**.
5. Activer **Login\_hint**.
6. Cliquer sur **Add**.

***

### **Étape 3 : Créer une portée d'API**

Cette étape permet à **PADS4 Teams integration** de **rejoindre des réunions Teams** au nom des utilisateurs.

1. Ouvrir **App Registration**.
2. Dans le menu de gauche, sélectionner **Expose an API**.
3. Cliquer sur **Add a Scope**.
4. Saisir un **Application ID URI** :

   ```
   api://your-custom-uri
   ```

   * L'Application ID URI **doit commencer par** `api://` et **ne peut pas contenir d'espaces ni de caractères spéciaux**.
5. Cliquer sur **Save and Continue**.
6. Définir les **valeurs de portée** :
   * **Scope Name** : Tout nom unique sans espaces ni caractères spéciaux.
   * **Who can consent?** : Sélectionner **Admins and Users**.
   * **Admin Consent Display Name** : Tout nom de référence.
   * **User Consent Display Name** : Tout nom de référence.
   * **State** : **Enabled**.
7. Cliquer sur **Add Scope**.

L'**URI de portée complète** (Application ID URI + nom de la portée) sera requis dans **PADS4 CTRL Center**.

***

### **Étape 4 : Créer un secret client**

Le **Client Secret** garantit que PADS4 Teams integration peut **rejoindre des réunions Teams** sans que les utilisateurs aient à se connecter à chaque fois.

1. Ouvrir **App Registration**.
2. Dans le menu de gauche, sélectionner **Certificates & Secrets**.
3. Cliquer sur **New Client Secret**.
4. Saisir une **description** et sélectionner une **durée d'expiration**.
   * Veiller à créer un nouveau **Client Secret** avant son expiration.
5. Cliquer sur **Add**.
6. **Copier immédiatement la valeur du Client Secret généré**, car elle ne sera plus visible ultérieurement.

***

### **Étape 5 : Rassembler les informations requises**

L'intégration PADS4 Teams nécessite les informations suivantes :

* **Application (Client) ID** – Trouvé dans **App Registration Overview**.
* **Directory (Tenant) ID** – Trouvé dans **App Registration Overview**.
* **Client Secret** – Copié depuis l'étape 4.
* **Scope Name** – Trouvé dans **Expose an API**.

***

## **2. Configuration d'Azure Communication Services**

Pour activer **l'intégration PADS4 Teams**, configurez **Azure Communication Services**.

### **Étape 1 : Créer une ressource Communication Services**

1. Ouvrez **Microsoft Azure Portal**.
2. Accédez à **Azure Communication Services**.
3. Cliquez sur **Create**.
4. Configurez les éléments suivants :
   * **Subscription**: Sélectionnez votre abonnement Azure.
   * **Resource Group**: Choisissez un groupe existant ou créez-en un nouveau.
   * **Name**: Tout nom de référence.
   * **Geography**: Sélectionnez la région préférée.
5. Cliquez sur **Review + Create**, puis sur **Create**.

***

### **Étape 2 : Récupérer la chaîne de connexion**

L'intégration PADS4 Teams nécessite la **Primary Connection String**.

1. Ouvrez la **ressource Communication Services**.
2. Dans le menu de gauche, sélectionnez **Keys**.
3. Copiez la **Primary Key Connection String**.

***

## **3. Configuration des utilisateurs et licences**

Pour intégrer **Teams** avec **PADS4**, les comptes utilisateurs doivent disposer des **licences Microsoft 365 E3** suivantes :

* **Exchange Online (Plan 2)**
* **Microsoft Teams**

### **Attribution des licences**

1. Ouvrez **Microsoft 365 Admin Center**.
2. Accédez à **Users** > **Active Users**.
3. Sélectionnez le **compte utilisateur**.
4. Cliquez sur **Licenses & Apps**.
5. Assign:
   * **Exchange Online (Plan 2)**
   * **Microsoft Teams**
6. Cliquez sur **Save Changes**.

***

## **4. Récapitulatif des informations requises**

Pour configurer **PADS4 Teams Integration**, rassemblez les informations suivantes :

| **Valeur requise**                | **Où la trouver ?**           |
| --------------------------------- | ----------------------------- |
| **Application (Client) ID**       | App Registration Overview     |
| **Directory (Tenant) ID**         | App Registration Overview     |
| **Client Secret**                 | Certificates & Secrets        |
| **Scope Name**                    | Expose an API                 |
| **Primary Key Connection String** | Communication Services → Keys |

***

## **5. Prochaines étapes**

* Saisissez ces valeurs dans **PADS4 CTRL Center** pour activer **l'intégration Microsoft Teams**.
* Assurez-vous que les **licences sont attribuées** aux utilisateurs concernés.
* Testez la **connexion** en tentant de rejoindre une réunion Teams via PADS4.

La configuration est maintenant terminée et PADS4 est intégré à Microsoft Teams.
