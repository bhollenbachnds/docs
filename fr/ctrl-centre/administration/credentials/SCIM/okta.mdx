- title: SCIM pour Okta

<Warning>
  Cet article nécessite que vous disposiez au moins de la version PADS4 2025.2.
</Warning>

Le provisionnement des utilisateurs est le processus de création, de mise à jour et de suppression automatiques des comptes utilisateurs. Il garantit que les utilisateurs obtiennent les accès appropriés lorsqu'ils rejoignent l'organisation, changent de rôle ou la quittent.

Pour activer le provisionnement des utilisateurs, notre système utilise la norme SCIM qui fournit une méthode cohérente et automatisée pour créer, mettre à jour et désactiver les utilisateurs.

Pour configurer le provisionnement des utilisateurs, vous devrez créer un fournisseur d'identité en suivant les instructions de configuration SSO dans ce document, activer l'option SCIM sur celui-ci, puis configurer la configuration SCIM sur le fournisseur d'identité externe.

## 1. Activer SCIM sur le fournisseur d'identité PADS4

Lors de la configuration de votre fournisseur d'identité PADS4, vous avez la possibilité d'activer une option de provisionnement SCIM.

Pour l'activer, lors de la création du fournisseur d'identité :

* Cliquer sur l'étape "SCIM Provisioning"
* Cliquer sur "Enable SCIM Provisioning"

![Scim Ms1](/images/ctrl-centre/administration/credentials/SCIM/scim-ms1.png)

Une fois activé, veuillez conserver les données suivantes :

* URL SCIM
* Jeton d'accès SCIM

Ces valeurs seront réutilisées ultérieurement lors de la configuration du provisionnement SCIM côté fournisseur d'identité externe.

Sur Okta, la fonctionnalité de provisionnement des utilisateurs peut être activée sur la même application que celle précédemment créée pour le SSO.

#### Créer la configuration de provisionnement

* Ouvrir l'application précédemment créée pour la configuration SSO

![Scim Okta1](/images/ctrl-centre/administration/credentials/SCIM/scim-okta1.png)

* Cliquer sur l'onglet "General", puis sur le bouton "Edit" dans le panneau "App settings"
* Cocher l'option "SCIM" dans le champ "Provisioning"
* Cliquer sur "Save"

![Scim Okta2](/images/ctrl-centre/administration/credentials/SCIM/scim-okta2.png)

Une fois l'option enregistrée, un nouvel onglet "**Provisioning**" devrait être disponible en haut.

* Cliquer sur l'onglet "**Provisioning**"
* Cliquer sur le bouton "**Edit**" et saisir les informations suivantes :
  * SCIM Connector URL – Ici, vous devrez renseigner l'URL fournie par le fournisseur d'identité PADS4 en tant que "SCIM URL" (ex. [https://pads4.mycompany.com](https://pads4.mycompany.com) /rdx/[nds.services](http://nds.services).user.scim/api/v1/scim)
  * Unique identifier field for users – Cela définira l'identifiant unique qui servira de référence entre Okta et PADS4. **La valeur par défaut est "username"**
  * Supported provisioning actions – Cliquer sur les actions suivantes
    * Push new users
    * Push profile updates
    * Push groups
  * Authentication Mode – Sélectionner "**HTTP Header**"
  * HTTP Header
    * Authorization – Ici, vous devrez renseigner le jeton fourni par le fournisseur d'identité PADS4 en tant que "SCIM Access token"

<Warning>
  Attention: This SCIM Connector URL must be publicly accessible because Okta uses it as a callback for user provisioning.
</Warning>

* Cliquer ensuite sur "Test Connector Configuration" pour vérifier que la configuration est correcte

![Scim Okta3](/images/ctrl-centre/administration/credentials/SCIM/scim-okta3.png)

* Si le test est concluant, cliquer sur le bouton "**Save**" pour appliquer cette configuration

![Scim Okta4](/images/ctrl-centre/administration/credentials/SCIM/scim-okta4.png)

Une fois la configuration de l'intégration enregistrée, de nouveaux onglets latéraux gauches devraient être disponibles

* Cliquer sur l'onglet "**To App**" et cliquer sur le bouton "Edit" dans la section "**Provisioning to App**"
* Cocher les cases "**Enable**" pour les sections suivantes :
  * Create users
  * Update user attributes
  * Deactivate users
* Cliquer sur le bouton "**Save**" pour appliquer les modifications

![Scim Okta5](/images/ctrl-centre/administration/credentials/SCIM/scim-okta5.png)

#### Configurer le mappage des utilisateurs et des groupes

Le mappage des utilisateurs et des groupes est déjà configuré par défaut lors de la création de l'application, mais vous pouvez simplement vérifier que vous disposez au moins des attributs suivants mappés sur votre application :

![Scim Okta6](/images/ctrl-centre/administration/credentials/SCIM/scim-okta6.png)

* **(Optionnel)** Ajouter des attributs optionnels

Vous pouvez également ajouter des attributs optionnels définis par PADS4

Pour les utiliser, vous devrez cliquer à nouveau sur "Add New Mapping" pour chacun de ces attributs et les configurer.

Actuellement, les attributs optionnels disponibles sont :

| **Attribut cible** | **Commentaire**                                                                                                      |
| :----------------- | :------------------------------------------------------------------------------------------------------------------- |
| nfc                | Cette propriété sera utilisée pour synchroniser toute valeur renseignée dans la propriété NFC de l'utilisateur PADS4 |

#### Attribuer le groupe autorisé à accéder à l'application

Sur Okta, vous devrez définir quels membres du groupe seront synchronisés avec l'application.

Cela est configuré de la même manière que la configuration de l'authentification unique, et les groupes d'utilisateurs déjà définis pour le SSO seront déjà appliqués à cette configuration.

Si vous avez besoin d'étendre cette liste de groupes d'utilisateurs, veuillez suivre ces instructions dans l'article SSO correspondant à la partie : **Attribuer des groupes d'utilisateurs à l'application**
