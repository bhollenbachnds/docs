<Warning>
  Cet article nécessite que vous ayez au moins la version 2025.2 de PADS4.
</Warning>

Le provisionnement des utilisateurs est le processus de création, de mise à jour et de suppression automatiques des comptes utilisateurs. Il garantit que les utilisateurs obtiennent les accès appropriés lorsqu'ils rejoignent l'organisation, changent de rôle ou la quittent.

Pour activer le provisionnement des utilisateurs, notre système utilise la norme SCIM qui fournit un moyen cohérent et automatisé de créer, mettre à jour et désactiver les utilisateurs.

Pour configurer le provisionnement des utilisateurs, vous devrez créer un fournisseur d'identité en suivant les instructions de configuration SSO dans ce document, activer l'option SCIM et configurer la configuration SCIM sur le fournisseur d'identité externe.

## 1. Activer SCIM sur le fournisseur d'identité PADS4

Lors de la configuration de votre fournisseur d'identité PADS4, vous avez la possibilité d'activer une option de provisionnement SCIM.

Pour l'activer, lors de la création du fournisseur d'identité :

* Cliquez sur l'étape "Provisionnement SCIM"
* Cliquez sur "Activer le provisionnement SCIM"

![Scim Ms1](/images/ctrl-centre/administration/credentials/SCIM/scim-ms1.png)

Une fois activé, veuillez conserver les données suivantes :

* URL SCIM
* Jeton d'accès SCIM

Ces valeurs seront réutilisées ultérieurement lors de la configuration du provisionnement SCIM du côté du fournisseur d'identité externe.

Sur Okta, la fonctionnalité de provisionnement des utilisateurs peut être activée sur la même application que vous avez précédemment créée pour le SSO.

#### Créer la configuration de provisionnement

* Ouvrez l'application que vous avez créée précédemment pour la configuration SSO

![Scim Okta1](/images/ctrl-centre/administration/credentials/SCIM/scim-okta1.png)

* Cliquez sur l'onglet "Général" puis cliquez sur le bouton "Modifier" du panneau "Paramètres de l'application"
* Cochez l'option "SCIM" dans le champ "Provisionnement"
* Cliquez sur "Enregistrer"

![Scim Okta2](/images/ctrl-centre/administration/credentials/SCIM/scim-okta2.png)

Une fois l'option enregistrée, un nouvel onglet "**Provisionnement**" devrait être disponible en haut.

* Cliquez sur l'onglet "**Provisionnement**"
* Cliquez sur le bouton "**Modifier**" et saisissez les informations suivantes :
  * URL du connecteur SCIM – Ici, vous devrez remplir l'URL qui a été donnée sur le fournisseur d'identité PADS4 comme "URL SCIM" (par exemple [https://pads4.mycompany.com](https://pads4.mycompany.com) /rdx/[nds.services](http://nds.services).user.scim/api/v1/scim)
  * Champ d'identifiant unique pour les utilisateurs – Cela définira l'identifiant unique qui sera la référence entre Okta et PADS4. **Par défaut, c'est "username"**
  * Actions de provisionnement prises en charge – Cliquez sur les actions suivantes
    * Pousser de nouveaux utilisateurs
    * Pousser les mises à jour de profil
    * Pousser les groupes
  * Mode d'authentification – Sélectionnez "**En-tête HTTP**"
  * En-tête HTTP
    * Autorisation – Ici, vous devrez remplir le jeton qui a été donné sur le fournisseur d'identité PADS4 comme "Jeton d'accès SCIM"

<Warning>
  Attention: This SCIM Connector URL must be publicly accessible because Okta uses it as a callback for user provisioning.
</Warning>

* Cliquez ensuite sur "Tester la configuration du connecteur" pour vous assurer que la configuration est correcte

![Scim Okta3](/images/ctrl-centre/administration/credentials/SCIM/scim-okta3.png)

* Si le test est réussi, cliquez sur le bouton "**Enregistrer**" pour appliquer cette configuration

![Scim Okta4](/images/ctrl-centre/administration/credentials/SCIM/scim-okta4.png)

Une fois que vous avez enregistré la configuration d'intégration, de nouveaux onglets de gauche devraient être disponibles

* Cliquez sur l'onglet "**Vers l'application**" et cliquez sur le bouton "Modifier" dans la section "**Provisionnement vers l'application**"
* Cochez les cases "**Activer**" pour les sections suivantes :
  * Créer des utilisateurs
  * Mettre à jour les attributs utilisateur
  * Désactiver les utilisateurs
* Cliquez sur le bouton "**Enregistrer**" pour appliquer les modifications

![Scim Okta5](/images/ctrl-centre/administration/credentials/SCIM/scim-okta5.png)

#### Configurer le mappage des utilisateurs et des groupes

Le mappage des utilisateurs et des groupes est déjà configuré par défaut lors de la création de l'application, mais vous pouvez vous assurer que vous avez au moins les attributs suivants mappés sur votre application :

![Scim Okta6](/images/ctrl-centre/administration/credentials/SCIM/scim-okta6.png)

* **(Facultatif)** Ajouter des attributs optionnels

Vous pouvez également ajouter des attributs optionnels définis par PADS4

Pour les utiliser, vous devrez cliquer à nouveau sur "Ajouter un nouveau mappage" pour chacun de ces attributs et les configurer.

Actuellement, les attributs optionnels disponibles sont :

| **Attribut cible** | **Commentaire**                                                                                                   |
| :----------------- | :---------------------------------------------------------------------------------------------------------------- |
| nfc                | Cette propriété sera utilisée pour synchroniser toute valeur remplie dans la propriété NFC de l'utilisateur PADS4 |

#### Attribuer un groupe autorisé pour accéder à l'application

Sur Okta, vous devrez définir quels membres du groupe seront synchronisés avec l'application.

Ceci est configuré de la même manière que la configuration Single Sign-on et les groupes d'utilisateurs déjà définis pour le SSO seront déjà appliqués à cette configuration.

Si vous avez besoin d'étendre cette liste de groupes d'utilisateurs, veuillez suivre ces instructions dans l'article SSO respectif à la partie : **Attribuer des groupes d'utilisateurs à l'application**
