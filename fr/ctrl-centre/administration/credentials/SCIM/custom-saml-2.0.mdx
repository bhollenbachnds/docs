---
- title: SCIM pour Custom SAML 2.0
---

<Warning>
  Cet article nécessite d'avoir au moins la version PADS4 2025.2.
</Warning>

Pour tout fournisseur d'identité prenant en charge la fonctionnalité de provisionnement des utilisateurs avec le standard SCIM, vous pouvez appliquer la configuration suivante :

* SCIM URL – Ici, vous devrez renseigner l'URL fournie dans le fournisseur d'identité PADS4 en tant que "SCIM URL" (ex. [https://pads4.mycompany.com](https://pads4.mycompany.com) /rdx/[nds.services](http://nds.services).user.scim/api/v1/scim)
* SCIM Token – L'authentification SCIM utilisée sur PADS4 repose sur une authentification par token Bearer, donc ici vous devrez renseigner le token fourni dans le fournisseur d'identité PADS4 en tant que "SCIM Access token"

<Warning>
  Attention: This SCIM URL must be publicly accessible because the identity provider uses it as a callback for user provisioning.
</Warning>

Veuillez vous assurer que votre configuration SCIM envoie des notifications push pour toute création/mise à jour/suppression d'utilisateurs, et de même pour les groupes d'utilisateurs.

Concernant le mappage des propriétés des utilisateurs et des groupes, veuillez renseigner au moins ces propriétés requises :

* userName
* familyName
* givenName
* preferredLanguage
* emails
