---
- title: Custom SAML 2.0
---

<Warning>
  Cet article nécessite que vous ayez au moins PADS4 version 2025.2.
</Warning>

Pour tout fournisseur d'identité qui prend en charge la fonctionnalité de provisionnement des utilisateurs avec la norme SCIM, vous pouvez appliquer la configuration suivante :

* SCIM URL – Ici, vous devrez remplir l'URL qui a été donnée sur le fournisseur d'identité PADS4 comme "SCIM URL" (par exemple [https://pads4.mycompany.com](https://pads4.mycompany.com) /rdx/[nds.services](http://nds.services).user.scim/api/v1/scim)
* SCIM Token – L'authentification SCIM utilisée sur PADS4 utilise une authentification par jeton Bearer, vous devrez donc saisir ici le jeton qui a été fourni sur le fournisseur d'identité PADS4 comme "SCIM Access token"

<Warning>
  Attention: This SCIM URL must be publicly accessible because the identity provider uses it as a callback for user provisioning.
</Warning>

Veuillez vous assurer que votre configuration SCIM envoie des notifications pour toute création/mise à jour/suppression d'utilisateurs, ainsi que pour les groupes d'utilisateurs.

Ensuite, concernant le mappage des propriétés utilisateur et groupe, veuillez remplir au moins ces propriétés requises :

* userName
* familyName
* givenName
* preferredLanguage
* emails
