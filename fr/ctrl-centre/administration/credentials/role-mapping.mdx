---
- title: Role Mapping
---

<Warning>
  Cet article nécessite que vous disposiez au moins de PADS4 version 2025.2.
</Warning>

Une fois votre fournisseur d'identité créé, vous devrez maintenant gérer les mappages de rôles du fournisseur d'identité.

Lorsque l'utilisateur se connecte, il sera assigné à des rôles pads4 en fonction des mappages de groupes d'utilisateurs du fournisseur d'identité vers les rôles pads4 que vous allez configurer.

## 1. Configurer les mappages

Pour configurer ces mappages :

* Sur votre fournisseur d'identité nouvellement créé, cliquez sur "Identity mappings"

![Role Mapping1](/images/ctrl-centre/administration/credentials/role-mapping1.png)

* Sur l'onglet "Role mappings" actuel, vous cliquerez sur "New" pour créer une nouvelle relation de mappage de rôle

![Role Mapping2](/images/ctrl-centre/administration/credentials/role-mapping2.png)

* Sur le formulaire de mappage de rôle, vous devrez renseigner deux informations
  * Le groupe du fournisseur SSO – qui correspond à la revendication de groupe d'utilisateurs que vous avez configurée sur le fournisseur d'identité externe
  * Les rôles PADS4 – qui déterminent les rôles attribués à un utilisateur en fonction du groupe du fournisseur d'identité externe auquel il appartient

![Role Mapping3](/images/ctrl-centre/administration/credentials/role-mapping3.png)

* Sélectionnez un fournisseur SSO pour votre mappage de rôle
  * Si le groupe du fournisseur SSO que vous attendez n'est pas encore créé, vous pouvez cliquer sur le bouton "New SSO Group" pour ajouter un autre groupe dans la liste

![Role Mapping4](/images/ctrl-centre/administration/credentials/role-mapping4.png)

* Le nom du groupe SSO est uniquement utilisé ici pour mémoriser l'identité du groupe que vous avez créé
* L'identifiant du groupe SSO doit correspondre à la valeur de revendication de groupe que vous avez définie dans la configuration des revendications du fournisseur d'identité externe
  * *Par défaut, la valeur de revendication de groupe est :*
    * *L'identifiant du groupe d'utilisateurs pour Azure*
    * *Le nom du groupe d'utilisateurs pour Okta*

Une fois ce groupe SSO renseigné, cliquez sur "Create" pour enregistrer le groupe et sélectionnez-le dans votre formulaire de création de mappage de rôle

![Role Mapping5](/images/ctrl-centre/administration/credentials/role-mapping5.png)

* Sélectionnez des rôles sur PADS4

Ici, vous devrez sélectionner les rôles qui seront attribués à l'utilisateur s'il est membre du groupe SSO fourni

Si un rôle est manquant, vous pouvez cliquer sur le bouton "New role" pour créer un nouveau rôle et le sélectionner immédiatement après.

![Role Mapping6](/images/ctrl-centre/administration/credentials/role-mapping6.png)

* Cliquez ensuite sur le bouton "Create" pour enregistrer le mappage de rôle.
* Ce processus doit être répété autant de fois que vous souhaitez spécifier différents groupes d'utilisateurs du fournisseur d'identité externe que vous souhaitez autoriser à accéder à l'application.

<Warning>
  Si un utilisateur de votre fournisseur d'identité externe est membre d'un groupe inconnu de la table de mappage de rôle ci-dessous, cet utilisateur n'aura aucun droit sur l'application par défaut
</Warning>

## 2. Mettre à jour les revendications du fournisseur d'identité (optionnel)

Si vous créez votre fournisseur d'identité à partir d'un fournisseur d'identité externe SAML 2.0 personnalisé ou si vous souhaitez étendre ou mettre à jour vos attributs de revendications, vous pourriez vouloir modifier le mappage des revendications sur le fournisseur d'identité PADS4 que vous avez créé.

Pour mettre à jour ces revendications, vous devrez :

* Sur votre fournisseur d'identité nouvellement créé, cliquez sur "Identity mappings"

![Role Mapping7](/images/ctrl-centre/administration/credentials/role-mapping7.png)

* Puis passez à l'onglet "Claims".

<Note>
  Si vous avez créé votre fournisseur d'identité à partir d'Azure ou d'Okta, vous pourrez trouver ici des mappages de revendications par défaut que vous pouvez modifier si nécessaire
</Note>

![Role Mapping8](/images/ctrl-centre/administration/credentials/role-mapping8.png)

* Ici, vous pouvez soit créer une nouvelle revendication, soit en mettre une à jour. Si vous créez une nouvelle revendication, vous devrez renseigner les informations suivantes
  * Le **claim name** – qui correspond au nom de la revendication qui sera récupérée dans la réponse SAML
  * Le **local attribute** – qui détermine la propriété qui sera renseignée avec la valeur de la revendication lors de la création de l'utilisateur PADS4

![Role Mapping9](/images/ctrl-centre/administration/credentials/role-mapping9.png)

* L'attribut local ne peut être choisi que dans une liste de propriétés d'utilisateur PADS4 déterminées, listées ci-dessous

| Attribut local   | Description                                                                                                                        |
| ---------------- | ---------------------------------------------------------------------------------------------------------------------------------- |
| **GroupIds**     | Correspond aux identifiants de groupe récupérés pour la logique d'association de mappage de rôle                                   |
| **FirstName**    | Correspond à la propriété prénom de l'utilisateur PADS4                                                                            |
| **SID**          | Correspond à la propriété nom de l'utilisateur PADS4                                                                               |
| **EmailAddress** | Correspond à une clé unique qui est le lien entre l'utilisateur dans le fournisseur d'identité externe et l'utilisateur dans PADS4 |
| **Name**         | Correspond à la propriété e-mail de l'utilisateur PADS4                                                                            |
| **DisplayName**  | Correspond à la propriété de connexion de l'utilisateur PADS4                                                                      |
| **NFC**          | Correspond à la propriété NFC de l'utilisateur PADS4                                                                               |
