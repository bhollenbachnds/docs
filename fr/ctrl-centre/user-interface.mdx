L'interface d'accès root dans PADS4 est un tableau de bord compact conçu pour l'administration au niveau système. Elle contient généralement trois sections principales : **Diagnostics**, **Migration**, et **Domaines**.

## Diagnostics

La section Diagnostics vous permet de vérifier si le système PADS4 est en bonne santé.

* Elle exécute une série de tests sur les différents services PADS4.
* Les résultats sont affichés par service, afin que vous puissiez voir quel composant fonctionne correctement et lequel nécessite une attention.
* Cela permet d'identifier rapidement les problèmes de configuration ou les pannes de service avant de procéder à d'autres configurations ou modifications.

## Migration

La section Migration est utilisée lorsqu'une étape de migration est requise lors du processus de configuration ou de mise à niveau.

* Elle fournit un aperçu de l'état de la migration et affiche la progression pendant que les données ou les paramètres sont en cours de migration.
* Vous pouvez surveiller si la migration a démarré, est en cours ou s'est terminée avec succès, ce qui permet de s'assurer que l'environnement est prêt avant de continuer à configurer les domaines.

## Domaines

Les domaines PADS4 offrent des capacités de gestion multi-locataires au sein d'un seul serveur PADS4.

* Chaque domaine fonctionne comme un environnement client indépendant avec ses propres utilisateurs, contenus et configurations.
* Depuis l'interface root, vous pouvez créer de nouveaux domaines, modifier le domaine "pads" par défaut, ou gérer les certificats SSL et les paramètres DNS spécifiques aux domaines.
* Cette section est le hub central pour l'administration des domaines au niveau du serveur.
