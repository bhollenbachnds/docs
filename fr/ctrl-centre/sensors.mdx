---
- title: Sensors
---

La **Sensors** fonctionnalité dans PADS4 vous permet d'intégrer **des caméras, des logiciels de gestion de caméras et d'autres appareils** qui génèrent des données en temps réel. Ces données peuvent être utilisées dans PADS4 pour déclencher des **incidents** ou créer des **rules** qui affectent les présentations d'affichage dynamique.

## **Certified Sensors**

Certains capteurs, notamment les caméras, peuvent fournir des données de **mesure d'audience** et de **perception faciale**, notamment :

* Nombre de personnes détectées
* Estimation du groupe d'âge
* Identification du genre
* Attributs faciaux (par exemple, présence d'une barbe)
* Si une personne porte un masque

Les **fonctionnalités disponibles dépendent du modèle de capteur**. Certains appareils nécessitent un **logiciel tiers** pour l'intégration. Pour obtenir la liste des appareils pris en charge, consultez la **page Certified Sensor**.

Ces données de capteur peuvent être utilisées dans PADS4 pour **déclencher des incidents** et **appliquer des rules** dans les présentations en fonction d'informations en temps réel.

## **Aperçu des Sensors dans PADS4**

La section **Overview** affiche tous les **Sensors** ajoutés. Un **indicateur vert** à côté d'un capteur signifie que PADS4 a établi une connexion réussie.

### **Ajout d'un nouveau Sensor**

Pour ajouter un capteur, suivez ces étapes :

1. Cliquez sur **+ New** dans la section **Overview**.
2. Sélectionnez le **type de Sensor** (par exemple, Camera, Sensor Management Software).
3. Choisissez le **modèle spécifique ou le type de logiciel**.
4. (Facultatif) Ajoutez une **description** et un **emplacement** pour faciliter l'identification.
5. Fournissez le **nom d'hôte et les identifiants** pour permettre à PADS4 de se connecter au capteur.

## **Utilisation des Areas pour regrouper les Sensors**

Une **Area** est un **regroupement physique ou logique de plusieurs capteurs**. Cela est utile pour **combiner plusieurs caméras de comptage de personnes** afin de suivre le nombre total de visiteurs sur plusieurs points d'entrée.

### **Création d'une nouvelle Area**

1. Cliquez sur **+ New** dans la section **Area**.
2. Attribuez un **nom** à l'Area.
3. (Facultatif) Fournissez une **description**.
4. Sélectionnez un ou plusieurs **Sensors** à inclure.
   * Les Sensors **peuvent être assignés à plusieurs Areas** si nécessaire.
5. Définissez des **Incident Rules** pour l'Area.
   * Lorsqu'une condition de règle est remplie, **un incident est généré** dans le **PADS4 CTRL Center**.

## **Configuration des Incident Rules pour les Sensors**

Les règles d'incident permettent une **détection automatisée de conditions spécifiques** et déclenchent des alertes. Ces règles peuvent être appliquées aux Areas pour améliorer la sécurité, la conformité ou l'efficacité opérationnelle.

### **Incident Rules disponibles**

| **Règle**                | **Description**                                                                                                                                   |
| ------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------- |
| **No Facemask**          | Si la caméra prend en charge la détection de masque, un incident est créé lorsqu'une personne sans masque est détectée.                           |
| **Maximum People Count** | Déclenche un incident si le nombre de personnes dans une Area dépasse la limite définie.                                                          |
| **Maximum Temperature**  | Déclenche un incident si la température corporelle détectée dépasse un seuil spécifié.                                                            |
| **Milestone Alarms**     | Permet l'intégration avec le **système de gestion vidéo Milestone** pour déclencher des incidents basés sur des conditions de caméra prédéfinies. |

**Remarques importantes :**

* Assurez-vous que les **valeurs de température** correspondent aux **paramètres d'unité (Celsius ou Fahrenheit) de la caméra**.
* Pour les **alarmes Milestone**, définissez le nom de l'alarme dans PADS4 pour qu'il corresponde à celui utilisé dans **Milestone**.

## **Incidents dans PADS4**

Les incidents détectés par les capteurs sont gérés dans le **PADS4 CTRL Center**. Pour plus de détails, consultez la page **How to Use Incidents**.

## **Utilisation des données de Sensor dans PADS4 Designer**

PADS4 permet l'**intégration des données de capteur** dans les présentations d'affichage dynamique. Les options disponibles incluent :

* **Déclencheurs basés sur des événements** provenant de caméras avec reconnaissance faciale détaillée.
* **Agrégation de données au niveau de l'Area** à partir de plusieurs capteurs.
* **Déclencheurs basés sur des incidents** pour des ajustements de contenu dynamiques.

En utilisant les **données de capteur**, vous pouvez créer des expériences d'affichage dynamique **réactives et en temps réel** basées sur **les conditions environnementales réelles**.
