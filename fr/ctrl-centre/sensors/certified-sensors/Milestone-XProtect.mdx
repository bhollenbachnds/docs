Ce guide explique comment intégrer **Milestone XProtect** avec **PADS4** pour récupérer des alarmes. PADS4 lit les données directement depuis la **base de données Milestone SQL Server**, permettant des **notifications push en temps réel** pour des performances optimisées.

Pour plus de détails sur Milestone XProtect, visitez : [www.milestonesys.com](https://www.milestonesys.com).

***

## **Étape 1 : Créer des alarmes dans Milestone XProtect**

Pour permettre à PADS4 de récupérer des alarmes :

1. **Ouvrez XProtect Management Client**.
2. Accédez à la catégorie **Alarms**.
3. **Créez une ou plusieurs définitions d'alarme**.
4. **Spécifiez le déclencheur** qui déclenchera l'alarme (par exemple, détection de ligne de franchissement, alerte de température).
5. Assurez-vous que le **nom de la définition d'alarme** correspond au nom utilisé dans **PADS4 CTRL Center** sous **Sensors** pour déclencher des incidents.

Pour plus de détails sur la création d'incidents, consultez la section **How to Use Sensors**.

***

## **Étape 2 : Configurer SQL Server pour autoriser l'accès à PADS4**

PADS4 ne se connecte pas directement à Milestone ; à la place, il **lit les données depuis la base de données SQL Server**. Cette approche permet à SQL Server d'**envoyer des notifications push** lorsque les données changent, améliorant ainsi les performances du système.

### **Autoriser les connexions réseau SQL Server**

1. Assurez-vous que l'instance SQL Server utilisée par Milestone **autorise les connexions réseau**.
2. Vérifiez les **paramètres du pare-feu** qui pourraient bloquer l'accès.

### **Tester la connectivité SQL Server**

1. Installez **Microsoft SQL Server Management Studio (SSMS)** sur le serveur PADS4.
2. Essayez de **vous connecter à la base de données Milestone** à l'aide de SSMS.
3. Si la connexion est réussie, **SQL Server et les paramètres du pare-feu sont correctement configurés**.

***

## **Étape 3 : Activer l'authentification SQL Server pour PADS4**

Pour permettre à PADS4 de se connecter à SQL Server, activez le **Mode d'authentification mixte** :

1. **Connectez-vous à SQL Server** à l'aide de **SSMS** et de **Windows Authentication**.
2. **Faites un clic droit** sur l'instance SQL Server et **sélectionnez Propriétés**.
3. Accédez à la page **Security**.
4. Sous **Server Authentication**, sélectionnez **SQL Server and Windows Authentication Mode**.
5. **Redémarrez SQL Server** pour que les modifications prennent effet.

***

## **Étape 4 : Créer un compte utilisateur SQL pour PADS4**

Un **compte utilisateur SQL** est requis pour accéder à la base de données Milestone. Vous pouvez en créer un en utilisant **SSMS** (Interface Utilisateur Graphique) ou **des scripts SQL**.

### **Option 1 : Créer un utilisateur SQL avec SSMS**

1. **Se connecter à SQL Server** via SSMS.
2. Développer **Security** > **Logins**.
3. **Faire un clic droit** sur **Logins** et sélectionner **New Login**.
4. **Saisir un nom de connexion** (par ex., `PADS4Reader`).
5. Sélectionner **SQL Server Authentication** et **définir un mot de passe**.
6. **Décocher "Enforce password policy"**.
7. Définir la **base de données par défaut** sur `Surveillance`.

#### **Accorder les autorisations de base de données**

1. Naviguer vers **User Mapping**.
2. Sélectionner la **base de données Surveillance**.
3. Attribuer le rôle **db\_owner** (pour autoriser l'accès complet).
4. Optionnellement, attribuer **db\_denydatawriter** (pour empêcher les modifications).

***

### **Option 2 : Créer un utilisateur SQL à l'aide d'un script**

Vous pouvez également créer un utilisateur avec un **accès en lecture seule** en utilisant le script SQL suivant :

```sql
USE [master]
CREATE LOGIN [PADS4Reader] WITH PASSWORD=N'<password>', DEFAULT_DATABASE=[Surveillance], CHECK_EXPIRATION=OFF, CHECK_POLICY=OFF
USE [Surveillance]
CREATE USER [PADS4Reader] FOR LOGIN [PADS4Reader]
ALTER ROLE [db_datareader] ADD MEMBER [PADS4Reader]
GRANT ALTER ON SCHEMA::[dbo] TO [PADS4Reader]
GRANT CONTROL ON SCHEMA::[dbo] TO [PADS4Reader];
GRANT IMPERSONATE ON USER::[dbo] TO [PADS4Reader];
GRANT CREATE PROCEDURE TO [PADS4Reader]
GRANT CREATE QUEUE TO [PADS4Reader]
GRANT CREATE SERVICE TO [PADS4Reader]
GRANT REFERENCES ON CONTRACT::[http://schemas.microsoft.com/SQL/Notifications/PostQueryNotification] TO [PADS4Reader]
GRANT SUBSCRIBE QUERY NOTIFICATIONS TO [PADS4Reader]
GRANT RECEIVE ON QueryNotificationErrorsQueue TO [PADS4Reader]
```

1. **Remplacer** `<password>` par un mot de passe sécurisé.
2. **Exécuter le script** dans SSMS.

Après l'exécution, le compte `PADS4Reader` est **créé et configuré**.

***

## **Étape 5 : Configurer PADS4 CTRL Center pour l'intégration Milestone**

1. **Ouvrir PADS4 CTRL Center**.
2. **Naviguer vers la section Sensors**.
3. **Ajouter un nouveau capteur Milestone**.
4. Saisir l'**adresse du SQL Server, le nom de la base de données, le nom d'utilisateur (`PADS4Reader`) et le mot de passe**.

Une fois configuré, **PADS4 peut récupérer les alarmes depuis Milestone XProtect**.

***

## **Conclusion**

Cette configuration permet la **récupération des alarmes en temps réel** depuis **Milestone XProtect** en utilisant les **notifications SQL Server**. L'intégration permet la **création automatisée d'incidents** dans PADS4 basée sur les **déclencheurs d'alarmes provenant des caméras connectées**.

Pour le dépannage, vérifier :

* SQL Server **accès réseau**.
* **Paramètres du pare-feu**.
* SQL **mode d'authentification**.
* Autorisations **utilisateur** dans SQL Server.

En suivant ces étapes, votre **système PADS4 surveillera et répondra efficacement aux alarmes Milestone**.
