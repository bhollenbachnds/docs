Ce guide explique comment intégrer **Milestone XProtect** avec **PADS4** pour récupérer des alarmes. PADS4 lit les données directement depuis la **base de données SQL Server de Milestone**, permettant des **notifications push en temps réel** pour des performances optimisées.

Pour plus de détails sur Milestone XProtect, visitez : [www.milestonesys.com](https://www.milestonesys.com).

***

## **Étape 1 : Créer des alarmes dans Milestone XProtect**

Pour permettre à PADS4 de récupérer des alarmes :

1. **Ouvrir XProtect Management Client**.
2. Accéder à la **Alarms** catégorie.
3. **Créer une ou plusieurs définitions d'alarme**.
4. **Spécifier le déclencheur** qui déclenchera l'alarme (par ex., détection de tripwire, alerte de température).
5. S'assurer que le **nom de la définition d'alarme** correspond au nom utilisé dans **PADS4 CTRL Center** sous **Sensors** pour déclencher des incidents.

Pour plus de détails sur la création d'incidents, consultez la section **How to Use Sensors**.

***

## **Étape 2 : Configurer SQL Server pour autoriser l'accès à PADS4**

PADS4 ne se connecte pas directement à Milestone ; à la place, il **lit les données depuis la base de données SQL Server**. Cette approche permet à SQL Server de **envoyer des notifications push** lorsque les données changent, améliorant ainsi les performances du système.

### **Autoriser les connexions réseau SQL Server**

1. Assurez-vous que l'instance SQL Server utilisée par Milestone **autorise les connexions réseau**.
2. Vérifiez les **paramètres du pare-feu** susceptibles de bloquer l'accès.

### **Tester la connectivité SQL Server**

1. Installez **Microsoft SQL Server Management Studio (SSMS)** sur le serveur PADS4.
2. Essayez de **vous connecter à la base de données Milestone** à l'aide de SSMS.
3. Si la connexion est réussie, **SQL Server et les paramètres du pare-feu sont correctement configurés**.

***

## **Étape 3 : Activer l'authentification SQL Server pour PADS4**

Pour permettre à PADS4 de se connecter à SQL Server, activez le **Mode d'authentification mixte**:

1. **Connectez-vous à SQL Server** à l'aide de **SSMS** et de **l'authentification Windows**.
2. **Faites un clic droit** sur l'instance SQL Server et **sélectionnez Propriétés**.
3. Accédez à la page **Sécurité**.
4. Sous **Authentification du serveur**, sélectionnez **Mode d'authentification SQL Server et Windows**.
5. **Redémarrez SQL Server** pour que les modifications prennent effet.

***

## **Étape 4 : Créer un compte utilisateur SQL pour PADS4**

Un **compte utilisateur SQL** est nécessaire pour accéder à la base de données Milestone. Vous pouvez en créer un à l'aide de **SSMS** (Interface graphique utilisateur) ou de **scripts SQL**.

### **Option 1 : Créer un utilisateur SQL à l'aide de SSMS**

1. **Connectez-vous à SQL Server** via SSMS.
2. Développez **Sécurité** > **Connexions**.
3. **Faites un clic droit** sur **Connexions** et sélectionnez **Nouvelle connexion**.
4. **Entrez un nom de connexion** (par ex., `PADS4Reader`).
5. Sélectionnez **Authentification SQL Server** et **définissez un mot de passe**.
6. **Décochez « Appliquer la stratégie de mot de passe »**.
7. Définissez la **base de données par défaut** sur `Surveillance`.

#### **Accorder des autorisations sur la base de données**

1. Accédez à **Mappage des utilisateurs**.
2. Sélectionnez la **base de données de surveillance**.
3. Attribuez le rôle **db\_owner** (pour autoriser un accès complet).
4. Attribuez éventuellement **db\_denydatawriter** (pour empêcher les modifications).

***

### **Option 2 : Créer un utilisateur SQL à l'aide d'un script**

Vous pouvez également créer un utilisateur avec un **accès en lecture seule** à l'aide du script SQL suivant :

```sql
USE [master]
CREATE LOGIN [PADS4Reader] WITH PASSWORD=N'<password>', DEFAULT_DATABASE=[Surveillance], CHECK_EXPIRATION=OFF, CHECK_POLICY=OFF
USE [Surveillance]
CREATE USER [PADS4Reader] FOR LOGIN [PADS4Reader]
ALTER ROLE [db_datareader] ADD MEMBER [PADS4Reader]
GRANT ALTER ON SCHEMA::[dbo] TO [PADS4Reader]
GRANT CONTROL ON SCHEMA::[dbo] TO [PADS4Reader];
GRANT IMPERSONATE ON USER::[dbo] TO [PADS4Reader];
GRANT CREATE PROCEDURE TO [PADS4Reader]
GRANT CREATE QUEUE TO [PADS4Reader]
GRANT CREATE SERVICE TO [PADS4Reader]
GRANT REFERENCES ON CONTRACT::[http://schemas.microsoft.com/SQL/Notifications/PostQueryNotification] TO [PADS4Reader]
GRANT SUBSCRIBE QUERY NOTIFICATIONS TO [PADS4Reader]
GRANT RECEIVE ON QueryNotificationErrorsQueue TO [PADS4Reader]
```

1. **Remplacez** `<password>` par un mot de passe sécurisé.
2. **Exécutez le script** dans SSMS.

Après l'exécution, le compte `PADS4Reader` est **créé et configuré**.

***

## **Étape 5 : Configurer PADS4 CTRL Center pour l'intégration Milestone**

1. **Ouvrez PADS4 CTRL Center**.
2. **Accédez à la section Sensors**.
3. **Ajoutez un nouveau sensor Milestone**.
4. Entrez l'**adresse SQL Server, le nom de la base de données, le nom d'utilisateur (`PADS4Reader`) et le mot de passe**.

Une fois configuré, **PADS4 peut récupérer les alarmes depuis Milestone XProtect**.

***

## **Conclusion**

Cette configuration permet la **récupération des alarmes en temps réel** depuis **Milestone XProtect** à l'aide des **notifications SQL Server**. L'intégration permet la **création automatisée d'incidents** dans PADS4 basée sur les **déclencheurs d'alarmes provenant des caméras connectées**.

Pour le dépannage, vérifiez :

* SQL Server **accès réseau**.
* **Paramètres du pare-feu**.
* SQL **mode d'authentification**.
* Autorisations **utilisateur** dans SQL Server.

En suivant ces étapes, votre **système PADS4 surveillera et répondra efficacement aux alarmes Milestone**.
