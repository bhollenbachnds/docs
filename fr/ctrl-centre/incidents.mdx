Les incidents sont **automatically created** lorsqu'une **defined rule** dans une **Area** est déclenchée. Ces règles surveillent les **sensor data** et génèrent des alertes lorsque des conditions spécifiques sont remplies.

Pour plus de détails sur **setting up sensors and defining rules**, consultez la page **How to Use Sensors**.

### **Comment les incidents sont créés**

* **Incidents are triggered by rules set in Areas** où les capteurs surveillent les données en temps réel.
* Une règle est généralement basée sur **a threshold or limit being exceeded** (par exemple, **maximum people count, high temperature, or no facemask detected**).
* Lorsqu'une condition de règle est remplie, **PADS4 automatically logs an incident**.

***

## **Affichage et gestion des incidents**

La page **Incidents page** fournit un **log of all active incidents** qui n'ont pas encore été résolus.

### **Détails de l'incident**

* Cliquer sur une entrée d'incident affichera des **detailed information** sur l'événement.
* Si une image est associée à l'incident, elle sera affichée dans les détails.

### **Résolution d'un incident**

* Pour résoudre un incident, cliquez sur le bouton **Resolve**.
* Cette action **notifies the system** que le problème a été traité.
* Une fois résolu, l'incident est **removed from the active list**.

***

## **Stockage et récupération des données d'incidents**

* **All incidents are stored in a database** pour **analysis, reporting, or integration** avec d'autres systèmes.
* Les données d'incidents peuvent être **retrieved and used** through:
  * **PADS4 Sensor Data Provider**
  * **Third-party tools** qui se connectent à la base de données

### **Choix d'une base de données**

* Pendant la **initial setup**, vous pouvez sélectionner le **type of database** où les données d'incidents seront stockées.
* Cela garantit que les **historical incidents** peuvent être **analyzed or exported** si nécessaire.

***

## **Points clés**

✔ **Incidents are automatically generated** lorsque des règles prédéfinies sont déclenchées.\
✔ **The Incidents page logs all unresolved incidents** avec des détails et des images.\
✔ **Incidents can be resolved** en un seul clic.\
✔ **Les données d'incident sont stockées dans une base de données** et peuvent être consultées via PADS4 ou des outils externes.
