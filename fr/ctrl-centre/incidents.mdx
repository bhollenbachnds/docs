Les incidents sont **créés automatiquement** lorsqu'une **règle définie** dans une **Area** est déclenchée. Ces règles surveillent les **données des capteurs** et génèrent des alertes lorsque des conditions spécifiques sont remplies.

Pour plus de détails sur **la configuration des capteurs et la définition des règles**, consultez la page **How to Use Sensors**.

### **Comment les incidents sont créés**

* **Les incidents sont déclenchés par des règles définies dans les Areas** où les capteurs surveillent des données en temps réel.
* Une règle est généralement basée sur **un seuil ou une limite dépassée** (par exemple, **nombre maximum de personnes, température élevée ou absence de masque détectée**).
* Lorsqu'une condition de règle est remplie, **PADS4 enregistre automatiquement un incident**.

***

## **Affichage et gestion des incidents**

La page **Incidents** fournit un **journal de tous les incidents actifs** qui n'ont pas encore été résolus.

### **Détails de l'incident**

* Cliquer sur une entrée d'incident affichera **des informations détaillées** sur l'événement.
* Si une image est associée à l'incident, elle sera affichée dans les détails.

### **Résolution d'un incident**

* Pour résoudre un incident, cliquez sur le **Resolve** bouton.
* Cette action **notifie le système** que le problème a été résolu.
* Une fois résolu, l'incident est **retiré de la liste active**.

***

## **Stockage et récupération des données d'incident**

* **Tous les incidents sont stockés dans une base de données** pour **l'analyse, la création de rapports ou l'intégration** avec d'autres systèmes.
* Les données d'incident peuvent être **récupérées et utilisées** through:
  * **PADS4 Sensor Data Provider**
  * **Outils tiers** qui se connectent à la base de données

### **Choix d'une base de données**

* Pendant **la configuration initiale**, vous pouvez sélectionner le **type de base de données** où les données d'incident seront stockées.
* Cela garantit que **les incidents historiques** peuvent être **analysés ou exportés** si nécessaire.

***

## **Points clés**

✔ **Les incidents sont générés automatiquement** lorsque des règles prédéfinies sont déclenchées.\
✔ **La page Incidents enregistre tous les incidents non résolus** avec des détails et des images.\
✔ **Les incidents peuvent être résolus** en un seul clic.\
✔ **Les données d'incident sont stockées dans une base de données** et peuvent être consultées via PADS4 ou des outils externes.
