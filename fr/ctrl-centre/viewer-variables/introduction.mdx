Les variables de visualisation, également connues sous le nom de **Dynamic Filtering**, vous permettent de créer **un contenu d'affichage dynamique intelligent et efficace**. Au lieu de concevoir plusieurs présentations pour différents visualiseurs, vous pouvez utiliser les variables de visualisation pour adapter automatiquement le contenu en fonction de valeurs prédéfinies.

***

## **Exemple de salle de réunion**

Imaginez que vous gérez 50 panneaux de porte de salle de réunion dans votre réseau d'affichage dynamique. Chaque panneau affiche un numéro de salle unique. Traditionnellement, vous auriez besoin de 50 présentations distinctes.

Avec les variables de visualisation, vous concevez une seule présentation et laissez le système insérer dynamiquement le numéro de salle correct pour chaque écran.

***

## **Création d'une variable de visualisation**

### **Steps:**

1. Ouvrez PADS4 CTRL Center.
2. Allez dans la catégorie de menu **Viewers**.
3. Sélectionnez **Variables** dans la section qui s'ouvre.
4. Créez une nouvelle variable de visualisation :
   * Entrez un nom, par exemple, **MeetingRoomName**.
   * Cliquez sur **Create** pour ajouter la variable.
5. Maintenant, chaque **Viewer** peut avoir sa propre valeur unique pour **MeetingRoomName**.

***

## **Conception d'une présentation avec des variables de visualisation**

1. Créez un champ de texte dans PADS4 Designer.
2. Insérez la variable de visualisation comme **#MeetingRoomName#**.
3. Programmez la présentation pour vos Viewers.

Chaque Viewer affichera son **Meeting Room Name** attribué sans nécessiter de présentations séparées.

***

## **Adaptation dynamique du contenu**

Les variables de visualisation ne se limitent pas aux numéros de salle. Vous pouvez les utiliser pour :

* Données basées sur la localisation
* Panneaux directionnels
* Adresses IP
* Heures d'ouverture
* Messages personnalisés

### **Example: SQL Integration**

Si vous récupérez des données d'une base de données, vous pouvez insérer des variables de visualisation dans les requêtes SQL :

```sql
SELECT * FROM Table WHERE [field1] = #MeetingRoomName#
```

Cela garantit que la requête filtre dynamiquement les données en fonction de la variable de visualisation.

***

## **Où les variables de visualisation peuvent-elles être utilisées ?**

* Champs de texte dans les présentations
* Requêtes SQL dans les fournisseurs de données
* Éléments de playlist pour la rotation dynamique du contenu

Les variables de visualisation **ne peuvent pas** être utilisées dans les champs de mot de passe.

***

## **Comment fonctionnent les variables de visualisation**

PADS4 analyse automatiquement tous les textes pour les variables de visualisation (formatées comme `#VariableName#`) et les remplace par les valeurs attribuées.

Si un **meeting room name** change, il suffit de mettre à jour la valeur dans **Viewer Variables**—il n'est pas nécessaire de modifier la présentation.

***

## **Conclusion**

Les variables de visualisation simplifient la gestion du contenu en permettant une personnalisation dynamique sur plusieurs Viewers. Cela garantit un système d'affichage dynamique évolutif, efficace et flexible avec un minimum de mises à jour manuelles.
