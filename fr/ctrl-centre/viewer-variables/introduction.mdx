---
- title: Variables Viewer et filtrage dynamique
- description: Utilisez les Variables Viewer dans PADS4 CTRL Center pour filtrer
    dynamiquement le contenu par affichage, comme les numéros de salle, afin
    qu'une seule présentation serve de nombreux Viewers.
---

Les Variables Viewer, également connues sous le nom de **Filtrage Dynamique**, vous permettent de créer **un contenu d'affichage dynamique intelligent et efficace**. Au lieu de concevoir plusieurs présentations pour différents spectateurs, vous pouvez utiliser les Variables Viewer pour adapter automatiquement le contenu en fonction de valeurs prédéfinies.

***

## **Exemple de salle de réunion**

Imaginez gérer 50 panneaux de porte de salle de réunion dans votre réseau d'affichage dynamique. Chaque panneau affiche un numéro de salle unique. Traditionnellement, vous auriez besoin de 50 présentations distinctes.

Avec les Variables Viewer, vous concevez une seule présentation et laissez le système insérer dynamiquement le numéro de salle correct pour chaque affichage.

***

## **Créer une Variable Viewer**

### **Steps:**

1. Ouvrez PADS4 CTRL Center.
2. Accédez à la **Viewers** catégorie de menu.
3. Sélectionnez **Variables** dans la section qui s'ouvre.
4. Créez une nouvelle Variable Viewer :
   * Entrez un nom, par exemple, **MeetingRoomName**.
   * Cliquez sur **Create** pour ajouter la variable.
5. Désormais, chaque **Viewer** peut avoir sa propre valeur unique pour **MeetingRoomName**.

***

## **Concevoir une présentation avec des Variables Viewer**

1. Créez un champ de texte dans PADS4 Designer.
2. Insérez la Variable Viewer sous la forme **#MeetingRoomName#**.
3. Planifiez la présentation pour vos Viewers.

Chaque Viewer affichera son **Nom de salle de réunion** attribué sans avoir besoin de présentations distinctes.

***

## **Adaptation dynamique du contenu**

Les Variables Viewer ne se limitent pas aux numéros de salle. Vous pouvez les utiliser pour :

* Données basées sur la localisation
* Panneaux directionnels
* Adresses IP
* Horaires d'ouverture
* Messages personnalisés

### **Example: SQL Integration**

Si vous récupérez des données depuis une base de données, vous pouvez insérer des Variables Viewer dans des requêtes SQL :

```sql
SELECT * FROM Table WHERE [field1] = #MeetingRoomName#
```

Cela garantit que la requête filtre dynamiquement les données en fonction de la Variable Viewer.

***

## **Où les Variables Viewer peuvent-elles être utilisées ?**

* Champs de texte dans les présentations
* Requêtes SQL dans les fournisseurs de données
* Éléments de playlist pour la rotation dynamique du contenu

Les Variables Viewer **ne peuvent pas** être utilisées dans les champs de mot de passe.

***

## **Fonctionnement des Variables Viewer**

PADS4 analyse automatiquement tout le texte à la recherche de Variables Viewer (formatées sous la forme `#VariableName#`) et les remplace par les valeurs assignées.

Si un **nom de salle de réunion** change, il suffit de mettre à jour la valeur dans **Variables Viewer**—il n'est pas nécessaire de modifier la présentation.

***

## **Conclusion**

Les Variables Viewer simplifient la gestion du contenu en permettant une personnalisation dynamique sur plusieurs Viewers. Cela garantit un système d'affichage dynamique évolutif, efficace et flexible avec un minimum de mises à jour manuelles.
