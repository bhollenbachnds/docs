Le **PADS4 Configuration** vous permet de définir et de gérer les paramètres système critiques pour votre installation PADS4.

***

![Pads4foundationconfiguration Pn](/images/install-pads4/install-pads4-server/PADS4Foundationconfiguration.png)

## **Configuration d'hébergement**

Spécifiez les paramètres de port pour l'hébergement **PADS4** et activez éventuellement **HTTPS**.

### **Paramètres de port**

* **Port HTTP :** Le port utilisé pour héberger le **PADS4 CTRL Center** (par défaut : **80**).
* **Port HTTPS (SSL) :** Port sécurisé pour la communication **SSL-encrypted** (par défaut : **443**).

### **Configuration SSL**

Pour activer **SSL**, configurez les éléments suivants :

* **Nom d'hôte DNS** – Nom de domaine enregistré.
* **Fichier de certificat** – Fournissez le fichier de certificat SSL (`.certificate`).
* **Fichier de clé privée** – Spécifiez le fichier de clé privée correspondant (`.private`).

### **Paramètres de sécurité supplémentaires**

* **Activer HSTS :** Impose des connexions **HTTPS-only** pour une communication sécurisée.
* **Version TLS :** Choisissez la **TLS version** requise (**1.0, 1.1, 1.2, or 1.3**) lors de l'utilisation de **SSL certificates** avec les domaines PADS4.

### **Configuration du Message Broker**

À partir de **PADS4 version 2022.3**, les utilisateurs peuvent sélectionner le **message broker** préféré :

* **Default:** **NDS Message Broker** (nouveau par défaut à partir de la version **2022.3**).
* **Alternative:** **RabbitMQ** (disponible pour la compatibilité ascendante).

### **Configuration du Reverse Proxy**

À partir de **PADS4 version 2025.1**, les utilisateurs peuvent sélectionner le **reverse proxy.**

* **Default: PADS4 Reverse Proxy** (nouveau par défaut à partir de la version **2025.1**).
* **Alternative: NGinx** (disponible pour la compatibilité ascendante).

***

## **Configuration de l'utilisateur Root**

* **Username:** Par défaut **`su\root`**.
* **Password:** Sélectionnez un mot de passe lors de la mise à niveau vers **PADS4 version 2023.1 or higher**.

***

## **Paramètres du serveur PADS4**

* **Adresse IP :** Par défaut, le **PADS4 Server** fonctionne sur **localhost (127.0.0.1)**. Généralement, **PADS4 Foundation** et **PADS4 Server** sont installés sur la même machine.
* **Port HTTP :** Le port de communication par défaut pour **PADS4 Server** est **9005**.

***

## **Services PADS4 Foundation**

* **Emplacement de stockage personnalisé :** Si nécessaire, modifiez l'**default storage** emplacement pour **PADS4 Foundation Services**.
* **Note:** Ce paramètre **does not** affecte le **content folder**. L'emplacement du **content folder** peut être défini à l'aide de **PADS4 System Management** pour chaque **domain**.

***

## **PADS4 CMS (Legacy)**

* **Langue par défaut :** Définissez la **default language** pour PADS4 CMS.
* **Hôte public (FQDN) :** Abandonné dans **PADS4 2023.1** en raison de l'introduction de **PADS4 Domains**.
* **Configuration de la base de données :** Sélectionnez le type de base de données pour le stockage :
  * **SQLite**
  * **SQL Server**

## **Application des modifications et redémarrage des services**

Après avoir appliqué les modifications de configuration, **PADS4 will restart all services** pour appliquer les nouveaux paramètres.

⚠ **Important:** Le redémarrage des **PADS4 processes** peut prendre jusqu'à **30 secondes**, selon les spécifications de votre matériel.

***

## **Conseils et dépannage**

### **Vérifier que PADS4 Server est en cours d'exécution**

Pour vérifier si **PADS4 Server** est opérationnel :

1. Ouvrez un **navigateur web**.
2. Saisissez l'URL suivante en fonction de vos **paramètres** :

   ```
   http(s)://localhost:<port>  
   ```

   * Utilisez **`http`** ou **`https`** selon votre configuration.
   * Remplacez `<port>` par le **port PADS4 Server** réel (par ex., **80, 443, port personnalisé**).

Si la **page de connexion PADS4 CTRL Center** apparaît, le serveur fonctionne correctement.
