---
- title: PADS4 Configuration
---

L'outil de configuration PADS4 vous permet de définir et de gérer les paramètres système clés de votre installation. Utilisez-le après l'installation pour personnaliser l'hébergement, la sécurité et les services.

<Frame>
  ![Pads4config](/images/install-pads4/PADS4Config.png)
</Frame>

### Configuration de l'hébergement

Définissez les ports d'accès pour l'interface PADS4 et sécurisez-les avec HTTPS.

* **Port HTTP** : Pour PADS4 CTRL Center (par défaut : 80).
* **Port HTTPS (SSL)** : Active la communication sécurisée (par défaut : 443).
* **Activer HSTS** : Redirige automatiquement HTTP vers HTTPS.
* **Version TLS :** Sélectionnez TLS 1.2+ ou 1.3. Il s'agit des versions TLS minimales utilisées. Si TLS 1.3 est disponible, il sera utilisé sur TLS 1.2+

### Configuration du courtier de messages

* **PADS4 Message Broker** Il s'agit du paramètre par défaut applicable à la plupart des installations.
* **RabbitMQ** Ceci est destiné aux installations héritées ou spécifiques

### **Configuration du proxy inverse**

* **PADS4 Reverse Proxy** Il s'agit du paramètre par défaut applicable à la plupart des installations
* **NGinx** Ceci est destiné aux installations héritées ou spécifiques

### Configuration de l'utilisateur root

* **Username:** su/root
* **Password:** Entrez un mot de passe fort pour l'utilisateur su\root ; [requis pour la connexion au système.](https://docs.pads4.com/ctrl-centre/account-creation)

### PADS4 Foundation Services

* **Emplacement de stockage personnalisé :** Si nécessaire, modifiez l'emplacement de **stockage par défaut** pour **PADS4 Foundation Services**.

<Note>
  Ce paramètre **n'affecte pas** le **dossier de contenu**. Le **dossier de contenu** l'emplacement peut être défini à l'aide de **PADS4 System Management** pour chaque **domaine**.
</Note>

### PADS4 CMS (legacy)

Ces paramètres sont destinés à PADS4 CMS 1.0.

* **Langue par défaut :** Définir la langue par défaut de la base de données.
* **Configuration de la base de données :** Sélectionnez le type de base de données pour le stockage :
  * **SQLite** Il s'agit du paramètre par défaut, applicable à la plupart des installations
  * **Microsoft SQL Server**
    * Nom d'hôte et port pour Microsoft SQL Server
    * Nom d'utilisateur et mot de passe pour Microsoft SQL Server

<Note>
  Lors du changement de base de données pendant que PADS4 est en cours d'utilisation, les données existantes ne seront pas migrées. La nouvelle base de données démarre vide.
</Note>

### Sélection des services

Pour améliorer les performances, désactivez les microservices inutilisés sur le PADS4 Server. Les options configurables incluent NDS.PADS4.FIDS.Server, NDS.Services.PowerBI, NDS.Services.Teams.Integration.

<Warning>
  La désactivation peut provoquer une instabilité. Contactez le support en cas de doute.
</Warning>

### Cliquez sur Modifier la base de données des capteurs.

* Sélectionnez le type de base de données que vous souhaitez utiliser. Vous pouvez choisir parmi les options suivantes :
  * SQLite (par défaut)
  * SQL Server
  * PostgreSQL
  * MongoDB
* La base de données sélectionnée est créée

<Note>
  Lors du changement de base de données pendant que PADS4 est en cours d'utilisation, les données existantes ne seront pas migrées. La nouvelle base de données démarre vide.
</Note>

### Enregistrer et redémarrer la configuration

Cliquez sur **Apply** pour enregistrer les paramètres et redémarrer automatiquement les services (environ 30 secondes).
