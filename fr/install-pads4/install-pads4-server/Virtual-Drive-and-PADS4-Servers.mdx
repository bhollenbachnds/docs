# Mise à niveau vers PADS4 Version 2021.3 et supérieure

Avec la sortie de **PADS4 version 2021.3**, un nouveau **Advanced Content Library System** a été introduit. Cette mise à jour améliore les fonctionnalités et la sécurité en passant d'une structure de dossiers ouverte à un environnement contrôlé. Par conséquent, les installations existantes peuvent nécessiter des ajustements mineurs pour s'adapter à ce changement.

## PADS4 Advanced Content Library

À partir de **PADS4 version 2021.3**, la bibliothèque de contenu n'est plus stockée dans le chemin de répertoire traditionnel :

```text
C:\ProgramData\PADS\Content\...
```

Elle est désormais gérée dans un **environnement fermé**, permettant à PADS4 d'introduire des fonctionnalités avancées et une sécurité améliorée tout en maintenant une gestion de contenu fluide.

## PADS4 Virtual Drive sur les serveurs PADS4 (Sub/Backup)

Lors de l'utilisation de **PADS4 Designer**, **Data Folder Data Providers**, **PADS4 CMS**, ou **PADS4 Workspace**, le système utilise désormais un **Virtual Drive** comme passerelle vers le nouvel environnement de contenu fermé.

Pour les installations existantes qui reposent sur le **dossier Content Library** traditionnel, la compatibilité est maintenue grâce à ce mécanisme. Le **Virtual Drive** est automatiquement installé lors du processus de mise à niveau.

### Considérations de performance

Les utilisateurs peuvent constater une légère variation des performances, car le **Virtual Drive** récupère le contenu dynamiquement depuis l'environnement fermé au lieu d'une structure de système de fichiers directe.

## Comment ça fonctionne ?

Lors de l'installation de **PADS4 Server version 2021.3 ou supérieure**, un **Virtual Drive Service** est automatiquement déployé. Ce service assure une intégration transparente avec le nouveau **Advanced Content Library**.

### Navigation dans le Virtual Drive

Après la mise à niveau, vous remarquerez peut-être que le dossier de la bibliothèque de contenu précédent :

```text
C:\ProgramData\PADS\Content\...
```

est désormais représenté comme un **Virtual Folder** (indiqué par une flèche sur l'icône du dossier). Vous pouvez toujours l'ouvrir et interagir avec lui comme avec un dossier normal.

### Fonctionnalité

Bien que le dossier semble contenir des fichiers, aucune donnée réelle n'est stockée localement. Au lieu de cela, le système récupère et affiche dynamiquement les informations sur les fichiers depuis le **PADS4 Server**. Les utilisateurs peuvent :

* **Ajouter, modifier et supprimer des fichiers**
* **Ouvrir des fichiers pour les modifier et enregistrer les modifications directement**
* **Synchroniser automatiquement les modifications avec le PADS4 Server**

De plus, tout nouveau fichier téléchargé via le **PADS4 Web Portal** apparaîtra dans le dossier **Virtual Drive**.

## Emplacement du fichier journal

Pour le dépannage et l'analyse des journaux, le **Virtual Drive Service** maintient un fichier journal dédié. Vous pouvez le trouver ici :

```text
C:\Program Files\NDS\PADS4 Foundation\NDS.Content.FileSystem.Service\NDS.Content.FileSystem.Service.log
```

## API pour la gestion de contenu

Avec **PADS4 version 2021.3**, une nouvelle **Content Management API** a été introduite, remplaçant l'ancien **Agent SDK**. Cette API offre des fonctionnalités et une flexibilité améliorées pour la gestion du contenu par programmation.

Pour plus de détails et des exemples d'implémentation, consultez la **PADS4 API Documentation on GitHub**.

## Content Library et Viewers

Le mécanisme de synchronisation des **Viewers** reste inchangé dans cette mise à jour. Les processus existants de synchronisation et de lecture du contenu continueront de fonctionner comme auparavant.
