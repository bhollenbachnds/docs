# Mise à niveau vers PADS4 version 2021.3 et supérieure

Avec la sortie de **PADS4 version 2021.3**, un nouveau **système de bibliothèque de contenu avancée** a été introduit. Cette mise à jour améliore les fonctionnalités et la sécurité en passant d'une structure de dossiers ouverte à un environnement contrôlé. Par conséquent, les installations existantes peuvent nécessiter des ajustements mineurs pour s'adapter à ce changement.

## Bibliothèque de contenu avancée PADS4

À partir de **PADS4 version 2021.3**, la bibliothèque de contenu n'est plus stockée dans le chemin de répertoire traditionnel :

```text
C:\ProgramData\PADS\Content\...
```

Elle est désormais gérée dans un **environnement fermé**, permettant à PADS4 d'introduire des fonctionnalités avancées et une sécurité améliorée tout en maintenant une gestion de contenu fluide.

## Lecteur virtuel PADS4 sur les serveurs PADS4 (Sub/Backup)

Lors de l'utilisation de **PADS4 Designer**, **fournisseurs de données de dossier de données**, **PADS4 CMS**, ou **PADS4 Workspace**, le système utilise désormais un **lecteur virtuel** comme passerelle vers le nouvel environnement de contenu fermé.

Pour les installations existantes qui reposent sur le **dossier de bibliothèque de contenu** traditionnel, la compatibilité est maintenue grâce à ce mécanisme. Le **lecteur virtuel** est automatiquement installé lors du processus de mise à niveau.

### Considérations de performance

Les utilisateurs peuvent constater une légère variation des performances, car le **lecteur virtuel** récupère le contenu dynamiquement depuis l'environnement fermé au lieu d'une structure de système de fichiers directe.

## Comment cela fonctionne-t-il ?

Lors de l'installation de **PADS4 Server version 2021.3 ou supérieure**, un **service de lecteur virtuel** est automatiquement déployé. Ce service assure une intégration transparente avec la nouvelle **bibliothèque de contenu avancée**.

### Navigation dans le lecteur virtuel

Après la mise à niveau, vous remarquerez peut-être que le dossier de bibliothèque de contenu précédent :

```text
C:\ProgramData\PADS\Content\...
```

est maintenant représenté comme un **dossier virtuel** (indiqué par une flèche sur l'icône du dossier). Vous pouvez toujours l'ouvrir et interagir avec lui comme avec un dossier normal.

### Fonctionnalité

Bien que le dossier semble contenir des fichiers, aucune donnée réelle n'est stockée localement. Au lieu de cela, le système récupère et affiche dynamiquement les informations sur les fichiers depuis le **PADS4 Server**. Les utilisateurs peuvent :

* **Ajouter, modifier et supprimer des fichiers**
* **Ouvrir des fichiers pour les modifier et enregistrer les modifications directement**
* **Synchroniser automatiquement les modifications avec le PADS4 Server**

De plus, tout nouveau fichier téléchargé via le **PADS4 Web Portal** apparaîtra dans le dossier **lecteur virtuel**.

## Emplacement du fichier journal

Pour le dépannage et l'analyse des journaux, le **service de lecteur virtuel** maintient un fichier journal dédié. Vous pouvez le trouver ici :

```text
C:\Program Files\NDS\PADS4 Foundation\NDS.Content.FileSystem.Service\NDS.Content.FileSystem.Service.log
```

## API pour la gestion de contenu

Avec **PADS4 version 2021.3**, une nouvelle **API de gestion de contenu** a été introduite, remplaçant l'ancien **Agent SDK**. Cette API offre des fonctionnalités et une flexibilité améliorées pour la gestion du contenu par programmation.

Pour plus de détails et des exemples d'implémentation, consultez la **documentation de l'API PADS4 sur GitHub**.

## Bibliothèque de contenu et Viewers

Le mécanisme de synchronisation **Viewers** reste inchangé dans cette mise à jour. Les processus existants de synchronisation du contenu et de lecture continueront de fonctionner comme avant.
