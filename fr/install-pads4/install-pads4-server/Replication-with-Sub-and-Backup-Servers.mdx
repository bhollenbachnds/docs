# Changements dans la synchronisation avec PADS4 Version 2021.3

Avec l'introduction de **PADS4 version 2021.3**, le **synchronization mechanism** entre les serveurs PADS4 a été mis à jour. Ce changement peut avoir un impact sur les installations existantes où **PADS4 Sub and/or Backup Servers** sont utilisés.

***

## Nouveau mécanisme de réplication dans PADS4 Version 2021.3

Dans les versions précédentes de PADS4, la réplication suivait un **one-way connection model**, où le **Sub/Backup Server** initiait une connexion au **Main Server** pour la synchronisation.

À partir de **PADS4 version 2021.3**, le processus de réplication a été amélioré. Le **Main Server** et les **Sub/Backup Servers** peuvent désormais tous deux **initiate** la connexion, permettant un processus de synchronisation plus **flexible and resilient**.

Cette mise à jour introduit une **networking requirement** clé :

* Un **host port** doit être **opened** dans le pare-feu pour la communication entre les serveurs.
* Les **Main Servers** et les **Sub/Backup Servers** doivent pouvoir établir une **bidirectional connection** pour que la réplication fonctionne correctement.

***

## Exigences de configuration du pare-feu

À partir de **PADS4 version 2021.3**, la réplication entre serveurs **requires** that:

* Le **Main Server** doit pouvoir initier une connexion avec le **Sub/Backup Server**.
* Le **Sub/Backup Server** doit pouvoir initier une connexion avec le **Main Server**.
* Les **host ports** appropriés doivent être **opened in the firewall** pour permettre le trafic dans les deux sens.

Cela diffère des versions précédentes, où seul le **Sub/Backup Server** était responsable de l'initiation de la connexion au **Main Server**.

***

## **Note importante**

<Note>**Une connexion réseau full-duplex est requise entre le Main Server et les Sub/Backup Servers.** Les deux serveurs doivent pouvoir établir une connexion directe l'un avec l'autre pour une réplication et une synchronisation correctes.</Note>

Cette nouvelle méthode de réplication assure une **more robust and efficient** processus de synchronisation, réduisant les perturbations potentielles dans les opérations d'affichage dynamique.
