# Changements dans la synchronisation avec PADS4 Version 2021.3

Avec l'introduction de **PADS4 version 2021.3**, le **mécanisme de synchronisation** entre les serveurs PADS4 a été mis à jour. Ce changement peut avoir un impact sur les installations existantes où des **PADS4 Sub et/ou Backup Servers** sont utilisés.

***

## Nouveau mécanisme de réplication dans PADS4 Version 2021.3

Dans les versions précédentes de PADS4, la réplication suivait un **modèle de connexion unidirectionnelle**, où le **Sub/Backup Server** initiait une connexion au **Main Server** pour la synchronisation.

À partir de **PADS4 version 2021.3**, le processus de réplication a été amélioré. Le **Main Server** et les **Sub/Backup Servers** peuvent désormais tous deux **initier** la connexion, permettant un processus de synchronisation plus **flexible et résilient**.

Cette mise à jour introduit une **exigence réseau** clé :

* Un **port hôte** doit être **ouvert** dans le pare-feu pour la communication entre les serveurs.
* Les **Main Servers** et les **Sub/Backup Servers** doivent pouvoir établir une **connexion bidirectionnelle** pour que la réplication fonctionne correctement.

***

## Exigences de Configuration du Pare-feu

À partir de **PADS4 version 2021.3**, la réplication entre serveurs **nécessite** that:

* Le **Main Server** doit être capable d'initier une connexion avec le **Sub/Backup Server**.
* Le **Sub/Backup Server** doit être capable d'initier une connexion avec le **Main Server**.
* Les **host ports** appropriés doivent être **ouverts dans le pare-feu** pour permettre le trafic dans les deux directions.

Cela diffère des versions précédentes, où seul le **Sub/Backup Server** était responsable d'initier la connexion au **Main Server**.

***

## **Note Importante**

<Note>**Une connexion réseau full-duplex est requise entre le Main Server et les Sub/Backup Servers.** Les deux serveurs doivent être capables d'établir une connexion directe entre eux pour une réplication et une synchronisation appropriées.</Note>

Cette nouvelle méthode de réplication assure un **processus de synchronisation plus robuste et efficace**, réduisant les perturbations potentielles dans les opérations d'affichage dynamique.
