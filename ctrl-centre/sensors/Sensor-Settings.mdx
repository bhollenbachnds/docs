Sensor settings in PADS4 allow you to manage how **sensor data, images, and incident reports** are stored in the database. You can configure **storage duration**, **image handling**, and **database type** for optimal system performance.

---

## **Configuring Sensor Data Storage**
- **Set the retention period** for sensor data.
- Older data is **automatically deleted** after the specified period.
- If needed, **third-party tools** can be used to **copy and export** database information.

---

## **Managing Sensor Images**
- Define how long **sensor images** should be stored.
- Due to **GPRS regulations**, image storage can be **disabled** for privacy compliance.

---

## **Incident Storage Settings**
- Specify how long **incident data** (generated by sensors) is retained in the database.
- After the defined period, **incidents will be automatically removed**.

---

## **Database Configuration**
### **Default Database**
- **SQLite** is used **by default** for storing sensor data.

### **Changing Database Settings**
- The database configuration can be modified using **NDS.DataStorage.ConfigurationTool.exe**.
- Location:  
  `C:\Program Files\NDS\PADS4 Foundation\NDS.DataStorage.ConfigurationTool`
- Alternatively, you can start this tool from the **Foundation Configuration Application** after installation.

---

## **Conclusion**
PADS4 offers flexible **sensor data management**, ensuring compliance with **storage policies and regulations** while optimizing system **performance and security**.
