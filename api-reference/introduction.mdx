This section describes all APIs available in a PADS4 Foundation deployment. Each API has its own Swagger/OpenAPI documentation accessible via the PADS4 CTRL Center.

> **Note:** The server address in all URLs below uses `https://saas.pads365.com` as an example. Replace this with your own PADS4 server address when accessing the APIs.

---

## How to Access the APIs

Navigate to `https://<your-server>/about` to see all running services. Services with a launch icon link directly to their Swagger UI:

```text
https://<your-server>/rdx/<ServiceName>/swagger/index.html
```

The underlying OpenAPI JSON spec is available at:

```text
https://<your-server>/rdx/<ServiceName>/swagger/v1/swagger.json
```

## Detailed Service Descriptions

### NDS.Messaging.Broker

An internal message broker that facilitates asynchronous communication between PADS4 microservices. This service manages the underlying message bus infrastructure and is not typically called directly by external clients.

---

### NDS.Nginx.Domains

Manages domain configuration for the PADS4 Nginx reverse proxy. Used to add, remove, and validate domain routing for multi-tenant deployments.

**Key endpoints:**

| Method | Path | Description |
| :-- | :-- | :-- |
| GET | `/api/v1/Domain` | List all configured domains |
| POST | `/api/v1/Domain` | Add a new domain |
| DELETE | `/api/v1/Domain` | Remove a domain |
| GET | `/api/v1/Domain/validate` | Validate a domain configuration |
| POST | `/api/v1/Domain/reset` | Reset domain configuration |
| POST | `/api/v1/Domain/restart` | Restart the domain service |

---

### NDS.Services.Authentication

The core identity and access management service. Handles users, roles, groups, claims, and external identity provider integrations (LDAP, SAML, OAuth, SCIM).

**Controllers / functional areas:**

- **Account** — Local account management (login, password reset)
- **User** — Create, read, update, delete users
- **Role** — Role management and assignment
- **Membership** — Group membership operations
- **Claim** — Attribute/claim mapping for identity tokens
- **LdapProvider** — Configure and query LDAP/Active Directory connections
- **ExternalAuthenticationProvider** — Register external identity providers (OAuth, SAML, etc.)
- **ExternalAuthentication** — Initiate external auth flows
- **ExternalUserGroupMapping** / **ExternalUserGroup** — Map external groups to PADS4 roles
- **AuthenticationIdentityProvider** — Identity provider configuration
- **Scim** — SCIM provisioning endpoints (users and groups)
- **ReplicationClient** — Internal replication diagnostics

**Sample endpoints:**

| Method | Path |
| :-- | :-- |
| GET/POST | `/api/v1/User` |
| GET/POST | `/api/v1/Role` |
| POST | `/api/v1/ExternalAuthentication/Logon` |
| GET/POST | `/api/v1/LdapProvider` |
| GET/POST | `/api/v1/Claim/GetAll` |

### NDS.Visitor.Management

A full visitor management system integrated with PADS4 displays. Manages visitor pre-registration, check-in/check-out, meetings, kiosk flows, and administrative reporting.

**Controllers / functional areas:**

- **Auth** — Login/logout for the visitor management system
- **dashboard/visitors** — Full visitor CRUD: register, check in/out, view history, QR codes, invitations (v1 and v2 API)
- **dashboard/visitors/external** — Manage external (non-internal) visitors and their visits
- **kiosk/visitors** — Self-service kiosk flows: validate badge, check in/out, pre-register, self-update
- **Lookup** — Reference data: available hosts and locations
- **settings** — System settings, feature flags, API keys, time zone configuration
- **admin/visitors** — Administrative bulk operations (archive)

**Key endpoints:**

| Method | Path | Description |
| :-- | :-- | :-- |
| POST | `/api/v2/dashboard/visitors` | Register a visitor |
| GET | `/api/v2/dashboard/visitors` | List visitors |

| PUT | `/api/v2/dashboard/visitors/checkout` | Check out a visitor |
| GET | `/api/v2/dashboard/visitors/history` | Visitor history |

| POST | `/api/v2/dashboard/visitors/confirmInvitation` | Confirm an invitation |
| POST | `/api/kiosk/visitors/checkin` | Kiosk self check-in |
| POST | `/api/kiosk/visitors/pre-register` | Kiosk pre-registration |
